## 1. plato-torch → fleet-simulator
**Connection:** **Strong**  
- `plato-torch/src/rooms/deadband_room.py` imports `fleet_simulator` directly:  
  ```python
  from fleet_simulator import FleetSimulator
  ```
- `DeadbandRoom` instantiates a `FleetSimulator` object and calls its `step()` method during training cycles.  
- Shared data: The room passes agent state vectors to the simulator; the simulator returns environment feedback and reward signals.  
- **Integration point:** `DeadbandRoom` uses the simulator to generate P0/P1/P2 constraint training data.

## 2. plato-torch → plato-ensign
**Connection:** **Strong**  
- `plato-torch/src/export/ensign_exporter.py` imports `plato_ensign` modules.  
- `ensign_exporter.py` reads trained room checkpoints, extracts LoRA adapters, and packages them into ensign distributables.  
- Shared data: Room training logs, adapter weights, and metadata (room ID, cycle count, success metrics).  
- **Integration point:** Export pipeline triggered after room training completes; produces `.ensign` files.

## 3. plato-torch → holodeck-rust
**Connection:** **Weak / Indirect**  
- No direct Rust/Python interop observed in current `plato-torch` source.  
- However, `holodeck-rust/README.md` mentions “sentiment‑aware NPCs that can be driven by PLATO room outputs.”  
- **Potential integration:** Plato rooms could generate NPC dialogue or behavior scripts that are sent to holodeck via a message queue (e.g., Redis) or REST API—but no such bridge is currently implemented.

## 4. fleet-simulator → holodeck-rust
**Connection:** **None observed**  
- `fleet-simulator` is a Python‑based training environment; `holodeck-rust` is a standalone MUD server.  
- No imports, configuration references, or shared protocols found in either codebase.  
- **Gap:** The simulator does not currently drive or interact with the holodeck’s NPCs or world state.

## 5. fleet-simulator → plato-ensign
**Connection:** **Indirect via plato-torch**  
- The simulator feeds data to `DeadbandRoom`, which is then exported by `plato‑ensign`.  
- No direct simulator‑to‑ensign pipeline exists.

## 6. holodeck-rust → plato-ensign
**Connection:** **None observed**  
- `holodeck‑rust` does not produce LoRA adapters or room checkpoints.  
- **Gap:** Holodeck interaction logs are not currently captured as training tiles for ensign export.

## 7. plato-ensign → holodeck-rust
**Connection:** **None observed**  
- Ensigns are LoRA adapters for inference‑time specialization; no code hooks found to load them into the Rust‑based MUD.
