## 1. Plato-Torch (Training Rooms)  
**Location:** `plato-torch/`  
**Core Purpose:** Provides 25 preset training rooms and self-training room framework.  
**Key Files:**  
- `presets/deadband_room.py` — Deadband Protocol training room  
- `presets/__init__.py` — Exports all presets  
- `room.py` — Base Room class  
- `trainer.py` — Training loop logic  

**Connections Found:**  
- **To Plato-Ensign:** `presets/deadband_room.py` imports `from plato_ensign import export_lora`. Direct call: `export_lora(room_state, "deadband_lora")`.  
- **To Fleet-Simulator:** `presets/deadband_room.py` imports `from fleet_simulator import FleetSim`. Used to generate simulated fleet interaction data for training.  
- **To Holodeck-Rust:** No direct import found in plato-torch code. No Rust bindings or API calls visible.  

**Integration Points:**  
- `DeadbandRoom` uses `FleetSim` to create training episodes.  
- `DeadbandRoom` can export its trained state to a LoRA via `plato_ensign`.  
- Rooms are configured via `config/room_config.yaml` — references `fleet_simulator_config_path`.

## 2. Plato-Ensign (LoRA Export)  
**Location:** `plato-ensign/`  
**Core Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `export.py` — Main export logic  
- `lora_builder.py` — Builds LoRA from room state  
- `config.yaml` — Export configuration  

**Connections Found:**  
- **To Plato-Torch:** `export.py` imports `from plato_torch.room import Room`. Called by DeadbandRoom preset.  
- **To Holodeck-Rust:** No import or reference found.  
- **To Fleet-Simulator:** No direct import.  

**Integration Points:**  
- Accepts a `Room` object from plato-torch, extracts training buffers, creates LoRA weights.  
- Outputs `.safetensors` files to `exports/` directory.

**Integration Points:**
