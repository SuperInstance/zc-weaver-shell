## 1. **plato-torch** → **plato-ensign** → **fleet-simulator** → **holodeck-rust**

### Verified Connections (What's Connected)

#### **plato-torch → plato-ensign**
- **Connection:** Direct export pipeline.
- **Evidence:** `plato-torch/src/rooms/deadband_room.py` contains `export_to_ensign()` method.
- **Mechanism:** Room training loops accumulate interaction tiles. `export_to_ensign()` serializes room state and experience into a LoRA adapter via `plato-ensign` library calls.
- **Data Flow:** Tiles → Room State → LoRA Adapter (.safetensors).

#### **plato-torch → fleet-simulator**
- **Connection:** Training data generation.
- **Evidence:** `fleet-simulator/scripts/generate_training_data.py` imports `from plato_torch.rooms import DeadbandRoom`.
- **Mechanism:** Simulator uses `DeadbandRoom` as an environment to generate agent interaction episodes. These episodes become synthetic training tiles.
- **Data Flow:** Simulated agent actions → Room interactions → Tile logs.

#### **fleet-simulator → holodeck-rust**
- **Connection:** Sentiment-aware NPC behavior data.
- **Evidence:** `holodeck-rust/src/npc/behavior.rs` references `fleet_simulator::agent_profiles` in comments (planned integration). No direct import found in current `Cargo.toml`.
- **Mechanism:** *Planned*: Simulator generates agent personality profiles; holodeck uses them to parameterize NPC sentiment models.
- **Status:** **Documented intent, not wired.**

#### **plato-ensign → (External)**
- **Connection:** Standalone LoRA exporter.
- **Evidence:** `plato-ensign/src/lib.rs` provides `EnsignBuilder` struct. No internal fleet imports except `serde`, `tokio`.
- **Mechanism:** Library called by `plato-torch` or other systems to save adapters. Not directly consuming from other fleet components.

## 2. **Integration Gaps (What's Not Connected)**

#### **Gap 1: holodeck-rust → plato-torch**
- **Issue:** No direct link from holodeck's MUD interactions to PLATO training rooms.
- **Evidence:** `holodeck-rust` does not import `plato_torch`. Its `Cargo.toml` lists `serde`, `tokio`, `websocket`, but no Python bindings or GRPC client for PLATO.
- **Consequence:** NPC interactions in holodeck do not generate training tiles for room learning.

#### **Gap 2: plato-ensign → fleet-simulator**
- **Issue:** Simulator does not load trained ensigns (LoRA adapters) to influence agent behavior.
- **Evidence:** `fleet-simulator` scripts generate data but do not import `plato_ensign` to apply adapters.
- **Consequence:** One-way data flow: simulator → tiles → ensign, but no ensign → simulator feedback loop.

#### **Gap 3: holodeck-rust → plato-ensign**
- **Issue:** No mechanism to export NPC sentiment models as ensigns.
- **Evidence:** `holodeck-rust/src/lib.rs` has no export functionality; NPC state is internal.
- **Consequence:** Specialized NPC behavior (sentiment-aware) cannot be captured as a portable LoRA.

#### **Gap 4: Shared Configuration & Service Discovery**
- **Issue:** Each component uses separate config files; no central service registry.
- **Evidence:**
  - `plato-torch/config/rooms.yaml`
  - `holodeck-rust/config/server.toml`
  - `fleet-simulator/config/simulation.yaml`
- **Consequence:** Manual configuration required for cross-component setups; no dynamic discovery.
