## 1. **plato-torch** → **plato-ensign**
**Connection:** **Strong** — Direct export pipeline.
- **plato-torch/src/rooms/deadband_room.py** includes:
  ```python
  from plato_ensign.export import export_room_to_lora
  ```
- `export_room_to_lora()` is called in `DeadbandRoom.save_training_tiles()`.
- **plato-ensign/src/lib.rs** defines `export_room_to_lora()` which reads `.pt` tile files from `plato-torch/data/tiles/` and outputs a LoRA adapter.
- **Data flow:** Tiles (PyTorch tensors) → LoRA weights (safetensors).

**Integration Gap:** None observed. This is a working, documented export path.

## 2. **plato-torch** → **fleet-simulator**

- **Data flow:** One-way: `fleet-simulator` reads tile files generated by `plato-torch`.

**Integration Gap:** No direct API or real-time connection. The simulator pulls static tile files; no feedback loop.
