## 1. Verified Connections (P1 Safe Channels)

### **plato-torch ↔ fleet-simulator**
- **Connection Type:** Data pipeline (training tiles → simulation)
- **Evidence:** `fleet-simulator/src/tile_ingest.rs` imports `plato_torch::tile::Tile`
- **Flow:** `Tile` structs from plato-torch rooms are ingested into fleet-simulator as training data
- **Status:** **ACTIVE** — Direct Rust dependency, compiled together

### **plato-torch ↔ plato-ensign**
- **Connection Type:** Export pipeline (room experience → LoRA adapters)
- **Evidence:** `plato-ensign/src/export.rs` calls `plato_torch::room::Room::export_experience()`
- **Flow:** Room accumulates tiles → plato-ensign exports to LoRA format
- **Status:** **ACTIVE** — Direct sibling modules in same workspace

### **holodeck-rust ↔ fleet-simulator**
- **Connection Type:** Shared data structures (NPC sentiment)
- **Evidence:** Both use `serde` for JSON serialization of `SentimentEvent` structs
- **Flow:** fleet-simulator can replay holodeck interactions as training scenarios
- **Status:** **PASSIVE** — No direct imports, but compatible formats

## 2. Integration Gaps (P0 Negative Space)

### **Missing: holodeck-rust → plato-torch**
- **Gap:** Holodeck NPC interactions not feeding into plato-torch rooms as tiles
- **Impact:** Sentiment-aware NPC experiences lost for ensign training
- **Root Cause:** No `Tile` generation from holodeck events
- **P0 Warning:** Don't assume holodeck data automatically becomes training tiles

### **Missing: fleet-simulator → holodeck-rust**
- **Gap:** Simulation results not driving holodeck NPC behavior
- **Impact:** Can't test fleet strategies in holodeck environment
- **Root Cause:** No event bridge between systems
- **P0 Warning:** Don't wire simulation outputs directly without validation layer

### **Missing: plato-ensign → holodeck-rust**
- **Gap:** Trained ensigns not deployable to holodeck NPCs
- **Impact:** Can't test specialized AI models in interactive environment
- **Root Cause:** Different model formats (LoRA vs. runtime weights)
- **P0 Warning:** Don't attempt direct model conversion without adapter
