## 1. plato-torch → fleet-simulator
**Connection:** ✅ **Connected via `fleet-simulator` as a data source.**  
- **File:** `plato-torch/src/rooms/deadband_room.py`  
  - Line 23: `from fleet_simulator.simulator import FleetSimulator`  
  - Line 84: `self.simulator = FleetSimulator(config_path="fleet_simulator/configs/training.json")`  
- **Usage:** The `DeadbandRoom` initializes a `FleetSimulator` instance to generate training episodes.  
- **Data Flow:** `fleet-simulator` produces episodic simulation data (agent states, actions, rewards) which `plato-torch` ingests as training tiles.  
- **Integration Gap:** The simulator config path is hardcoded. No dynamic configuration passing from room to simulator.
