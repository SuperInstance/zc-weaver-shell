## 1. Verified Connections (What's Connected)

### plato-torch → plato-ensign
- **Connection:** Direct dependency via Python imports.
- **Evidence:** `plato-torch/ensign_exporter.py` imports `from plato_ensign.lora_exporter import export_room_to_lora`.
- **Data Flow:** Room training logs (tiles) from plato-torch are exported as LoRA adapters via plato-ensign.
- **Integration Point:** `export_room_to_lora()` function call with room ID and tile batch.

### plato-torch → fleet-simulator
- **Connection:** Indirect via shared data schema.
- **Evidence:** `plato-torch/data/tile_schema.py` defines `TrainingTile` class with fields: `agent_id`, `timestamp`, `action`, `reward`, `state`.  
  `fleet-simulator/src/simulation/tile_generator.rs` uses identical field names in `SimulatedTile` struct.
- **Data Flow:** No runtime connection observed. Both use same tile structure for training data generation.
- **Integration Point:** Schema alignment allows potential future ingestion of simulated tiles into plato-torch rooms.

### holodeck-rust → (none observed)
- **Connection:** No direct imports to other fleet components found.
- **Evidence:** `holodeck-rust/Cargo.toml` shows only standard Rust dependencies (serde, tokio, etc.). No references to plato-torch, fleet-simulator, or plato-ensign in source files.
- **Note:** May operate as standalone MUD server. Sentiment-aware NPC system appears self-contained.

### fleet-simulator → (none observed)
- **Connection:** No outgoing dependencies found.
- **Evidence:** `fleet-simulator/Cargo.toml` contains only simulation and math crates. No references to other fleet repos in `src/`.
- **Note:** Generates synthetic training tiles but not currently connected to plato-torch ingestion pipeline.

### plato-ensign → (plato-torch only)
- **Connection:** Single incoming connection from plato-torch.
- **Evidence:** `plato-ensign/setup.py` lists no dependencies beyond PyTorch and transformers.
- **Data Flow:** Receives room data from plato-torch, outputs LoRA adapters to filesystem.

## 2. Integration Gaps (What's Not Connected)

### Gap 1: holodeck-rust Isolation
- **Issue:** No connection to training pipeline (plato-torch) or simulation (fleet-simulator).
- **Impact:** NPC interactions and sentiment data not captured as training tiles.
- **Potential Integration:** Add tile exporter from holodeck to plato-torch room.

### Gap 2: fleet-simulator → plato-torch Pipeline Missing
- **Issue:** Simulated tiles generated but not ingested into active training rooms.
- **Impact:** Synthetic training data unused.
- **Potential Integration:** Add `SimulatedTile` to `TrainingTile` converter and plato-torch import API.

### Gap 3: No Shared Configuration Layer
- **Issue:** Each component has separate config (YAML, TOML, Python dicts).
- **Impact:** Inconsistent hyperparameters, manual synchronization required.
- **Potential Integration:** Central fleet config service or shared config schema.

### Gap 4: plato-ensign Output Not Consumed
- **Issue:** LoRA adapters exported to disk but no downstream consumption observed.
- **Impact:** Trained ensigns not automatically deployed.
- **Potential Integration:** Connect to cudaclaw runtime or model serving layer.
