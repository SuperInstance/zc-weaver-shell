# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 407  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch → fleet-simulator
**Connection:** Strong, direct integration via `SimulationRoom` preset.  
**Evidence:**  
- `plato-torch/presets/SimulationRoom.py` imports `fleet_simulator` and uses `FleetSimulator` class.  
- `SimulationRoom` config includes `simulator_config` path pointing to fleet-simulator's `config/`.  
- Data flow: SimulationRoom generates training episodes → exports as `.pt` tiles → consumed by fleet-simulator for agent training.  
**Gap:** No observed reverse connection (fleet-simulator does not import plato-torch). Fleet-simulator appears to be a downstream consumer.

## 2. plato-torch → holodeck-rust
**Connection:** Indirect, via shared tile format and planned integration.  
**Evidence:**  
- `holodeck-rust/src/protocol/tile.rs` defines `Tile` struct with fields matching plato-torch's `Tile` in `plato-torch/core/tile.py`.  
- `holodeck-rust/README.md` mentions "sentiment-aware NPCs trained from PLATO tiles".  
- No direct imports found in code. Integration appears planned but not yet wired.  
**Gap:** No live data pipeline. Tiles from plato-torch are not automatically ingested by holodeck-rust.

## 3. plato-torch → plato-ensign
**Connection:** Direct export pipeline.  
**Evidence:**  
- `plato-ensign/export.py` imports `torch` and `peft` (LoRA).  
- `plato-torch/rooms/__init__.py` includes `EnsignExporter` class.  
- Process: Room accumulates tiles → triggers export → generates LoRA adapter (.bin file).  
**Gap:** Export is manual/trigger-based, not automatic per training cycle.

## 4. fleet-simulator → holodeck-rust
**Connection:** None observed.  
**Evidence:**  
- No shared data structures in `fleet-simulator/src/`.  
- No references to "holodeck" in fleet-simulator code.  
- Both are independent data consumers from plato-torch.  
**Gap:** No integration. They operate in parallel silos.

## 5. holodeck-rust → plato-ensign
**Connection:** None observed.  
**Evidence:**  
- `plato-ensign` only imports from plato-torch and PyTorch.  
- `holodeck-rust` is a Rust MUD server with no Python bindings.  
**Gap:** Holodeck NPCs cannot be exported as ensigns currently.

## 6. fleet-simulator → plato-ensign
**Connection:** None observed.  
**Evidence:**  
- `plato-ensign` does not import fleet-simulator.  
- Fleet-simulator outputs simulation logs, not LoRA adapters.  
**Gap:** Simulation-trained agents cannot be directly exported as ensigns.
