## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets, self-training rooms, tile processing.

### Verified Connections:
- **plato-ensign:** Direct import in `plato-torch/src/export.rs`. Exports room experience (tile data) to LoRA adapters via `plato_ensign::export_lora()`.
- **fleet-simulator:** Referenced in `plato-torch/config/training_presets.toml` under `data_source = "fleet-simulator"` for preset `deadband_room`.
- **holodeck-rust:** No direct import found. Configuration file `plato-torch/config/holodeck_bridge.toml` exists but is commented out (placeholder).

### Integration Gaps:
- **holodeck-rust:** Configuration present but inactive. No code imports or active data flow.
- **plato-relay:** Not referenced. DeadbandRoom preset does not specify a relay endpoint.

## 2. fleet-simulator
**Location:** `fleet-simulator/`  
**Purpose:** Generate synthetic training data for fleet scenarios.

### Verified Connections:
- **plato-torch:** Provides data via `fleet-simulator/output/tiles/`. Referenced in plato-torch config as data source.
- **holodeck-rust:** No direct import. Output format (JSON tiles) is generic and could be consumed, but no explicit bridge.

### Integration Gaps:
- **holodeck-rust:** No active pipeline. Simulator tiles are not automatically ingested by holodeck.
- **plato-ensign:** No direct export path from simulator to ensign.

## 3. holodeck-rust
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs, interactive training environment.

### Verified Connections:
- **GhostInjector:** Referenced in `holodeck-rust/src/ghost_injector.rs` (module exists). Function `inject_ghost_tile()` reads from `memory/ghost_tiles/` but currently logs a stub.
- **plato-torch:** No active import. The `holodeck_bridge.toml` in plato-torch is not read by holodeck.

### Integration Gaps:
- **plato-torch:** No live data exchange. Holodeck does not consume training tiles from plato-torch rooms.
- **fleet-simulator:** No automatic ingestion of simulator-generated scenarios.
- **GhostInjector:** Implementation is incomplete (`unimplemented!()`).

## 4. plato-ensign
**Location:** `plato-ensign/`  
**Purpose:** Export room experience (tile collections) into LoRA adapters for specialist models.

### Verified Connections:
- **plato-torch:** Directly called from plato-torch's export module. Accepts tile dataset paths and outputs LoRA weights.
- **cudaclaw:** Referenced in `plato-ensign/src/cuda_backend.rs` for GPU-accelerated training (optional CUDA kernel).

### Integration Gaps:
- **holodeck-rust:** No export path from holodeck interactions to ensign.
- **fleet-simulator:** Simulator tiles cannot be directly converted to LoRA; requires plato-torch as intermediary.

## 5. GhostInjector
