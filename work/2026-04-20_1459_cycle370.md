## 1. plato-torch → fleet-simulator
**Connection:** Strong, direct dependency.  
- `plato-torch/rooms/deadband_room.py` imports `from fleet_simulator import FleetSimulator`.  
- The `DeadbandRoom` class instantiates a `FleetSimulator` object (`self.simulator`) and calls its `step()` method during room updates.  
- Data flow: Room passes agent actions to simulator; simulator returns state observations.  
- **Integration point:** `DeadbandRoom` uses the simulator as its environment backend. This is a core, wired connection.

## 2. plato-torch → plato-ensign
**Connection:** Optional export pathway.  
- `plato-torch/training/trainer.py` contains a function `export_to_ensign()` that saves LoRA adapters.  
- The export is triggered by a CLI flag `--export-ensign` in `plato-torch/scripts/train.py`.  
- Data flow: After training, room experience (weights) can be serialized as a `.safetensors` file for plato-ensign.  
- **Integration point:** One‑way export; plato‑ensign does not import plato‑torch. The connection is a file‑based handoff.

## 3. holodeck-rust → (plato-torch, fleet-simulator, plato-ensign)
**Connection:** None observed in code.  
- `holodeck-rust/src/` shows no Rust `use` statements for Python modules.  
- No Python‑side imports of `holodeck_rust` found in plato‑torch or fleet‑simulator.  
- **Integration gap:** Holodeck is isolated. The “Wire GhostInjector into holodeck” task implies a planned but not yet implemented bridge.

## 4. fleet-simulator → plato-ensign
**Connection:** None observed.  
- `fleet-simulator/` does not import or reference plato‑ensign.  
- Simulator outputs are raw state tensors, not ensign‑compatible adapters.  
- **Integration gap:** Simulator experience is not directly exported to ensigns; only room‑trained weights are.

## 5. plato-ensign → Others
**Connection:** Stand‑alone tool.  
- `plato-ensign/ensign/cli.py` only imports PyTorch and Hugging Face libraries.  
- It loads `.safetensors` files (produced by plato‑torch) and applies them to base models.  
- **Integration point:** Passive consumer of exported room adapters.

## 6. Ghost Tiles & Deadband Protocol
**Connection:** Conceptual, not yet wired.  
- `plato-torch/rooms/deadband_room.py` implements the P0/P1/P2 deadband loop.  
- “Ghost tiles” are mentioned in fleet‑context but not found as a code module.  
- **Integration gap:** No `GhostInjector` class or import found; likely a planned feature.
