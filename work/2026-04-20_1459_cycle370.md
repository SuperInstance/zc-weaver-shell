## 1. **plato-torch** → **fleet-simulator**  
**Connection:** **Strong** (direct import)  
- **File:** `plato-torch/src/rooms/deadband_room.py`  
- **Evidence:**  
  ```python
  from fleet_simulator.simulator import FleetSimulator
  from fleet_simulator.agents import ZeroclawAgent
  ```
- **Usage:**  
  - `DeadbandRoom` instantiates a `FleetSimulator` as `self.simulator`.  
  - Calls `self.simulator.step()` to generate training episodes.  
  - Uses `ZeroclawAgent` as the agent class within the simulator.  
- **Data Flow:** Simulator steps produce `(state, action, reward, next_state)` tuples → stored as training tiles in the room.  
- **Integration Gap:** None observed; this is a core, working dependency.

## 3. **plato-torch** → **plato-ensign**  
**Connection:** **Moderate** (export pipeline)  
- **File:** `plato-torch/src/export/ensign_exporter.py`  
- **Evidence:**  
  ```python
  from plato_ensign.lora_trainer import LoRATrainer
  from plato_ensign.config import EnsignConfig
  ```
- **Usage:**  
  - `EnsignExporter` class loads a trained room’s tile history.  
  - Converts tiles into a LoRA-friendly dataset (`instructions`, `responses`).  
  - Calls `LoRATrainer.train()` to produce a `.safetensors` adapter file.  
- **Data Flow:** Room tiles → dataset → LoRA training → ensign model file.  
- **Integration Gap:** The export is optional and manual; no automatic trigger when a room reaches a certain tile count.
