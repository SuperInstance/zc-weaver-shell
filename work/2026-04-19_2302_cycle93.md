## 1. Verified Current Connections

Based on reading the actual fleet repos (`plato-torch`, `fleet-simulator`, `holodeck-rust`, `plato-ensign`), here are the **verified** connections:

### **plato-torch → plato-ensign**
- **Connection:** Direct export pipeline  
- **How:** `plato-torch` rooms generate `.tile` files (training interactions). `plato-ensign` ingests these tiles via `ensign/loader.py` and trains LoRA adapters.  
- **Evidence:** `plato-ensign/README.md` states: "Exports room experience from plato-torch into LoRA adapters."  
- **Status:** **Connected** — functional pipeline.

### **plato-torch → fleet-simulator**
- **Connection:** Data source for simulation  
- **How:** `fleet-simulator` can load tile data from `plato-torch` rooms to generate simulated agent interactions.  
- **Evidence:** `fleet-simulator/src/data.rs` has a `TileLoader` struct that reads `.tile` files.  
- **Status:** **Connected** — one‑way data flow.

### **holodeck-rust → plato-torch**
- **Connection:** NPC sentiment data as training tiles  
- **How:** `holodeck-rrust` generates NPC interaction logs that can be formatted as `.tile` files for `plato-torch` rooms.  
- **Evidence:** `holodeck-rust/src/npc/sentiment.rs` includes `export_to_tile()` function.  
- **Status:** **Connected** — experimental, not yet in production pipeline.

### **plato-ensign → fleet-simulator**
- **Connection:** Trained LoRA adapters used to guide simulated agents  
- **How:** `fleet-simulator` can load a trained ensign (LoRA) to influence agent behavior in simulations.  
- **Evidence:** `fleet-simulator/src/agents/mod.rs` references `load_lora_adapter()`.  
- **Status:** **Connected** — optional, used in advanced simulations.

## 2. Integration Gaps (Actionable)

### **Gap 1: holodeck-rust → plato-ensign (direct)**
- **Issue:** No direct pipeline from holodeck NPC interactions to ensign training.  
- **Impact:** NPC sentiment data must first become plato‑torch tiles, then ensign tiles — extra steps.  
- **Action:** Write a `holodeck_to_ensign` bridge that converts NPC logs directly to ensign‑ready format.

### **Gap 2: fleet-simulator → holodeck-rust (bi‑directional)**
- **Issue:** Simulations cannot currently drive holodeck NPCs in real‑time.  
- **Impact:** Cannot test NPC reactions to simulated fleet scenarios.  
- **Action:** Create a `SimulationDriver` plugin for holodeck that consumes fleet‑simulator event streams.

### **Gap 3: plato-ensign → holodeck-rust (feedback)**
- **Issue:** Trained ensigns are not used to improve NPC behavior in holodeck.  
- **Impact:** NPCs remain static, not learning from fleet training.  
- **Action:** Add an `EnsignNPC` module in holodeck that loads a LoRA adapter to modulate NPC responses.

### **Gap 4: DeadbandRoom → plato-relay (per task board)**
- **Issue:** DeadbandRoom (a plato‑torch preset) does not broadcast its P0/P1/P2 state to the fleet via plato‑relay.  
- **Impact:** Fleet cannot see deadband protocol in action across rooms.  
- **Action:** Integrate `plato-relay` client into DeadbandRoom to publish state changes.
