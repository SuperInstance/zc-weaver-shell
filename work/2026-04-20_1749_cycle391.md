## Integration Gaps (What’s Not Connected)

1. **fleet‑simulator → plato‑torch**  
   - Simulator tiles are JSON; plato‑torch uses Python pickle by default.  
   - No automatic ingestion pipeline.

2. **holodeck‑rust → Any other component**  
   - Standalone MUD server. No tile export, no integration with training rooms.

3. **plato‑ensign ← fleet‑simulator / holodeck‑rust**  
   - Ensign only receives data from plato‑torch rooms. Simulation/NPC data not used for LoRA training.

4. **GhostInjector (mentioned in backlog)**  
   - Not found in current repos. Likely planned component to inject ghost tiles into holodeck or training rooms.
