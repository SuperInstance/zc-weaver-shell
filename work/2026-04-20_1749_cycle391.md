## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** Provides 25 training room presets and self-training rooms.  
**Key Files:**  
- `presets/deadband_room.py` — Deadband Protocol training room.  
- `rooms/base_room.py` — Abstract base class for all rooms.  
- `trainers/self_trainer.py` — Self-training loop.  
- `data/tile_loader.py` — Loads interaction tiles for training.

**Connections Found:**  
- **To plato-ensign:** `plato-torch/export/ensign_exporter.py` imports `from plato_ensign.core import LoRAExporter`. Direct export path.  
- **To fleet-simulator:** `plato-torch/data/simulated_tile_generator.py` imports `from fleet_simulator.scenarios import DeadbandScenario`. Uses simulator to generate training tiles.  
- **To holodeck-rust:** No direct imports found. Potential integration via tile format (JSON).  
- **Internal:** Rooms use `tile_loader` to read `.tile` files from `memory/` directories.

## 2. fleet-simulator (Fleet Simulation)
**Location:** `fleet-simulator/`  
**Purpose:** Generates training data via simulated fleet interactions.  
**Key Files:**  
- `scenarios/deadband_scenario.py` — Simulates P0/P1/P2 decision trees.  
- `agents/sim_agent.py` — Base agent class for simulations.  
- `output/tile_writer.py` — Writes simulation results as `.tile` files.

**Connections Found:**  
- **To plato-torch:** `fleet-simulator/output/tile_writer.py` writes tiles in same format as `plato-torch/data/tile_loader.py` expects.  
- **To holodeck-rust:** No direct imports. Simulation agents could be mirrored as NPCs, but no code link found.  
- **To plato-ensign:** No direct link. Simulated tiles could be exported indirectly via plato-torch.

## 3. holodeck-rust (MUD Server)
**Location:** `holodeck-rust/`  
**Purpose:** Multi‑user dungeon with sentiment‑aware NPCs.  
**Key Files:**  
- `src/npc/sentiment_engine.rs` — NPC emotional state machine.  
- `src/world/tile_map.rs` — World tile storage (different from Plato tiles).  
- `src/protocol/websocket.rs` — WebSocket server for clients.

**Connections Found:**  
- **To plato-torch:** None. No Rust‑Python bindings observed.  
- **To fleet-simulator:** None.  
- **To plato-ensign:** None.  
- **Potential Integration Point:** NPCs could be trained from Plato room tiles (sentiment data), but no importer exists.

## 4. plato-ensign (LoRA Exporter)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `core/lora_exporter.py` — Converts room checkpoints to LoRA.  
- `adapters/room_adapter.py` — Maps room state to adapter config.  
- `utils/merge_loras.py` — Merges multiple LoRAs.

**Connections Found:**  
- **To plato-torch:** Direct import in `plato-torch/export/ensign_exporter.py`.  
- **To fleet-simulator:** None.  
- **To holodeck-rust:** None.  
- **Internal:** Reads `.safetensors` files from `plato-torch/checkpoints/`.
