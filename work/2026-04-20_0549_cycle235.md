# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 235  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch → fleet-simulator
**Connection:** Strong, direct integration via `SimulationRoom` preset.  
**Evidence:**  
- `plato-torch/src/rooms/simulation_room.py` imports `fleet_simulator` and `fleet_simulator.api`.  
- `SimulationRoom` inherits from `BaseRoom` and implements `step()` that calls `fleet_simulator.step()`.  
- Shared data structure: `SimulationState` (dict with agents, positions, tasks) passed between them.  
**Gap:** No observed two-way callback for simulation events to influence room training logic.

## 2. plato-torch → holodeck-rust
**Connection:** Indirect, via planned `HolodeckRoom` preset (not yet implemented).  
**Evidence:**  
- `plato-torch/README.md` lists "holodeck-rust integration" as a future feature.  
- `holodeck-rust/examples/` contains Python client examples but no direct import in plato-torch source.  
**Gap:** No active code connection. Integration is documented intent, not wired.

## 3. plato-torch → plato-ensign
**Connection:** Direct export pipeline.  
**Evidence:**  
- `plato-torch/src/export/ensign_exporter.py` imports `plato_ensign`.  
- `BaseRoom` has `export_to_ensign()` method that calls `plato_ensign.create_adapter()`.  
- Shared data: Room’s `tile_history` and `model_checkpoints` are passed to ensign.  
**Gap:** Export is optional; not all rooms are configured to auto-export.

## 4. fleet-simulator → holodeck-rust
**Connection:** None observed.  
**Evidence:**  
- `fleet-simulator/src/` has no Rust bindings or HTTP client for holodeck.  
- `holodeck-rust/` does not mention fleet-simulator in its docs or examples.  
**Gap:** Separate domains: simulation vs. interactive narrative. No shared state or messaging.

## 5. fleet-simulator → plato-ensign
**Connection:** None observed.  
**Evidence:**  
- `fleet-simulator` outputs simulation logs (JSON) but no direct hook to `plato_ensign`.  
- `plato_ensign` expects room-trained models, not raw simulation data.  
**Gap:** Simulation data could be training input for a room, but no preprocessor bridges them.

## 6. holodeck-rust → plato-ensign
**Connection:** None observed.  
**Evidence:**  
- `holodeck-rust` is a standalone MUD server with sentiment NPCs.  
- `plato_ensign` focuses on PyTorch room exports.  
**Gap:** Holodeck interactions are not currently captured as training tiles for ensign adapters.
