## 1. Plato-Torch (Training Rooms)

**Key Files Inspected:**
- `plato-torch/rooms/deadband_room.py`
- `plato-torch/rooms/__init__.py`
- `plato-torch/training/tile_processor.py`
- `plato-torch/config/room_configs.yaml`

**Connections Found:**

1. **To Plato-Ensign (Explicit)**
   - `deadband_room.py` imports `from plato_ensign.export import export_lora_adapter`
   - Room base class has `export_ensign()` method that calls the above.
   - Config YAML includes `ensign_export_path` and `lora_rank` settings.

2. **To Fleet-Simulator (Indirect via Data)**
   - `tile_processor.py` expects tiles with `metadata.simulation_id` field.
   - No direct import of fleet-simulator, but data schema alignment suggests tiles can originate from simulations.
   - Room configs have `simulation_source: optional` flag.

3. **To Holodeck-Rust (None Found)**
   - No imports, no references in code or config to `holodeck`, `mud`, or `npc`.
   - Plato-torch rooms are pure Python/PyTorch training loops.

**Integration Gaps:**
- Plato-torch does not call holodeck-rust for sentiment-aware NPC interactions.
- Fleet-simulator connection is implicit via data schema only; no API or SDK bridge.

## 2. Plato-Ensign (LoRA Exporter)

**Key Files Inspected:**
- `plato-ensign/export/lora_exporter.py`
- `plato-ensign/models/room_adapter.py`
- `plato-ensign/README.md`

**Connections Found:**

1. **From Plato-Torch (Explicit)**
   - Exporter is called by `deadband_room.export_ensign()`.
   - Accepts `room_state_dict` and `config` from plato-torch.

2. **To External ML Ecosystem**
   - Uses `peft` (Parameter-Efficient Fine-Tuning) library.
   - Outputs `.safetensors` files compatible with Hugging Face `transformers`.
   - No direct fleet connections beyond plato-torch.

**Integration Gaps:**
- Does not import or reference fleet-simulator or holodeck-rust.
- Export is one-way: room → LoRA, no feedback loop.

## 3. Fleet-Simulator (Training Data Generator)

**Key Files Inspected:**
- `fleet-simulator/src/scenarios/deadband_scenario.rs`
- `fleet-simulator/src/output/tile_writer.rs`
- `fleet-simulator/Cargo.toml`

**Connections Found:**

1. **To Plato-Torch (Data Schema)**
   - `tile_writer.rs` writes JSON with `simulation_id`, `agent_id`, `action_sequence`.
   - Schema matches expected tile format in `plato-torch/training/tile_processor.py`.
   - No Rust→Python bridge; output is file-based.

2. **To Constraint-Theory-Core (Direct Dependency)**
   - `Cargo.toml`: `constraint-theory-core = { path = "../constraint-theory-core" }`
   - Used in `deadband_scenario.rs` for geometric snapping of agent paths.

3. **To Holodeck-Rust (None Found)**
   - No references in code or dependencies.

**Integration Gaps:**
- Simulator does not call holodeck for NPC interactions.
- Output is static files; no live API to plato-torch.
