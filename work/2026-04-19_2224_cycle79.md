## 2. Integration Gaps (What's Not Connected)

### **Gap 1: holodeck-rust → plato-ensign** (No Direct Export)
- **Issue:** NPC interactions in `holodeck-rust` are not explicitly exported as LoRA adapters.
- **Impact:** Lost training potential from live multi-user interactions.
- **Action:** Wire `holodeck-rust` sentiment logs to `plato-ensign` export pipeline.

### **Gap 2: fleet-simulator → plato-ensign** (Missing Direct Sim-to-Ensign)
- **Issue:** Simulation data trains rooms, but not directly exported as specialized ensigns.
- **Impact:** Sim-specific strategies (e.g., deadband navigation) not captured as standalone adapters.
- **Action:** Create `SimulationEnsign` export from `fleet-simulator` runs.

### **Gap 3: constraint-theory-core → holodeck-rust** (Geometric Snapping in MUD)
- **Issue:** `constraint-theory-core` (Rust geometric snapping) not integrated into `holodeck-rust` spatial mechanics.
- **Impact:** MUD lacks advanced spatial reasoning for environment interactions.
- **Action:** Integrate snapping library for object placement/NPC pathing.

### **Gap 4: flux-runtime → plato-torch** (Bytecode VM for Rooms)
- **Issue:** `flux-runtime` (self-assembling bytecode VM) not used as a room execution engine.
- **Impact:** Rooms lack portable, efficient bytecode runtime for complex logic.
- **Action:** Prototype a `FluxRoom` preset in `plato-torch`.

### **Gap 5: GhostInjector → holodeck-rust** (Backlog Item)
- **Issue:** Ghost tiles (dead agents' lessons) not injected into `holodeck-rust` as NPC knowledge.
- **Impact:** MUD NPCs don't learn from historical fleet failures (P0 knowledge).
- **Action:** Implement `GhostInjector` module to load ghost tiles into NPC memory.
