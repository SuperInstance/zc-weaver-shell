# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 316  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.  

## 1. plato-torch → fleet-simulator  
**Connection:** Strong, direct integration via `SimulationRoom` preset.  
- **plato-torch/src/rooms/simulation_room.py** imports `fleet_simulator` as a module.  
- Uses `fleet_simulator.Simulation` class to generate training episodes.  
- Simulation results are formatted into training tiles (`Tile` objects) and sent to the PLATO room server.  
- **Data flow:** `fleet_simulator` → `SimulationRoom` → PLATO server.  
- **Gap:** No reverse dependency — `fleet-simulator` does not import `plato-torch`.  

## 2. plato-torch → holodeck-rust  
**Connection:** Indirect, via shared tile format.  
- **holodeck-rust/src/protocol/tile.rs** defines `Tile` struct (Rust).  
- **plato-torch/src/protocol/tile.py** defines `Tile` class (Python).  
- Both use same field names (`id`, `agent_id`, `content`, `metadata`).  
- **Gap:** No direct import or API call between them. Serialization/deserialization must be manually aligned (JSON). Risk of schema drift.  

## 3. plato-torch → plato-ensign  
**Connection:** Direct export pipeline.  
- **plato-ensign/src/export.py** imports `plato_torch` and accesses `Room` instances.  
- Exports room experience (accumulated tiles) into LoRA adapters via `peft`.  
- Triggered by CLI command: `ensign export --room-id <id>`.  
- **Gap:** One-way flow. `plato-torch` does not depend on `plato-ensign`.  

## 4. fleet-simulator → holodeck-rust  
**Connection:** None observed.  
- **fleet-simulator** is pure Python, simulates fleet coordination scenarios.  
- **holodeck-rust** is a Rust MUD server with sentiment-aware NPCs.  
- No shared types, no imports, no configuration references.  
- **Potential integration point:** Simulation episodes could generate NPC dialogue trees for holodeck.  

## 5. holodeck-rust → plato-ensign  
**Connection:** None observed.  
- **holodeck-rust** produces `Tile` objects via player/NPC interactions.  
- **plato-ensign** currently only exports from `plato-torch` rooms.  
- **Gap:** Holodeck tiles are not automatically exported to ensigns.  

## 6. fleet-simulator → plato-ensign  
**Connection:** Indirect via plato-torch.  
- `fleet-simulator` data flows into `plato-torch` rooms.  
- Those rooms can be exported by `plato-ensign`.  
- No direct link.  

## Integration Map (Visual Summary)  
```
plato-torch ─┬─→ fleet-simulator (direct import)  
             ├─→ plato-ensign (export pipeline)  
             └─→ holodeck-rust (shared tile format, no API)  

fleet-simulator ──→ plato-torch only (one-way)  

holodeck-rust ──→ plato-torch (tile format alignment)  
               ──→ plato-ensign (no link)  

plato-ensign ──→ plato-torch only (one-way)  
```
