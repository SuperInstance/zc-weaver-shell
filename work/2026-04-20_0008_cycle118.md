# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 118

Based on reading the actual files in the fleet repos, here is a factual map of current connections and identified integration gaps between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

---

## 1. Current Verified Connections

### **plato-torch → plato-ensign** (Strong, Direct)
- **Connection:** `plato-torch` exports trained room states (`.pt` checkpoints) that `plato-ensign` can convert into LoRA adapters.
- **Evidence:**  
  - `plato-ensign/README.md`: "Exports room experience from plato-torch into LoRA adapters."  
  - `plato-torch/export_ensign.py` exists — exports room checkpoints for ensign processing.
- **Data Flow:** Room training tiles → plato-torch checkpoint → plato-ensign conversion → LoRA adapter.

### **plato-torch → fleet-simulator** (Indirect, via Data)
- **Connection:** `fleet-simulator` generates synthetic training data (agent interactions) that can be consumed by `plato-torch` rooms.
- **Evidence:**  
  - `fleet-simulator/README.md`: "Generates training data for fleet agents."  
  - `plato-torch/data_loader.py` includes a `SimulationDataset` class that loads `.jsonl` logs from `fleet-simulator/output/`.
- **Data Flow:** fleet-simulator logs → plato-torch data loader → training tiles.

### **holodeck-rust → plato-torch** (Weak, Potential)
- **Connection:** `holodeck-rust` produces NPC interaction logs with sentiment metadata. These could be training data for `plato-torch` sentiment-aware rooms.
- **Evidence:**  
  - `holodeck-rust/src/npc/sentiment.rs` tracks NPC emotional state.  
  - `plato-torch/presets/SentimentRoom.py` exists — designed for sentiment-aware training.  
  - No current import of holodeck logs in plato-torch data loaders.
- **Status:** Possible integration, not yet wired.

## 2. Integration Gaps (What’s Not Connected)

### **plato-ensign → holodeck-rust** (Missing)
- **Gap:** LoRA adapters from `plato-ensign` are not loaded into `holodeck-rust` NPCs.
- **Impact:** NPCs cannot be personalized or improved via fleet training.
- **Potential Hook:** `holodeck-rust` could load `.safetensors` adapters to modify NPC behavior.

### **fleet-simulator → holodeck-rust** (Missing)
- **Gap:** No simulation scenarios feed into `holodeck-rust` MUD world.
- **Impact:** Holodeck scenarios are static, not dynamically generated from fleet simulations.
- **Potential Hook:** `fleet-simulator` could output scenario `.json` that `holodeck-rust` consumes to spawn NPCs/quests.

### **holodeck-rust → plato-ensign** (Missing)
- **Gap:** NPC interaction logs from `holodeck-rust` are not exported to `plato-ensign` for adapter training.
- **Impact:** Valuable in-game interaction data is not captured for model specialization.
- **Potential Hook:** Add an export pipeline from holodeck logs → ensign conversion.

### **plato-torch → holodeck-rust** (Bidirectional Gap)
- **Gap:** No real-time feedback loop. Trained rooms don’t affect holodeck, holodeck logs don’t continuously train rooms.
- **Impact:** Training is offline, not live-adaptive.
- **Potential Hook:** `plato-torch` could expose a live inference endpoint for `holodeck-rust` NPCs to query.
