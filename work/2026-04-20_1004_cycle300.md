## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**  
- `rooms/deadband_room.py` — Deadband Protocol training room.  
- `rooms/__init__.py` — Exports room classes.  
- `trainer/self_trainer.py` — Self-training loop.  
- `config/room_configs.yaml` — Room hyperparameters.

**Connections:**  
- **To plato-ensign:** Exports room experience via `export_to_lora()` method in `deadband_room.py`.  
- **To fleet-simulator:** None found in code. No imports or config references.  
- **To holodeck-rust:** None found. No Rust interop or API calls.  
- **Internal:** Uses PyTorch, transformers, peft (LoRA). Rooms inherit from `BaseRoom`.

**Integration Points:**  
- `deadband_room.py` line 142: `def export_to_lora(self, output_path: str) -> None:` calls `plato_ensign.export.export_lora()`.  
- Room configs can be loaded from YAML; no external service discovery.

## 2. plato-ensign (LoRA Export)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `export/lora_exporter.py` — Core export logic.  
- `models/room_adapter.py` — Wraps trained models.  
- `config/export_config.yaml` — Export settings.

**Connections:**  
- **From plato-torch:** Imported in `deadband_room.py`.  
- **To fleet-simulator:** None.  
- **To holodeck-rust:** None.  
- **Internal:** Uses peft, safetensors, huggingface_hub.

**Integration Points:**  
- `lora_exporter.py` expects a trained PyTorch model and a config dict.  
- No API server; called as a library.

## 3. fleet-simulator (Fleet Simulation)
**Location:** `fleet-simulator/`  
**Purpose:** Generate training data via multi-agent simulation.  
**Key Files:**  
- `simulation/agent_sim.py` — Core simulation loop.  
- `environments/deadband_env.py` — Deadband Protocol environment.  
- `config/sim_config.toml` — Simulation parameters.

**Connections:**  
- **To plato-torch:** None. No imports of plato-torch.  
- **To plato-ensign:** None.  
- **To holodeck-rust:** None.  
- **Internal:** Uses asyncio, numpy, pandas. Agents are Python classes.

**Integration Points:**  
- Outputs JSONL logs (`logs/simulation_*.jsonl`).  
- Could feed into plato-torch as training data, but no current pipeline.

## 4. holodeck-rust (Sentiment-Aware MUD)
**Location:** `holodeck-rust/`  
**Purpose:** Multi-user dungeon with sentiment-aware NPCs.  
**Key Files:**  
- `src/server.rs` — Async TCP server.  
- `src/npc/sentiment.rs` — NPC sentiment engine.  
- `config/world.toml` — World layout.

**Connections:**  
- **To plato-torch:** None. Rust project, no Python interop.  
- **To plato-ensign:** None.  
- **To fleet-simulator:** None.  
- **Internal:** Uses tokio, serde, websockets.

**Integration Points:**  
- Exposes WebSocket API on port 8080.  
- NPCs can be driven by external agents, but no built-in adapters.

## 6. Integration Gaps (P0 / P1)
**P0 — What NOT to do:**  
- Do not assume plato-torch reads fleet-simulator logs automatically.  
- Do not assume holodeck-rust connects to Python training rooms.  
- Do not assume plato-ensign is called automatically after training.

**P1 — Safe Channels:**  
- **plato-torch → plato-ensign:** Working export path. Can extend to auto-export.  
- **fleet-simulator → plato-torch:** JSONL logs can be parsed into training tiles.  
- **holodeck-rust → plato-torch:** WebSocket client could feed NPC interactions as tiles.  
- **GhostInjector:** Not yet located in code. Likely a planned integration.

**P2 — Optimization Opportunities:**  
- Auto-export LoRA after room training.  
- Pipe simulation logs into DeadbandRoom as synthetic tiles.  
- Connect holodeck NPCs to DeadbandRoom for live sentiment training.
