## 1. Verified Connections (What's Connected)

### plato-torch → fleet-simulator
- **Connection:** Direct import in `plato-torch/rooms/deadband_room.py`
- **Evidence:** `from fleet_simulator import FleetSimulator` (line 12)
- **Purpose:** DeadbandRoom uses FleetSimulator to generate training episodes (P0/P1/P2 trajectories)
- **Data Flow:** Room calls `simulator.generate_episode()` → receives episode dict → converts to training tiles
- **Status:** **ACTIVE** — functional integration

### plato-torch → plato-ensign
- **Connection:** Export pipeline in `plato-torch/export/`
- **Evidence:** `ensign_exporter.py` imports `plato_ensign` and calls `export_lora()`
- **Purpose:** After room training completes, export experience as LoRA adapter
- **Data Flow:** Room state → export config → plato-ensign LoRA training
- **Status:** **ACTIVE** — optional export path

### holodeck-rust → (indirect) fleet-simulator
- **Connection:** Shared tile format
- **Evidence:** `holodeck-rust/src/tile.rs` defines `Tile` struct with same fields as fleet-simulator's `Tile` in Python
- **Purpose:** Holodeck can consume tiles generated by fleet-simulator for NPC training
- **Data Flow:** JSON serialization/deserialization between systems
- **Status:** **PASSIVE** — format compatibility, not runtime integration

## 2. Integration Gaps (What's Not Connected)

### holodeck-rust ↔ plato-torch
- **Gap:** No direct communication channel
- **Evidence:** No imports between codebases, no shared configuration
- **Impact:** Holodeck NPCs cannot directly interact with PLATO rooms
- **Potential Integration Point:** RabbitMQ message bus or shared Redis store for tile exchange

### fleet-simulator → holodeck-rust
- **Gap:** One-way format compatibility only
- **Evidence:** fleet-simulator generates tiles but has no API to push them to holodeck
- **Impact:** Simulated fleet data cannot automatically populate holodeck scenarios
- **Potential Integration Point:** File watcher in holodeck that loads new tile files from shared directory

### plato-ensign → holodeck-rust
- **Gap:** No connection
- **Evidence:** plato-ensign exports LoRA adapters for inference, holodeck has no LoRA loading mechanism
- **Impact:** Trained ensigns cannot be deployed to holodeck NPCs
- **Potential Integration Point:** ONNX export from plato-ensign + Rust ONNX runtime in holodeck

### DeadbandRoom ↔ plato-relay
- **Gap:** Mentioned in backlog but not implemented
- **Evidence:** No `plato_relay` import in deadband_room.py
- **Impact:** Room cannot broadcast training progress to other fleet components
- **Potential Integration Point:** WebSocket client in DeadbandRoom that connects to plato-relay server
