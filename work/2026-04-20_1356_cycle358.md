## 1. plato-torch
**Repo:** `plato-torch`  
**Purpose:** Training room presets, self-training rooms.  
**Key files:**
- `presets/DeadbandRoom.py` — Implements Deadband Protocol training.
- `presets/__init__.py` — Exports `DeadbandRoom`, `MirrorPlayRoom`, `GhostRoom`.
- `trainer/room_trainer.py` — Main training loop, loads room presets.
- `utils/tile_loader.py` — Loads interaction tiles from disk.

**Connections found:**
- **To plato-ensign:** `presets/DeadbandRoom.py` line 42 imports `from plato_ensign.lora_exporter import export_lora`. Direct integration — rooms can export trained LoRA adapters.
- **To fleet-simulator:** No direct imports found. However, `presets/DeadbandRoom.py` uses simulated agent interactions internally (lines 89–112) but does not import the fleet-simulator package.
- **To holodeck-rust:** No Python imports. Possible indirect link via shared tile format (JSON).

**Integration gap:** plato-torch does not import fleet-simulator or holodeck-rust directly. It operates as a standalone training system.

## 2. fleet-simulator
**Repo:** `fleet-simulator`  
**Purpose:** Generate training data via multi-agent simulation.  
**Key files:**
- `src/simulator/core.rs` — Core simulation engine.
- `src/agents/mod.rs` — Agent definitions (Oracle1, JetsonClaw1, Forgemaster).
- `src/output/tiles.rs` — Exports interaction tiles as JSON.

**Connections found:**
- **To plato-torch:** No Rust → Python imports. Outputs tile JSON files to `data/tiles/`. These files are read by plato-torch’s `tile_loader.py`.
- **To holodeck-rust:** No direct imports. Both are Rust projects but separate crates.
- **To plato-ensign:** No connection.

**Integration gap:** fleet-simulator is a data generator only. Its output must be manually moved or symlinked into plato-torch’s data directory for training.

## 3. holodeck-rust
**Repo:** `holodeck-rust`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key files:**
- `src/server/mod.rs` — MUD server core.
- `src/npc/sentiment.rs` — NPC sentiment system.
- `src/world/tiles.rs` — World tile storage (different format from training tiles).

**Connections found:**
- **To plato-torch:** No imports. May share tile JSON structure but not verified.
- **To fleet-simulator:** No imports.
- **To plato-ensign:** No connection.

**Integration gap:** holodeck-rust appears isolated. Its “tiles” are world map tiles, not training tiles. No evident pipeline to plato-torch.

## 4. plato-ensign
**Repo:** `plato-ensign`  
**Purpose:** Export room experience into LoRA adapters.  
**Key files:**
- `lora_exporter.py` — Main export function.
- `config/__init__.py` — LoRA configuration.

**Connections found:**
- **To plato-torch:** Exported function `export_lora` is imported by `DeadbandRoom.py`. Called when room training completes.
- **To fleet-simulator:** No connection.
- **To holodeck-rust:** No connection.

**Integration gap:** plato-ensign only connects to plato-torch. It does not ingest data from fleet-simulator or holodeck-rust.
