# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 238  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch
**Location:** `fleet/plato-torch/`  
**Purpose:** Training room presets, self-training rooms, tile processing.

### Verified Connections:
- **To plato-ensign:** Exports room experience via `export_room()` function in `plato-torch/src/rooms/deadband_room.py`. Creates LoRA adapters from accumulated tiles.
- **To fleet-simulator:** Uses `fleet_simulator` as a data source in `plato-torch/src/data/simulator_source.py`. Imports `SimulatorDataSource`.
- **To holodeck-rust:** No direct import found. Plato-torch appears agnostic to MUD backend.

### Integration Gaps:
- No direct hook for GhostInjector (ghost tile injection).
- DeadbandRoom preset not wired to plato-relay (messaging layer).
- Room configurations are file-based; no dynamic service discovery.

## 2. fleet-simulator
**Location:** `fleet/fleet-simulator/`  
**Purpose:** Generate training data via multi-agent simulation.

### Verified Connections:
- **To plato-torch:** Provides `SimulatorDataSource` class. Used by plato-torch to pull simulated interaction tiles.
- **To holodeck-rust:** No import found. Simulator appears standalone.
- **To plato-ensign:** No direct export path. Data flows via plato-torch.

### Integration Gaps:
- Simulator output format may not match holodeck-rust sentiment-aware NPC input.
- No live coupling to Deadband Protocol (P0/P1/P2) decision loops.

## 3. holodeck-rust
**Location:** `fleet/holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs, narrative environment.

### Verified Connections:
- **To plato-torch:** None found. Holodeck is a standalone Rust service.
- **To fleet-simulator:** None found.
- **To plato-ensign:** None found. However, `holodeck-rust/src/npc/sentiment.rs` defines `SentimentState` that could be exported as tiles.

### Integration Gaps:
- No GhostInjector endpoint.
- No import of DeadbandRoom or plato-torch data structures.
- NPC sentiment engine isolated; not feeding training loops.

## 4. plato-ensign
**Location:** `fleet/plato-ensign/`  
**Purpose:** Export room experience into LoRA adapters.

### Verified Connections:
- **To plato-torch:** Called via `export_room()` from DeadbandRoom. Uses `plato_ensign.export.export_lora()`.
- **To fleet-simulator:** None direct.
- **To holodeck-rust:** None found.

### Integration Gaps:
- Only supports plato-torch rooms. No hook for holodeck-rust NPC state.
- Export is manual/triggered; no continuous sync.

## 5. GhostInjector
**Location:** `fleet/plato-torch/src/utils/ghost_injector.py`  
**Purpose:** Inject ghost tiles (dead agents' lessons) into training rooms.

### Verified Connections:
- **To plato-torch:** Injects into `Room` base class via `inject_ghost_tiles()`.
- **To holodeck-rust:** None. Ghost tiles are a plato-torch concept.

### Integration Gap:
- GhostInjector not connected to holodeck-rust NPC memory.

## 6. plato-relay
**Location:** `fleet/plato-relay/`  
**Purpose:** Messaging layer for room synchronization.

### Verified Connections:
- **To plato-torch:** `DeadbandRoom` has a `relay_client` attribute but not wired in current preset.
- **To others:** Not yet integrated.

### Integration Gap:
- DeadbandRoom preset does not activate relay client.
