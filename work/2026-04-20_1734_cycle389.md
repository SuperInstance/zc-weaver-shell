## 1. plato-torch (Training Rooms)

**Key Files:**
- `plato-torch/src/rooms/deadband_room.py` — Primary training room implementation
- `plato-torch/src/rooms/__init__.py` — Room registry
- `plato-torch/src/tile_processor.py` — Processes training tiles

**Connections Found:**
- **To plato-ensign:** Direct import in `deadband_room.py`:

  Used in `export_ensign()` method to create LoRA adapters from room experience.
- **To fleet-simulator:** No direct imports found. However, `deadband_room.py` references "simulation data" in docstrings but no actual integration.
- **To holodeck-rust:** No imports or references found.

- Configuration uses `room_config.yaml` — no shared config with other systems.

## 2. plato-ensign (LoRA Export)

**Key Files:**
- `plato-ensign/src/lib.rs` — Core export logic
- `plato-ensign/src/adapters/` — LoRA serialization

**Connections Found:**
- **To plato-torch:** Used as library (Python bindings via PyO3). Called from `deadband_room.py`.
- **To fleet-simulator:** No references.
- **To holodeck-rust:** No references.

**Integration Points:**

- Outputs `.safetensors` files (LoRA weights).
