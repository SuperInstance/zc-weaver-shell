## 1. plato-torch
**Repository:** `plato-torch`  
**Purpose:** Training room presets, self-training rooms, tile ingestion and processing.  
**Key Files:** `DeadbandRoom.py`, `presets/`, `training/`

### Verified Connections:
- **To plato-ensign:** Direct import in `DeadbandRoom.py`:
  ```python
  from plato_ensign import export_room_experience, load_lora_adapter
  ```
  Used to export room experience as LoRA adapters after training cycles.
- **To fleet-simulator:** No direct import found. However, `presets/` contains `FleetSimRoom.py` which appears designed to ingest simulation data, but the simulator itself is not imported.
- **To holodeck-rust:** No direct import. No Rust interop visible in Python files.

### Integration Gaps:
- **fleet-simulator:** Preset exists but no live data pipe. Could be fed via file or API.
- **holodeck-rust:** No connection. Sentiment-aware NPC data not ingested into rooms.

## 2. fleet-simulator
**Repository:** `fleet-simulator`  
**Purpose:** Generate training data via multi-agent fleet simulations.  
**Key Files:** `simulator.py`, `agents/`, `scenarios/`

### Verified Connections:
- **To plato-torch:** None. Output appears to be JSON logs (`logs/`). No import of plato-torch modules.
- **To plato-ensign:** None.
- **To holodeck-rust:** None.

### Integration Gaps:
- **plato-torch:** Simulation logs are not automatically ingested into a training room. Manual or scripted transfer needed.
- **holodeck-rust:** No shared agent definitions or scenario integration.

## 3. holodeck-rust
**Repository:** `holodeck-rust`  
**Purpose:** MUD server with sentiment-aware NPCs, real-time interaction environment.  
**Key Files:** `src/`, `Cargo.toml`

### Verified Connections:
- **To plato-torch:** None. Rust project, no Python interop in current `src/`.
- **To fleet-simulator:** None.
- **To plato-ensign:** None.

### Integration Gaps:
- **plato-torch:** NPC interactions not captured as training tiles. No WebSocket or REST API exposed for tile ingestion.
- **fleet-simulator:** No shared agent state or scenario sync.

## 4. plato-ensign
**Repository:** `plato-ensign`  
**Purpose:** Export room experience from plato-torch into LoRA adapters.  
**Key Files:** `ensign.py`, `lora/`

### Verified Connections:
- **To plato-torch:** Directly imported by `DeadbandRoom.py` (see above). Called via `export_room_experience(room_state)`.
- **To fleet-simulator:** None.
- **To holodeck-rust:** None.

### Integration Gaps:
- **fleet-simulator:** Simulation data cannot be directly exported as LoRA; must pass through a plato-torch room first.
- **holodeck-rust:** NPC sentiment data not available for adapter training.
