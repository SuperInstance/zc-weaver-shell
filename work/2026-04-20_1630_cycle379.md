## 1. plato-torch → fleet-simulator → holodeck-rust → plato-ensign

### **plato-torch → fleet-simulator**
**Connection:** ✅ **Direct import**  
**File:** `plato-torch/src/training/rooms/deadband_room.py`  
**Evidence:**  
```python
from fleet_simulator.agent import FleetAgent
from fleet_simulator.scenario import DeadbandScenario
```
**Usage:**  
- `DeadbandRoom` instantiates `FleetAgent` and `DeadbandScenario`  
- Simulated agents generate interaction tiles  
- Tiles are stored in `room.tile_buffer` for later training  

**Data Flow:**  
`fleet_simulator` → generates `AgentAction` and `AgentObservation` → converted to `InteractionTile` → stored in `plato-torch` buffer

### **plato-torch → plato-ensign**
**Connection:** ✅ **Direct export path**  
**File:** `plato-torch/src/export/ensign_exporter.py`  
**Evidence:**  
```python
from plato_ensign.lora_adapter import LoRAAdapter
from plato_ensign.tile_processor import TileProcessor
```
**Usage:**  
- `EnsignExporter` takes trained room state  
- Converts to LoRA weights via `plato_ensign`  
- Saves adapter to `./ensigns/{room_name}.safetensors`  

**Data Flow:**  
`plato-torch` trained model → `EnsignExporter` → `plato_ensign.LoRAAdapter` → LoRA file

## 2. Integration Gaps (What's Not Connected)

### **Gap 1: fleet-simulator ↔ holodeck-rust**
**Current State:** Separate simulation environments  
**Impact:** Fleet agents cannot interact with holodeck NPCs  
**Potential Bridge:**  
- Add `holodeck_client` module to `fleet-simulator`  
- Send `AgentObservation` to holodeck WebSocket endpoint  
- Receive NPC responses as part of observation space  

---

### **Gap 2: holodeck-rust → plato-torch**
**Current State:** No tile generation from NPC interactions  
**Impact:** NPC sentiment data not used for training  
**Potential Bridge:**  
- Add `TileGenerator` to holodeck-rust  
- Export NPC dialogues as `InteractionTile` JSON  
- Ingest into `plato-torch` via file watcher  

---

### **Gap 3: plato-ensign ← holodeck-rust**
**Current State:** No LoRA export from NPC behaviors  
**Impact:** Cannot create "NPC personality" ensigns  
**Potential Bridge:**  
- Add Python bindings to holodeck-rust (`pyo3`)  
- Create `HolodeckExporter` in `plato-ensign`  
- Train LoRA on NPC dialogue patterns
