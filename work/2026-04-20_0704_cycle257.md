## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** Provides 25 training room presets and self-training rooms.  
**Key Files:**  
- `plato-torch/src/rooms/deadband_room.py` — Deadband Protocol training room.  
- `plato-torch/src/rooms/__init__.py` — Room registry.  
- `plato-torch/src/tile_server.py` — Receives and validates training tiles.  

**Connections Observed:**  
- **To plato-ensign:** `deadband_room.py` imports `from plato_ensign import export_room`. Direct export path for LoRA adapters.  
- **To fleet-simulator:** No direct import found. However, `deadband_room.py` references `fleet_simulator` in a docstring as a potential data source.  
- **To holodeck-rust:** No import or reference found in code.  

**Integration Points:**  
- `tile_server.py` accepts POST requests at `/submit_tile`. This is the ingestion endpoint for agent work (tiles).  
- Rooms are registered via decorator `@register_room`.

## 2. plato-ensign (LoRA Export)  
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `plato-ensign/src/exporter.py` — Main export logic.  
- `plato-ensign/src/lora_builder.py` — Builds LoRA from tile batches.  

**Connections Observed:**  
- **From plato-torch:** Called by `deadband_room.export()` method.  
- **To fleet-simulator/holodeck-rust:** No imports found.  

**Integration Points:**  
- `exporter.py` expects a `room_id` and `tile_batch` (list of dicts).  
- Output is a `.safetensors` file saved to `exports/`.

## Integration Summary

### Connected:
1. **plato-torch → plato-ensign** — Direct Python import. Room can export to LoRA.  
2. **plato-torch tile server** — External endpoint for agent tiles (REST).  

### Not Connected (Gaps):
1. **fleet-simulator → plato-torch** — No automated data flow. JSON output must be manually transformed into tiles.  
2. **holodeck-rust → any training system** — NPC interactions are not captured as tiles.  
3. **cudaclaw / flux-runtime / constraint-theory-core** — Standalone; not wired into the training or simulation loop.  
4. **Ghost tiles / Mirror Play** — Referenced in docs but no implementation found in code.
