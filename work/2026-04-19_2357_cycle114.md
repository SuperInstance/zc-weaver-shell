# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 114  
**Phase:** 4 — Build & Test  
**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

## 1. Verified Connections (What's Connected)

### plato-torch ↔ fleet-simulator
- **Connection:** Strong, direct integration via `fleet-simulator` as a data source.
- **Evidence:** `plato-torch/rooms/deadband_room.py` imports `FleetSimulator` class from `fleet_simulator.simulator`.
- **Purpose:** The `DeadbandRoom` preset uses the fleet simulator to generate training episodes (agent trajectories) which become training tiles.
- **Data Flow:** Simulator runs → produces episode dict → room converts to training tiles → tiles stored in room's tile buffer.

### plato-torch ↔ plato-ensign
- **Connection:** Direct export pipeline.
- **Evidence:** `plato-torch/export/ensign_exporter.py` exists and is referenced in room configurations.
- **Purpose:** After a room accumulates enough training tiles, the ensign exporter can package them into a LoRA adapter (`.safetensors`).
- **Data Flow:** Room tile buffer → ensign exporter → LoRA weights file.

### plato-torch internal
- **Connection:** Rooms presets (like `DeadbandRoom`) inherit from `BaseRoom` and use `TileBuffer`, `TrainingOrchestrator`.
- **Evidence:** Clear class hierarchy and imports within `plato-torch/`.
- **Purpose:** Modular training system where rooms are configurable training environments.

## 2. Integration Gaps (What's Not Connected)

### holodeck-rust ↔ plato-torch
- **Gap:** No direct connection found in code.
- **Evidence:** No imports of `holodeck` in `plato-torch` files, and vice versa.
- **Potential:** `holodeck-rust` is a sentiment-aware MUD server that could generate rich narrative training data. Currently isolated.
- **Integration Point:** Could feed NPC dialogue/event logs into `DeadbandRoom` as another tile source.

### holodeck-rust ↔ fleet-simulator
- **Gap:** No connection.
- **Evidence:** Separate repos, no shared code or data references.
- **Potential:** Fleet agents simulated in `fleet-simulator` could have personas/interactions driven by `holodeck` NPC engine.

### holodeck-rust ↔ plato-ensign
- **Gap:** No connection.
- **Evidence:** `plato-ensign` only referenced from `plato-torch` export.
- **Potential:** `holodeck` interactions could be exported as specialized narrative ensigns.

### fleet-simulator ↔ plato-ensign
- **Gap:** Indirect only via `plato-torch`.
- **Evidence:** `fleet-simulator` does not call `plato-ensign` directly.
- **Note:** This is likely intentional—simulator is a data source, not a training room.
