## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self‑training rooms, tile ingestion.

### Verified Connections:
- **To plato‑ensign:**  
  `plato-torch/src/rooms/deadband_room.py` imports `from plato_ensign import EnsignExporter`.  
  The `DeadbandRoom` class has an `export_to_ensign()` method that calls `EnsignExporter.save_lora_adapter()`.

- **To fleet‑simulator:**  
  `plato-torch/config/training_presets.yaml` references `fleet_simulator_scenarios` as a data source for room 14 (“Tactical Sim Room”).  
  No direct Python import; uses YAML‑configured data paths.

- **To holodeck‑rust:**  
  No direct imports or configuration references found in plato‑torch codebase.

### Integration Gaps:
- No live socket or API connection to holodeck‑rust.  
- No shared session‑state protocol with fleet‑simulator (only static scenario files).  
- Tile ingestion is file‑based (`/tmp/tiles/`), not streaming.

## 2. fleet‑simulator (Fleet Simulation)
**Location:** `fleet-simulator/`  
**Purpose:** Generate training data via simulated fleet interactions.

### Verified Connections:
- **To plato‑torch:**  
  `fleet-simulator/scripts/generate_training_scenarios.py` writes output to `../plato-torch/data/scenarios/`.  
  Used by “Tactical Sim Room” preset.

- **To holodeck‑rust:**  
  No imports or shared data paths found.

- **To plato‑ensign:**  
  No direct connection.

### Integration Gaps:
- Simulator runs as a batch job, not a live service.  
- No feedback loop from plato‑torch rooms to adjust simulations.  
- No integration with holodeck‑rust NPC sentiment engine.
