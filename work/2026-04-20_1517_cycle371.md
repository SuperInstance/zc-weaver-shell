- **Location:** `plato-ensign/src/`

- **Output:** LoRA adapters (`.safetensors`).
- **Connection to holodeck-rust:** None found. No Rust bindings or API calls.
- **Gap:** Adapters are not automatically loaded into holodeck. Manual deployment step required.

- **Gap:** NPCs do not use trained LoRAs from plato-ensign. Sentiment is hard-coded or uses a generic model.

### **fleet-simulator** (Fleet Simulation for Training Data)
- **Location:** `fleet-simulator/src/`

- **Output:** Simulation logs (JSON) that could become training tiles.
- **Connection to plato-torch:** None. Simulator does not feed rooms directly.
- **Gap:** Simulation data is not automatically converted to tiles or ingested by rooms.

## 3. **Integration Gaps (P0: What NOT to Do)**

1. **Do NOT assume automatic pipeline from rooms → ensign → holodeck.**  
   - Current state: manual file movement required.
   - Risk: Tiles and adapters become stale.

2. **Do NOT assume holodeck NPCs use trained adapters.**  
   - Current state: sentiment is internal, static.
   - Risk: NPC behavior does not improve with fleet training.

3. **Do NOT assume fleet-simulator feeds training data.**  
   - Current state: simulator runs in isolation.
   - Risk: Simulation insights are not captured as tiles.
