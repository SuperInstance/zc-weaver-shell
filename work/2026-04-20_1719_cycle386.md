## 1. Repository Structure & Dependencies

Based on file inspection of the fleet repositories:

### **plato-torch** (training rooms)
- **Purpose**: Implements 25 training room presets, including DeadbandRoom.
- **Key files**: `rooms/deadband_room.py`, `rooms/__init__.py`, `trainer/plato_trainer.py`
- **Dependencies found**:
  - `torch`, `transformers` (core ML)
  - `numpy`, `pandas` (data handling)
  - `fastapi`, `uvicorn` (API server)
  - **No direct imports** of `fleet-simulator`, `holodeck-rust`, or `plato-ensign` in core room files.
  - Contains `Tile` and `Room` base classes that define the training data structure.

### **fleet-simulator** (training data generation)
- **Purpose**: Generates synthetic fleet interaction data for training.
- **Key files**: `simulator/core.py`, `agents/zeroclaw_agent.py`
- **Dependencies found**:
  - `numpy`, `pandas`
  - `torch` (optional, for some agent models)
  - **No direct imports** of `plato-torch`, `holodeck-rust`, or `plato-ensign`.
  - Outputs `SimulationTile` JSON structures that mirror `Tile` format in plato-torch.

### **holodeck-rust** (sentiment-aware MUD server)
- **Purpose**: Multi-user dungeon server with NPCs that react to player sentiment.
- **Key files**: `src/npc/sentiment.rs`, `src/world/room.rs`
- **Dependencies found**:
  - `serde`, `tokio` (Rust ecosystem)
  - **No direct imports** of other fleet repos (Rust/C++ boundary).
  - Exposes HTTP/WebSocket API for client connections.
  - Defines `Event` and `Dialogue` structs that could map to `Tile` actions.

### **plato-ensign** (LoRA adapter export)
- **Purpose**: Exports trained room experience as LoRA adapters.
- **Key files**: `export/lora_exporter.py`, `models/ensign_model.py`
- **Dependencies found**:
  - `torch`, `peft` (LoRA library)
  - `transformers`
  - **Imports from plato-torch**: `from plato_torch.rooms import Room, Tile` (direct connection).
  - **No imports** of `fleet-simulator` or `holodeck-rust`.

## 3. Integration Gaps (What's Not Connected)

| Gap | Description | Impact |
|-----|-------------|--------|
| **plato-torch → fleet-simulator** | No import or direct data pipe. | Training rooms cannot directly consume simulated fleet data. |
| **plato-torch → holodeck-rust** | No Rust/Python bridge. | Room interactions cannot drive NPC sentiment or MUD events. |
| **fleet-simulator → holodeck-rust** | No connection. | Simulated agents cannot interact with holodeck world. |
| **plato-ensign → fleet-simulator/holodeck** | Only connects to plato-torch. | Ensigns cannot capture simulation or holodeck experience. |
| **GhostInjector (any)** | Not found in current repo imports. | Ghost tiles from dead agents cannot be injected into live systems. |
