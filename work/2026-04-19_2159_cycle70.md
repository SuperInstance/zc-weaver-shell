## 1. Verified Current Connections

Based on reading the fleet-knowledge files and repository structures:

### **plato-torch → plato-ensign** (Strong Connection)
- **Connection:** Direct export pipeline.  
- **Mechanism:** `plato-torch` training rooms accumulate interaction tiles. `plato-ensign` exports room experience into LoRA adapters.  
- **Evidence:** Both listed under "Key Technologies" as complementary systems. Plato-ensign's purpose is to export from plato-torch rooms.  
- **Status:** **Connected** — This is a core, documented workflow.

### **plato-torch → fleet-simulator** (Potential Data Flow)
- **Connection:** Training data supply.  
- **Mechanism:** `fleet-simulator` generates simulation data for training. This data could feed into `plato-torch` rooms as training tiles.  
- **Evidence:** Both are listed as key technologies. Fleet-simulator's stated purpose is "Fleet simulation for training data."  
- **Status:** **Likely connected** — Logical data pipeline exists, but explicit integration code not yet verified in this scan.

### **holodeck-rust → plato-torch** (Sentiment-Aware NPCs as Training Source)
- **Connection:** NPC interactions as tile source.  
- **Mechanism:** `holodeck-rust` is a MUD server with sentiment-aware NPCs. Their interactions could be captured as training tiles for `plato-torch` rooms.  
- **Evidence:** Both are key technologies. Architecture mentions "Rooms train from accumulated interaction tiles" — holodeck could be a tile source.  
- **Status:** **Potential connection** — Conceptually aligned, but no explicit wiring seen yet.

### **plato-ensign → cudaclaw / flux-runtime** (Deployment Targets)
- **Connection:** Ensign LoRA adapters as deployable units.  
- **Mechanism:** Trained ensigns (specialist models) could be loaded into `cudaclaw` (GPU runtime) or `flux-runtime` (VM) for execution.  
- **Evidence:** All are key technologies. Cudaclaw is a "GPU-resident agent runtime" — natural target for trained models.  
- **Status:** **Planned connection** — Architecturally implied, integration specifics TBD.

- **Opportunity:** Ensure DeadbandRoom training is federated across the fleet.

- **Opportunity:** Create a canonical simulation-to-training pipeline, ensuring synthetic data becomes valid tiles.
