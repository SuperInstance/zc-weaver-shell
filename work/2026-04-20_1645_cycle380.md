## 1. plato-torch
**Location:** `fleet-knowledge/plato-torch/`  
**Purpose:** Training room presets, self‑training rooms, tile ingestion.  
**Key files:**
- `presets/DeadbandRoom.py` – Implements Deadband Protocol as a training room.
- `presets/__init__.py` – Exports `DeadbandRoom`, `MirrorRoom`, `GhostRoom`.
- `tile_ingestor.py` – Ingests training tiles (JSON) into room state.
- `room_server.py` – HTTP server that hosts rooms, manages sessions.

**Connections:**
- **plato‑ensign** – `DeadbandRoom.py` imports `from plato_ensign import export_lora` (line 47). Room can export its experience as a LoRA adapter.
- **fleet‑simulator** – No direct import found. However, `tile_ingestor.py` expects tiles that could be generated by a simulator.
- **holodeck‑rust** – No direct import. No shared data structures observed.
- **External** – Uses `torch`, `transformers`, `flask`.

**Integration gaps:**
- No explicit hook to pull training data from `fleet‑simulator`.
- No API endpoint for `holodeck‑rust` to submit NPC interaction tiles.

## 2. fleet‑simulator
**Location:** `fleet-knowledge/fleet-simulator/`  
**Purpose:** Generate synthetic training data by simulating fleet interactions.  
**Key files:**
- `simulator/core.py` – Main simulation loop, agent definitions.
- `output/tile_generator.py` – Converts simulation events to tile format.
- `config/agent_profiles.json` – Profiles for Oracle1, JetsonClaw1, Forgemaster.

**Connections:**
- **plato‑torch** – `tile_generator.py` writes tiles in the same JSON format that `tile_ingestor.py` reads. No direct import, but format‑compatible.
- **plato‑ensign** – No direct import.
- **holodeck‑rust** – No direct import.
- **External** – Uses `numpy`, `pandas`.

**Integration gaps:**
- Tiles are written to disk (`output/tiles/`). No live stream to `plato‑torch` server.
- No configuration to simulate `holodeck‑rust` NPC sentiment events.

## 3. holodeck‑rust
**Location:** `fleet-knowledge/holodeck-rust/`  
**Purpose:** MUD server with sentiment‑aware NPCs, generates interaction tiles.  
**Key files:**
- `src/npc/sentiment.rs` – NPC emotion engine.
- `src/tile_exporter.rs` – Exports NPC‑player interactions as tiles.
- `config/world.toml` – Room and NPC definitions.

**Connections:**
- **plato‑torch** – `tile_exporter.rs` writes JSON tiles. Format matches `plato‑torch` tile schema. No network send.
- **fleet‑simulator** – No direct link.
- **plato‑ensign** – No direct import.
- **External** – Uses `serde`, `tokio`, `websocket`.

**Integration gaps:**
- Tiles are written to local file system only. No integration with `plato‑torch` ingestion API.
- No mechanism to import room definitions from `plato‑torch` presets.

## 4. plato‑ensign
**Location:** `fleet-knowledge/plato-ensign/`  
**Purpose:** Export room experience (accumulated tiles) as LoRA adapters.  
**Key files:**
- `export_lora.py` – Main export function, called by `DeadbandRoom`.
- `utils/merge_adapters.py` – Merges multiple LoRAs.
- `config/presets.yaml` – LoRA rank, alpha settings.

**Connections:**
- **plato‑torch** – Directly imported by `DeadbandRoom`. Called when room reaches experience threshold.
- **fleet‑simulator** – No direct import.
- **holodeck‑rust** – No direct import.
- **External** – Uses `peft`, `transformers`.

**Integration gaps:**
- Only triggered from `plato‑torch` rooms. No standalone mode to export from `fleet‑simulator` or `holodeck‑rust` tiles.
- No pipeline to push exported LoRA to a model registry.

## 6. Immediate Integration Gaps (P0/P1)
**P0 (negative space):**
- Do not assume tiles are automatically streamed between components.
- Do not modify core data structures without verifying schema compatibility.

**P1 (safe channels):**
- Create a shared tile schema definition (JSON Schema) in `fleet‑knowledge/tile_schema.json`.
- Add a `--stream` option to `fleet‑simulator` and `holodeck‑rust` that POSTs tiles to `plato‑torch` ingestion endpoint.
- Add a configuration flag in `DeadbandRoom` to enable/disable `plato‑ensign` export.

**P2 (optimization):**
- Build a tile‑router service that fans out tiles to multiple rooms.
- Add a LoRA registry for versioned ensigns.
