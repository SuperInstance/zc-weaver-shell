## Identified Integration Gaps (P1 Opportunities)

1. **holodeck-rust client SDK for Python:** plato-torch uses raw HTTP. A lightweight client crate (`holodeck-client`) could improve reliability and type safety.
2. **fleet-simulator event bridge to holodeck:** No shared state between simulation and MUD. Could allow simulator events to influence NPC sentiment.
3. **plato-ensign training from holodeck logs:** holodeck-rust NPC interactions are not currently captured as training tiles for ensign export.
4. **Unified configuration:** Each component has its own config (YAML, TOML, ENV). A shared config schema could reduce duplication.
