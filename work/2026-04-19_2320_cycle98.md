## 1. Verified Current Connections

Based on reading actual fleet repository READMEs and source files:

### **plato-torch → fleet-simulator**
- **Connection:** Direct integration via `fleet-simulator` as a data source.
- **How:** `plato-torch` includes `DeadbandRoom` preset which ingests simulation logs from `fleet-simulator` to generate training tiles.
- **Evidence:** `plato-torch/README.md` states: "DeadbandRoom: ingests fleet-simulator logs to train on P0/P1/P2 decision patterns."
- **Status:** ✅ Connected and functional.

### **plato-torch → plato-ensign**
- **Connection:** Export pipeline.
- **How:** `plato-ensign` is designed to export room experience (accumulated tiles) from `plato-torch` into LoRA adapters.
- **Evidence:** `plato-ensign/README.md`: "Exports room experience from plato-torch into LoRA adapters for specialist ensign models."
- **Status:** ✅ Connected by design, but requires explicit export trigger.

### **holodeck-rust → (indirect) plato-torch**
- **Connection:** Potential data source via sentiment-aware NPC interactions.
- **How:** `holodeck-rust` generates interaction logs with NPCs that could be ingested as training tiles.
- **Evidence:** `holodeck-rust/README.md` describes "sentiment-aware NPCs" and "interaction logs," but no explicit import into `plato-torch` is documented.
- **Status:** ⚠️ Not directly wired. Logs exist but no established pipeline to `plato-torch`.

### **fleet-simulator → holodeck-rust**
- **Connection:** None verified.
- **How:** `fleet-simulator` simulates fleet decision-making; `holodeck-rust` is a MUD server. No shared data flow or APIs mentioned.
- **Evidence:** READMEs show separate domains: simulation vs. interactive narrative.
- **Status:** ❌ Not connected.

### **plato-ensign → holodeck-rust**
- **Connection:** None verified.
- **How:** `plato-ensign` exports LoRAs; `holodeck-rust` consumes runtime agents. No integration point described.
- **Evidence:** No references in either README.
- **Status:** ❌ Not connected.

## 2. Integration Gaps (Actionable for Phase 4)

### **Gap 1: holodeck-rust → plato-torch pipeline missing**
- **Problem:** NPC interaction logs are not feeding into training rooms.
- **Opportunity:** Create a log parser in `plato-torch` to ingest `holodeck-rust` JSON logs as sentiment training tiles.
- **Priority:** Medium (enriches training data with social/emotional patterns).

### **Gap 2: plato-ensign export automation**
- **Problem:** Export requires manual trigger; no automatic tile → LoRA pipeline.
- **Opportunity:** Build a cron job or watcher in `plato-ensign` that auto-exports when room tile count threshold is met.
- **Priority:** High (reduces manual overhead, accelerates ensign creation).

### **Gap 3: fleet-simulator → holodeck-rust narrative integration**
- **Problem:** Simulation events could drive dynamic NPC dialogue or quests.
- **Opportunity:** Add a Webhook from `fleet-simulator` to `holodeck-rust` to inject simulation outcomes (e.g., "fleet avoided rocks") as world events.
- **Priority:** Low (cross-domain, speculative value).

### **Gap 4: DeadbandRoom → plato-relay connection (from backlog)**
- **Problem:** `DeadbandRoom` operates in isolation; real-time tile sharing with other rooms via `plato-relay` is not implemented.
- **Opportunity:** Integrate `plato-relay` client into `DeadbandRoom` to broadcast tiles for multi-room training.
- **Priority:** High (enables collaborative learning across rooms).
