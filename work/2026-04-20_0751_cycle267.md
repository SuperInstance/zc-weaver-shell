**Cycle:** 267

## 1. Plato-Torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**
- `presets/DeadbandRoom.py` — P0/P1/P2 training logic.
- `presets/__init__.py` — exports `DeadbandRoom`, `MirrorRoom`, `OracleRoom`.
- `plato_torch/room_server.py` — HTTP server for tile ingestion.
- `plato_torch/tile.py` — `TrainingTile` dataclass (source, content, metadata).

**Connections:**
- **To Plato-Ensign:** `presets/DeadbandRoom.py` imports `from plato_ensign import export_room`. Direct integration for LoRA export.
- **To Fleet-Simulator:** No direct import found. Potential integration via shared `TrainingTile` format.
- **To Holodeck-Rust:** No direct import. Possible bridge through sentiment data in tile metadata.

## 2. Plato-Ensign (LoRA Export)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**
- `plato_ensign/export.py` — `export_room(room_name, output_path)`.
- `plato_ensign/lora_train.py` — Uses `peft` and `transformers`.
- `setup.py` — Declares dependency on `plato-torch`.

**Connections:**
- **From Plato-Torch:** Direct import in `DeadbandRoom.py`. Plato-Torch calls `export_room`.
- **To Fleet-Simulator/Holodeck-Rust:** No imports found. Standalone export module.

## 3. Fleet-Simulator (Training Data)
**Location:** `fleet-simulator/`  
**Purpose:** Fleet simulation for training data.  
**Key Files:**
- `src/simulation/engine.rs` — Discrete-event simulation core.
- `src/agents/mod.rs` — Agent behavior definitions.
- `src/output.rs` — Exports `SimulationTile` struct.

**Connections:**
- **To Plato-Torch:** No Rust→Python imports. Potential integration: `SimulationTile` could be converted to `TrainingTile` via a bridge script (not yet found).
- **To Holodeck-Rust:** No direct imports. Separate codebase.
- **To Plato-Ensign:** No connection.

## 4. Holodeck-Rust (Sentiment-Aware MUD)
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**
- `src/npc/sentiment.rs` — Sentiment modeling.
- `src/world/tile.rs` — `WorldTile` struct (different from `TrainingTile`).
- `src/server.rs` — WebSocket server.

**Connections:**
- **To Plato-Torch:** No direct imports. Possible integration: sentiment data could feed into `TrainingTile.metadata`.
- **To Fleet-Simulator:** No imports.
- **To Plato-Ensign:** No connection.

|-------------|-------------|-----------------|------------------|
