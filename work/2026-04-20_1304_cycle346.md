## Integration Gaps Identified

### 1. **plato-torch ↔ holodeck-rust — No Connection**
- **Gap:** plato-torch training rooms cannot currently interact with holodeck-rust NPCs or environments.
- **Potential Integration Point:** Create a `HolodeckRoom` preset in plato-torch that connects via WebSocket to holodeck-rust server. This would allow agents to train against sentiment-aware NPCs.

### 2. **fleet-simulator ↔ plato-torch — Indirect Data Flow**
- **Gap:** Tiles generated by fleet-simulator are not automatically fed into plato-torch rooms for training.
- **Potential Integration Point:** Add a `SimulationTileLoader` to plato-torch that reads fleet-simulator output JSON and creates training batches.

### 3. **holodeck-rust ↔ fleet-simulator — No Connection**
- **Gap:** Two Rust codebases that don't share components or data.
- **Potential Integration Point:** Could share agent definitions or world state structures through a common Rust crate (`fleet-core-types`).
