# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 158

Based on reading the actual files in the fleet repos, here is the current state of integration between plato-torch, fleet-simulator, holodeck-rust, and plato-ensign.

## 1. Verified Connections (What's Connected)

### plato-torch ↔ fleet-simulator
- **Connection:** Strong, bidirectional data flow.
- **Evidence:**  
  - `plato-torch/training_rooms/deadband_room.py` imports `fleet_simulator` and uses `FleetSimulator` class to generate training data.  
  - `fleet_simulator/__init__.py` exports `FleetSimulator` and `generate_training_scenario`.  
  - The DeadbandRoom preset (`plato-torch/presets/deadband_room.py`) configures the simulator to produce P0/P1/P2 scenario tiles.
- **Data Flow:** Simulator generates synthetic fleet interaction data → Room ingests it as training tiles.

### plato-torch ↔ plato-ensign
- **Connection:** Direct export pipeline.
- **Evidence:**  
  - `plato-ensign/exporter.py` has a `RoomExporter` class that takes a trained room (from plato-torch) and exports a LoRA adapter.  
  - `plato-torch/training_rooms/base.py` defines `save_checkpoint` and `export` methods that plato-ensign calls.  
  - The ensign CLI (`plato-ensign/cli.py`) has `export-room` command that loads a room checkpoint.
- **Data Flow:** Room trains on tiles → Exporter extracts LoRA weights → Saved as `.safetensors` file.

### fleet-simulator → holodeck-rust (Planned)
- **Connection:** Not yet implemented, but designed.
- **Evidence:**  
  - `fleet-simulator/README.md` states: "Simulated fleet interactions can be fed into holodeck-rust for sentiment-aware NPC training."  
  - `holodeck-rust/src/lib.rs` has `Scenario` struct that could accept external event streams.
- **Status:** Integration point identified but not wired.

## 2. Integration Gaps (What's Not Connected)

### holodeck-rust ↔ plato-torch
- **Gap:** No direct connection.
- **Evidence:**  
  - `holodeck-rust` is a standalone MUD server written in Rust.  
  - `plato-torch` is Python/PyTorch. No Rust bindings or API bridge exists.  
  - No imports or references between the two codebases.
- **Potential:** Could connect via TCP socket or REST API. holodeck could send NPC interaction logs to a room as training tiles.

### holodeck-rust ↔ plato-ensign
- **Gap:** No connection.
- **Evidence:**  
  - `plato-ensign` only exports from plato-torch rooms.  
  - holodeck has no export path to LoRA adapters.
- **Potential:** If holodeck interactions were recorded as tiles, they could train a room, then be exported.

### fleet-simulator ↔ plato-ensign (Direct)
- **Gap:** Simulator does not directly produce ensigns.
- **Evidence:**  
  - `fleet-simulator` generates data, not models.  
  - `plato-ensign` only exports from trained rooms.
- **Note:** This is by design—simulator feeds rooms, rooms feed ensigns.
