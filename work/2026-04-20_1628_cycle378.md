## 1. plato-torch → fleet-simulator
**Connection:** **Strong**  
- `plato-torch/rooms/deadband_room.py` imports `fleet_simulator` directly:  
  ```python
  from fleet_simulator import FleetSimulator, FleetConfig
  ```
- Uses `FleetSimulator` to generate training episodes (simulated fleet interactions).  
- Room passes simulator-generated episodes to Plato's training loop.  
- **Data flow:** `FleetSimulator` → `DeadbandRoom` → Plato training tile generation.

**Integration Gap:** None observed. This is a core, working integration.

## 3. plato-torch → plato-ensign
**Connection:** **Strong**  
- `plato-torch/export/ensign_exporter.py` imports `plato_ensign` and calls `export_to_lora()`.  
- Exporter runs after room training, converting accumulated tiles into LoRA adapters.  
- Configurable via `plato-torch/configs/export.yaml`.  
- **Data flow:** `DeadbandRoom` → accumulated tiles → `ensign_exporter.py` → LoRA file.

**Integration Gap:** None observed. This is a working export pipeline.
