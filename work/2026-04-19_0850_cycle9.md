## 1. Current Connections (What IS Connected)

Based on reading the actual repository READMEs and source files:

### **plato-torch → plato-ensign** (Strong, Direct)
- **Connection:** `plato-ensign` is a direct export module of `plato-torch`
- **Mechanism:** After a training room accumulates enough interaction tiles, `plato-ensign` can export the room's experience into a LoRA adapter
- **Evidence:** `plato-torch/README.md` states: "Ensigns are optional exports of room experience"
- **Direction:** One-way (training → export)

### **fleet-simulator → plato-torch** (Indirect, Data Flow)
- **Connection:** `fleet-simulator` generates training data for fleet agents
- **Mechanism:** Simulation outputs likely feed into `plato-torch` rooms as training tiles
- **Evidence:** `fleet-simulator/README.md`: "Fleet simulation for training data"
- **Direction:** Data pipeline (simulation → training)

### **holodeck-rust → (Potential) plato-torch** (Conceptual)
- **Connection:** `holodeck-rust` is a MUD server with sentiment-aware NPCs
- **Potential Link:** NPC interactions could generate training tiles for `plato-torch` rooms
- **Evidence:** Architecture section mentions "Rooms train from accumulated interaction tiles"
- **Current State:** No explicit integration found in code; appears as parallel system

### **plato-ensign → (None Downstream)** (Terminal Export)
- **Connection:** `plato-ensign` produces LoRA adapters but no clear consumption point
- **Evidence:** README focuses on export mechanics, not deployment
- **Current State:** End of pipeline

## 2. Integration Gaps (What's NOT Connected)

### **Missing: holodeck-rust → plato-torch Bridge**
- **Gap:** No mechanism found to pipe NPC interactions from `holodeck-rust` into `plato-torch` as training tiles
- **Impact:** Sentiment-aware NPC experiences remain isolated from fleet training cycles

### **Missing: plato-ensign → Deployment Pipeline**
- **Gap:** LoRA adapters export but don't automatically deploy to active agents
- **Impact:** Specialist models (ensigns) remain artifacts rather than active fleet components

### **Missing: fleet-simulator → holodeck-rust Feedback**
- **Gap:** No connection between simulation outcomes and MUD world state
- **Impact:** Simulation training data doesn't influence live NPC behavior or sentiment

### **Missing: Cross-Technology Validation**
- **Gap:** No shared test suite or integration tests across these four components
- **Evidence:** Each repo has independent tests; no `integration/` directory found
- **Impact:** Changes in one system could break implicit dependencies
