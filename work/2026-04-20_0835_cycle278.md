## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets, self‑training rooms, tile ingestion.  
**Key files:**
- `presets/DeadbandRoom.py` – Implements Deadband Protocol (P0/P1/P2).
- `presets/__init__.py` – Exports `DeadbandRoom`, `MirrorRoom`, `GhostRoom`.
- `tile_ingestor.py` – Ingests JSON tiles, validates schema.
- `room_server.py` – Flask server with `/submit_tile` endpoint.

**Connections found:**
- **plato‑ensign:** `presets/DeadbandRoom.py` imports `from plato_ensign import export_lora` (line 42). Room can export its experience as a LoRA adapter.
- **fleet‑simulator:** No direct import. However, `tile_ingestor.py` expects a tile schema that could be produced by the simulator (shared JSON format).
- **holodeck‑rust:** No direct import. No Rust interop visible in Python files.

## 2. fleet‑simulator
**Location:** `fleet-simulator/`  
**Purpose:** Generate training data by simulating fleet interactions.  
**Key files:**
- `src/simulator.rs` – Core simulation loop.
- `src/tile_generator.rs` – Produces JSON tiles matching plato‑torch schema.
- `Cargo.toml` – Depends on `serde`, `serde_json`, `rand`.

**Connections found:**
- **plato‑torch:** No Rust‑to‑Python bindings. Output is JSON files written to disk. Implicit connection via tile schema alignment.
- **plato‑ensign:** No reference.
- **holodeck‑rust:** No shared dependencies in `Cargo.toml`.

## 3. holodeck‑rust
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment‑aware NPCs.  
**Key files:**
- `src/server.rs` – TCP server, handles connections.
- `src/npc/sentiment.rs` – NPC emotion engine.
- `Cargo.toml` – Depends on `tokio`, `serde`, `uuid`.

**Connections found:**
- **plato‑torch:** None. No Python interop, no HTTP endpoint to receive tiles.
- **fleet‑simulator:** None. Separate crates.
- **plato‑ensign:** None.

## 4. plato‑ensign
**Location:** `plato-ensign/`  
**Purpose:** Export room experience into LoRA adapters.  
**Key files:**
- `export_lora.py` – Function called by `DeadbandRoom`.
- `lora_train.py` – Training logic using PyTorch.
- `requirements.txt` – Lists `torch`, `peft`, `transformers`.

**Connections found:**
- **plato‑torch:** Direct import by `DeadbandRoom`. Ensign is a library for plato‑torch rooms.
- **fleet‑simulator:** None.
- **holodeck‑rust:** None.
