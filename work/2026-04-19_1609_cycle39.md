## 2. Integration Gaps (P0 — Negative Space to Map)

### **Gap 1: GhostInjector → holodeck-rust**
- **Issue:** Ghost tiles (dead agents' lessons as P0 knowledge) not wired into holodeck NPC sentiment system  
- **Risk:** Valuable P0 knowledge isolated from interactive training environment  
- **Action:** Wire GhostInjector to feed ghost tiles into holodeck NPC decision trees  

### **Gap 2: DeadbandRoom → plato-relay**
- **Issue:** Deadband Protocol training room not connected to fleet communication relay  
- **Risk:** Protocol training occurs in isolation without real-time fleet feedback  
- **Action:** Connect DeadbandRoom API to plato-relay for live protocol validation  

### **Gap 3: flux-runtime → cudaclaw**
- **Issue:** Self-assembling bytecode VM not integrated with GPU-resident agent runtime  
- **Risk:** VM bytecode cannot leverage GPU acceleration for agent execution  
- **Action:** Create binding layer between flux-runtime bytecode and cudaclaw GPU kernels  

### **Gap 4: plato-ensign → fleet-simulator**
- **Issue:** LoRA adapters (exported room experience) not feeding back into simulation  
- **Risk:** One-way knowledge flow — trained models don't improve simulation intelligence  
- **Action:** Create adapter injection pipeline from plato-ensign outputs to fleet-simulator agent brains  

### **Gap 5: Mirror Play → holodeck-rust**
- **Issue:** Two PLATO systems iterating screen-to-screen not connected to MUD environment  
- **Risk:** Valuable cross-system learning not captured in interactive NPC training  
- **Action:** Bridge Mirror Play outputs to holodeck-rust NPC dialogue systems
