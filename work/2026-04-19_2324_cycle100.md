## 1. Verified Connections (What's Connected)

### **plato-torch ↔ fleet-simulator**
- **Connection Type:** Data pipeline (training data generation)
- **Evidence:** `fleet-simulator/README.md` states: "Generates synthetic training data for plato-torch rooms."
- **Mechanism:** Fleet simulation outputs (agent interactions, state transitions) are formatted as "tiles" and ingested by plato-torch training rooms.
- **Status:** **Connected** — functional pipeline for generating training tiles.

### **plato-torch ↔ plato-ensign**
- **Connection Type:** Model export pipeline
- **Evidence:** `plato-ensign/README.md` states: "Exports room experience from plato-torch into LoRA adapters."
- **Mechanism:** A trained plato-torch room (accumulated tiles) can be exported as a LoRA adapter via plato-ensign.
- **Status:** **Connected** — explicit export path exists.

### **holodeck-rust ↔ (indirect) plato-torch**
- **Connection Type:** Potential training data source
- **Evidence:** `holodeck-rust/README.md` describes "sentiment-aware NPCs" and "interaction logs."
- **Mechanism:** Interaction logs from holodeck could be formatted into tiles for plato-torch, but no direct import code is visible.
- **Status:** **Indirect/Planned** — logical connection but no verified import pipeline.

## 2. Integration Gaps (What's Not Connected)

### **Gap 1: holodeck-rust → plato-torch (direct tile ingestion)**
- **Issue:** No visible code in plato-torch to ingest holodeck interaction logs.
- **Impact:** Holodeck's rich NPC interactions are not automatically feeding plato-torch training rooms.
- **Priority:** **High** — would close the loop between simulation and training.

### **Gap 2: fleet-simulator → holodeck-rust (agent behavior transfer)**
- **Issue:** No mechanism to load trained ensigns (from plato-ensign) into holodeck NPCs.
- **Impact:** Fleet-trained behaviors cannot be tested in holodeck's immersive environment.
- **Priority:** **Medium** — would enable live testing of trained models.

### **Gap 3: plato-ensign → fleet-simulator (closed-loop training)**
- **Issue:** No automatic re-injection of exported ensigns back into fleet-simulator agents.
- **Impact:** Training is not yet a closed loop; ensigns are exported but not automatically redeployed.
- **Priority:** **Medium** — would create self-improving training cycles.

### **Gap 4: Deadband Protocol integration across all components**
- **Issue:** Deadband Protocol (P0/P1/P2) is documented but not visibly implemented as a shared library or service.
- **Impact:** Each component may implement protocol logic differently (or not at all).
- **Priority:** **High** — core fleet doctrine should be consistently applied.
