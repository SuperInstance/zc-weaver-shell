# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 396  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**  
- `rooms/deadband_room.py` — Deadband Protocol implementation.  
- `rooms/__init__.py` — Room registry.  
- `trainer/room_trainer.py` — Training loop.  
- `data/tile_loader.py` — Loads interaction tiles.  

**Connections:**  
- **plato-ensign:** Exports room experience to LoRA adapters.  
  - `plato-torch/export/ensign_exporter.py` imports `from plato_ensign.core import LoRAExporter`.  
  - `ensign_exporter.py` calls `LoRAExporter.save_adapter()` after training.  
- **fleet-simulator:** Provides training data.  
  - `plato-torch/data/tile_loader.py` includes `from fleet_simulator.generator import TileGenerator`.  
  - `tile_loader.py` uses `TileGenerator.generate_batch()` to simulate fleet interactions.  
- **holodeck-rust:** Not directly imported. No Rust bindings visible in Python code.  
- **constraint-theory-core, cudaclaw, flux-runtime:** No imports found.

**Integration Gaps:**  
- No direct link to `holodeck-rust`. Tiles from holodeck must be saved to shared storage before plato-torch can load them.  
- `DeadbandRoom` preset is isolated; not wired to `plato-relay` (messaging) or `ghost-injector` (P0 knowledge injection).

## 2. fleet-simulator
**Location:** `fleet-simulator/`  
**Purpose:** Generate training data by simulating fleet interactions.  
**Key Files:**  
- `generator/tile_generator.py` — Creates synthetic interaction tiles.  
- `agents/sim_agent.py` — Simulated agent behaviors.  
- `scenarios/deadband_scenario.py` — Deadband Protocol scenarios.  

**Connections:**  
- **plato-torch:** Provides training data.  
  - `fleet-simulator/generator/tile_generator.py` is imported by `plato-torch/data/tile_loader.py`.  
  - Tiles follow `plato-torch/data/schemas.py` schema.  
- **holodeck-rust:** Not directly connected.  
- **plato-ensign:** No direct imports.  
- **Other technologies:** No imports found for constraint-theory-core, cudaclaw, flux-runtime.

**Integration Gaps:**  
- Simulator does not pull real-time data from holodeck-rust (live MUD interactions).  
- Tile generation is synthetic; not yet augmented with ghost tiles (dead agents' P0 knowledge).

## 3. holodeck-rust
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**  
- `src/server/mod.rs` — MUD server core.  
- `src/npc/sentiment.rs` — NPC sentiment system.  
- `src/world/tile_store.rs` — Stores interaction tiles.  

**Connections:**  
- **plato-torch:** Indirect via tile storage.  
  - `tile_store.rs` writes tiles to `./data/tiles/` in JSON format.  
  - Plato-torch’s `tile_loader.py` can load from this directory if configured.  
- **fleet-simulator:** No direct link.  
- **plato-ensign:** No Rust bindings.  
- **Other technologies:** No imports found for constraint-theory-core, cudaclaw, flux-runtime.

**Integration Gaps:**  
- No automatic tile export to plato-torch; manual file copy required.  
- NPC sentiment system not yet connected to Deadband Protocol (P0/P1/P2 decision making).  
- Missing `GhostInjector` integration to inject P0 knowledge into NPC behaviors.

## 4. plato-ensign
**Location:** `plato-ensign/`  
**Purpose:** Export room experience into LoRA adapters.  
**Key Files:**  
- `core/lora_exporter.py` — LoRA adapter creation.  
- `adapters/deadband_adapter.py` — Deadband-specific adapter.  

**Connections:**  
- **plato-torch:** Direct import.  
  - `plato-torch/export/ensign_exporter.py` imports `LoRAExporter`.  
  - Triggered after room training completes.  
- **fleet-simulator, holodeck-rust, others:** No direct imports.

**Integration Gaps:**  
- Only exports from plato-torch rooms; no direct export from holodeck-rust NPC interactions.  
- Adapters are generic; not yet specialized for individual fleet agents (Oracle1, JetsonClaw1, Forgemaster).

## 5. Summary of Current Connections
```
plato-torch ← imports → plato-ensign (LoRA export)
plato-torch ← imports → fleet-simulator (tile generation)
holodeck-rust → writes tiles → file system → plato-torch (manual load)
```

**Missing Integrations:**  
- holodeck-rust → plato-torch (automated tile pipeline).  
- GhostInjector → holodeck-rust (P0 knowledge injection).  
- DeadbandRoom → plato-relay (real-time messaging).  
- constraint-theory-core, cudaclaw, flux-runtime not integrated with any listed component.
