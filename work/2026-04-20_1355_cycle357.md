## 1. **plato-torch** (Training Rooms)
**Location:** `plato-torch/`

### Key Files:
- `rooms/deadband_room.py` – Primary training room implementing Deadband Protocol.
- `rooms/__init__.py` – Exports `DeadbandRoom`.
- `plato_torch/__init__.py` – Core package exports.
- `requirements.txt` – Lists `torch`, `transformers`, `datasets`.

### Verified Connections:
- **Imports:** `torch`, `transformers`, `datasets` (external).
- **Exports:** `DeadbandRoom` class (used by `plato-ensign` for LoRA extraction).
- **Configuration:** `config/room_presets.yaml` defines 25 room presets, including `deadband`.
- **Data Flow:** Accepts interaction tiles (JSON) → trains via PyTorch → outputs model checkpoints.

### Integration Points:
- **To plato-ensign:** Room checkpoints are input for LoRA adapter creation.
- **To fleet-simulator:** Not directly connected. No shared modules or imports.
- **To holodeck-rust:** No direct connection. Different language (Python vs Rust).
