## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets and self-training rooms.  
**Key Files:**  
- `rooms/deadband_room.py` – Implements Deadband Protocol training loop.  
- `rooms/__init__.py` – Exports room classes (DeadbandRoom, MirrorRoom, etc.).  
- `trainer/self_trainer.py` – Self‑training logic with tile accumulation.  
- `config/room_configs.yaml` – Defines room parameters and model endpoints.

**Verified Connections:**  
- **plato‑ensign:** `deadband_room.py` imports `from plato_ensign import export_lora` (line 47). Calls `export_lora(room_state)` after training cycles.  
- **fleet‑simulator:** No direct import found. However, `room_configs.yaml` contains a `simulator_endpoint: http://localhost:8081` (unused in current code).  
- **holodeck‑rust:** No import or reference in Python files.  
- **External:** Uses `torch`, `transformers`, `httpx` (for tile submission).

**Integration Gaps:**  
- Simulator endpoint configured but not called in training loops.  
- No hook to inject simulator‑generated data into room training.  
- No sentiment‑aware NPC integration from holodeck.

## 2. fleet‑simulator
**Location:** `fleet‑simulator/`  
**Purpose:** Generate fleet‑interaction training data.  
**Key Files:**  
- `src/simulator/core.rs` – Core simulation engine.  
- `src/api/server.rs` – HTTP server with `/step` and `/reset` endpoints.  
- `config/simulator.toml` – Defines agent profiles and scenario parameters.

**Verified Connections:**  
- **plato‑torch:** None. The configured endpoint is not consumed.  
- **holodeck‑rust:** None.  
- **plato‑ensign:** None.  
- **External:** Standalone Rust binary; exposes REST API.

**Integration Gaps:**  
- Simulator runs isolated; no pipeline to feed data into plato‑torch rooms.  
- No shared agent‑state format with holodeck.

## 3. holodeck‑rust
**Location:** `holodeck‑rust/`  
**Purpose:** MUD server with sentiment‑aware NPCs.  
**Key Files:**

- `src/world/room.rs` – Room state and event handling.  
- `src/main.rs` – Starts TCP server (port 7878).

**Verified Connections:**  
- **plato‑torch:** None.  
- **fleet‑simulator:** None.  
- **plato‑ensign:** None.  
- **External:** Uses `serde`, `tokio`; no cross‑repo dependencies.

**Integration Gaps:**

- No tile‑export mechanism.

## 4. plato‑ensign
**Location:** `plato‑ensign/`  
**Purpose:** Export room experience into LoRA adapters.  
**Key Files:**  
- `src/exporter.rs` – `export_lora()` function, called by plato‑torch.  
- `src/adapters/lora_builder.rs` – Builds LoRA weights from room state.  
- `config/ensign.yaml` – LoRA rank, alpha, target modules.

**Verified Connections:**  
- **plato‑torch:** Direct import and function call (as noted above).  
- **fleet‑simulator:** None.  
- **holodeck‑rust:** None.  
- **External:** Uses `peft`, `safetensors`.

**Integration Gaps:**  
- Only exports from plato‑torch rooms; no integration with simulator or holodeck data.  
- LoRA config is static; not adaptive based on source.
