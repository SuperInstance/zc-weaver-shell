## 1. plato-torch ‚Üí fleet-simulator
**Connection:** **Strong**  
- `plato-torch/src/rooms/deadband_room.py` imports `fleet_simulator` directly:
  ```python
  from fleet_simulator import FleetSimulator, FleetState
  ```
- Uses `FleetSimulator` to generate training episodes (P0/P1/P2 trajectories).
- `DeadbandRoom` config includes `simulator_config` path pointing to `fleet-simulator/configs/`.
- Shared data structure: `FleetState` (JSON) passed as context to training loops.

**Integration Gap:**  
- No direct feedback loop from trained models back into simulator. Simulator generates static episodes; room trains on them. No live parameter tuning based on training performance.

---

## 2. plato-torch ‚Üí plato-ensign
**Connection:** **Strong**  
- `plato-torch/src/export/ensign_exporter.py` exists.
- Exporter reads room training logs (`room_logs/`), converts interaction tiles into LoRA adapter via `plato-ensign` CLI.
- Configurable export triggers: after N tiles, or manual.
- Shared data: Tile schema (`tile_type`, `content`, `metadata`) matches `plato-ensign/src/tile_processor.rs`.

**Integration Gap:**  
- Export is optional/offline step. No automatic continuous export; requires manual or scheduled job.

---

## 3. fleet-simulator ‚Üí holodeck-rust
**Connection:** **None observed**  
- `fleet-simulator/` has no Rust dependencies, no imports of `holodeck`.
- Simulator outputs `FleetState` JSON; holodeck expects `Event` stream via WebSocket.
- No shared data structures found.

**Potential Integration Point:**  
- Simulator could emit events to holodeck‚Äôs MUD server for real-time visualization/training.

---

## 4. holodeck-rust ‚Üí plato-torch
**Connection:** **Weak (config only)**  
- `holodeck-rust/config/holodeck.toml` has `training_room_endpoint` field (commented).
- Could POST sentiment-NPC interactions to a PLATO room for tile generation.
- No actual HTTP client observed in holodeck source.

**Integration Gap:**  
- One-way potential; not implemented.

---

## 5. plato-ensign ‚Üí Others
**Connection:** **Output-only**  
- `plato-ensign` consumes tiles from any room, outputs LoRA adapters.
- Adapters can be loaded by `plato-torch` rooms (theoretically), but no observed reload mechanism.
- No direct coupling to simulator or holodeck.
