# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 201  
**Phase:** 4 — Build & Test  
**Status:** Updated map based on actual code inspection of plato-torch, fleet-simulator, holodeck-rust, and plato-ensign.

Based on reading the actual repository files, here is the current state of connections between the four core fleet technologies.

## 1. plato-torch (Training Rooms)
**Purpose:** Provides 25 training room presets and self-training rooms.  
**Key Files:** `rooms/deadband_room.py`, `rooms/__init__.py`, `trainers/`, `tiles/`.  
**Current Connections:**  
- **To plato-ensign:** Direct export path. `plato-torch/rooms/base_room.py` has `export_to_ensign()` method that calls `plato_ensign.export.export_room()`. Verified in code.  
- **To fleet-simulator:** No direct import found. Potential integration via tile format (`Tile` class) but no explicit imports.  
- **To holodeck-rust:** No connection found. No Rust imports or API calls in Python code.  

**Integration Status:** Connected to plato-ensign (live). Isolated from fleet-simulator and holodeck-rust.

## 2. fleet-simulator (Fleet Simulation)
**Purpose:** Generates training data via simulated fleet interactions.  
**Key Files:** `src/simulator.rs`, `src/agents/`, `src/tiles.rs`.  
**Current Connections:**  
- **To plato-torch:** Indirect. `fleet-simulator/src/tiles.rs` defines `SimTile` struct that could be converted to plato-torch `Tile` format, but no serialization/deserialization bridge exists.  
- **To plato-ensign:** No connection found. No mention in Rust code.  
- **To holodeck-rust:** No connection found. Separate crates.  

**Integration Status:** Isolated. Generates its own tile format, not yet bridged to plato-torch or holodeck.

## 3. holodeck-rust (Sentiment-Aware MUD)
**Purpose:** Multi‑user dungeon server with NPC sentiment modeling.  
**Key Files:** `src/server.rs`, `src/npc/`, `src/world/`.  
**Current Connections:**  
- **To plato-torch:** None. No Python interop or tile ingestion.  
- **To fleet-simulator:** None. Separate simulation domains.  
- **To plato-ensign:** None. No export hooks.  

**Integration Status:** Fully isolated. Operates as a standalone Rust MUD server.

## 4. plato-ensign (LoRA Adapter Export)
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:** `export/export_room.py`, `lora/`, `configs/`.  
**Current Connections:**  
- **To plato-torch:** Direct import. `plato_ensign.export.export_room()` expects a `BaseRoom` instance from plato-torch.  
- **To fleet-simulator:** None. No Rust/Python bridge.  
- **To holodeck-rust:** None.  

**Integration Status:** Tightly coupled with plato-torch only.

2. **holodeck‑rust ↔ plato‑torch**

   - **Safe Channel:** Add a `TileLogger` system to holodeck that writes JSON tiles to a shared directory watched by plato‑torch.
