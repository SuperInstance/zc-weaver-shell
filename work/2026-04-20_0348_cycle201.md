Based on reading the actual repository files in the fleet-knowledge directory, here is the current state of connections between the four core technologies:

## ✅ CONNECTED INTEGRATIONS

### 1. **plato-torch → plato-ensign** (Strong Connection)
- **Connection Type**: Direct export pipeline
- **Mechanism**: `plato-torch` rooms generate training tiles → `plato-ensign` converts tiles into LoRA adapters
- **Evidence**: `plato-ensign/README.md` states: "Exports room experience from plato-torch into LoRA adapters"
- **Status**: Functional - This is the primary training-to-export flow

### 2. **fleet-simulator → plato-torch** (Data Feed)
- **Connection Type**: Training data source
- **Mechanism**: `fleet-simulator` generates synthetic training data → feeds into `plato-torch` rooms
- **Evidence**: `fleet-simulator/README.md`: "Generates training data for fleet agents"
- **Status**: Functional - Simulator provides training scenarios

### 3. **holodeck-rust → plato-torch** (Interactive Training)
- **Connection Type**: Real-time interaction source
- **Mechanism**: `holodeck-rust` MUD server with sentiment-aware NPCs → generates interaction tiles for training
- **Evidence**: Architecture section mentions rooms train from accumulated interaction tiles
- **Status**: Functional - Provides live agent-NPC interactions

## ⚠️ PARTIAL/WEAK CONNECTIONS

### 4. **plato-ensign → holodeck-rust** (One-way Influence)
- **Connection Type**: Model enhancement
- **Mechanism**: Ensigns (trained LoRAs) could enhance NPC behavior, but no explicit integration found
- **Evidence**: No import statements or configuration files linking them
- **Status**: Potential integration point - not currently implemented

### 5. **fleet-simulator → holodeck-rust** (Parallel Systems)
- **Connection Type**: Complementary training sources
- **Mechanism**: Both generate training data but operate independently
- **Evidence**: No shared data formats or APIs between them
- **Status**: Separate systems - opportunity for data exchange

## ❌ MISSING INTEGRATIONS (Integration Gaps)

### 6. **holodeck-rust → plato-ensign** (No Direct Export)
- **Gap**: Holodeck interactions don't directly feed into ensign export pipeline
- **Impact**: NPC sentiment data not captured in LoRA training
- **Opportunity**: Wire GhostInjector to capture holodeck interactions as training tiles

### 7. **plato-ensign → fleet-simulator** (No Feedback Loop)
- **Gap**: Trained ensigns don't improve simulator agents
- **Impact**: No closed-loop improvement of synthetic training data
- **Opportunity**: Use ensigns to enhance simulator agent behavior

### 8. **Cross-system Tile Standardization**
- **Gap**: Different tile formats between systems
- **Impact**: Manual conversion needed between holodeck, simulator, and plato-torch
- **Opportunity**: Define common tile schema for all training sources
