# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 180  
**Phase:** 4 — Build & Test  
**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

---

## 1. **plato-torch** → **plato-ensign**  
**Connection:** **Direct and functional**  
- `plato-torch` exports training room experience via `export_room()` method.  
- `plato-ensign` ingests `.pt` checkpoint files and converts them to LoRA adapters.  
- Pipeline: `Room.train()` → `Room.export_room()` → `ensign-cli convert --input room.pt --output lora.safetensors`.  
- **Integration gap:** No automated trigger—export is manual. Could be wired to room completion events.

---

## 2. **fleet-simulator** → **plato-torch**  
**Connection:** **Data pipeline (one-way)**  
- `fleet-simulator` generates `.jsonl` training logs from simulated fleet interactions.  
- `plato-torch` includes a `DataLoader` that reads these logs and creates training tiles.  
- Path: `fleet-simulator/outputs/` → `plato-torch/data/raw/` → `TileDataset`.  
- **Integration gap:** No real-time streaming; files are copied manually. Simulator does not call Plato API.

---

## 3. **holodeck-rust** → **plato-torch**  
**Connection:** **Planned but not implemented**  
- `holodeck-rust` is a MUD server with sentiment-aware NPCs.  
- `plato-torch` has a `HolodeckRoom` preset (stub) intended to ingest NPC interaction logs.  
- Current state: `HolodeckRoom` exists but reads from static files, not live socket.  
- **Integration gap:** No live connection between holodeck server and Plato room. Could use WebSocket or REST.

---

## 4. **plato-ensign** → **holodeck-rust**  
**Connection:** **None**  
- `plato-ensign` produces LoRA adapters for inference.  
- `holodeck-rust` uses hardcoded behavior trees for NPCs.  
- **Integration gap:** No mechanism to load a trained ensign into an NPC. Could embed `llama.cpp` or similar in Rust.

---

## 5. **fleet-simulator** → **holodeck-rust**  
**Connection:** **None**  
- Separate concerns: simulator trains fleet coordination, holodeck trains human‑like interaction.  
- **Integration gap:** No shared scenario format. Could align on a common event schema (e.g., `(actor, action, sentiment)`).

---

## 6. **Ghost Tiles & Deadband Protocol**  
**Connection:** **Conceptual, not wired**  
- Ghost tiles (dead agents’ lessons) are stored as P0 negative‑space examples.  
- `DeadbandRoom` in `plato-torch` includes a `GhostInjector` class to load them.  
- **Integration gap:** Ghost tiles are not automatically fed into holodeck or simulator. Could be used to initialize NPC aversion profiles.
