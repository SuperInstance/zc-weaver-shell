# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 195  
**Phase:** 4 — Build & Test  
**Status:** Updated map based on actual code inspection of plato-torch, fleet-simulator, holodeck-rust, and plato-ensign repositories.

## 1. Current Verified Connections

### plato-torch → plato-ensign (Strong, Direct)
- **Connection:** `plato-torch` exports trained room state via `export_room()` function.
- **Mechanism:** Room state (interaction tiles, weights) serialized to `.pt` files.
- **plato-ensign** imports these via `load_room_checkpoint()` and converts to LoRA adapters.
- **Evidence:** `plato-ensign/src/ensign.rs` calls `plato_torch::export::export_room()`.

### fleet-simulator → plato-torch (Data Pipeline)
- **Connection:** `fleet-simulator` generates training episodes (JSONL).
- **plato-torch** ingests via `DataLoader` in `training/`.
- **Evidence:** `plato-torch/training/data_loader.py` reads `fleet-simulator/output/episodes/*.jsonl`.

### holodeck-rust → fleet-simulator (NPC Sentiment Data)
- **Connection:** `holodeck-rust` logs NPC sentiment events to `logs/sentiment_events.ndjson`.
- **fleet-simulator** reads these logs to enrich episode generation.
- **Evidence:** `fleet-simulator/src/enrichment.rs` has `ingest_holodeck_sentiment()`.

## 2. Integration Gaps (What's Not Connected)

### plato-ensign → holodeck-rust (Missing)
- **Gap:** No mechanism to load trained ensigns (LoRA) into holodeck NPCs.
- **Potential:** Could allow NPCs to learn from room training.
- **Current:** holodeck NPCs use static sentiment models.

### GhostInjector → Any System (Isolated)
- **Gap:** `GhostInjector` tool exists in `tools/` but not wired.
- **Purpose:** Injects "ghost tiles" (P0 lessons) into training streams.
- **Current:** Not called by plato-torch or fleet-simulator.

### DeadbandRoom → plato-relay (Not Implemented)
- **Gap:** DeadbandRoom preset exists but lacks relay integration.
- **plato-relay** handles multi-room synchronization.
- **Current:** DeadbandRoom operates as standalone training room.

### cudaclaw / flux-runtime → Core Training (Disconnected)
- **Gap:** These GPU/runtime systems are separate repos.
- **No visible imports** in plato-torch or fleet-simulator.
- **Potential:** Could accelerate training or bytecode generation.
