## 1. plato-torch
**Repository:** `plato-torch`  
**Primary Role:** Training room server with 25 preset rooms, self-training capabilities.  
**Key Files:**  
- `rooms/deadband_room.py` — Deadband Protocol training room.  
- `rooms/__init__.py` — Room registry.  
- `server.py` — Main server loop.  
- `models/tile_model.py` — Tile data structure.  

**Verified Connections:**  
- **plato-ensign:** Direct import in `rooms/deadband_room.py`:  
  ```python
  from plato_ensign import export_room_experience
  ```
  Used to export room experience as LoRA adapters.  
- **fleet-simulator:** No direct import found in code. Configuration reference in `configs/room_presets.yaml` mentions `fleet-simulator` as a data source for training scenarios.  
- **holodeck-rust:** No direct import. Not referenced in code or configs.  

**Integration Points:**  
- `tile_model.py` defines `TrainingTile` class used across fleet.  
- Room presets can load external scenario data (like from fleet-simulator).  
- Export hook to plato-ensign is built-in.

## 2. fleet-simulator
**Repository:** `fleet-simulator`  
**Primary Role:** Generate training data through multi-agent fleet simulations.  
**Key Files:**  
- `src/simulation/engine.rs` — Core simulation engine.  
- `src/agents/mod.rs` — Agent definitions.  
- `src/output/tile_exporter.rs` — Exports simulation events as training tiles.  

**Verified Connections:**  
- **plato-torch:** No direct Rust→Python imports. Outputs JSON files in `TrainingTile` format. These files are read by plato-torch via file system (configured in `configs/room_presets.yaml`).  
- **plato-ensign:** No direct connection.  
- **holodeck-rust:** No direct import.  

**Integration Points:**  
- Exports tiles to `data/tiles/` directory.  
- Tile format matches plato-torch’s `TrainingTile` schema (verified by field alignment).  
- Can be triggered as a data-generation pipeline step.

## 3. holodeck-rust
**Repository:** `holodeck-rust`  
**Primary Role:** MUD server with sentiment-aware NPCs for interactive training.  
**Key Files:**  
- `src/server/mud_server.rs` — Main server.  
- `src/npc/sentiment_engine.rs` — NPC sentiment logic.  
- `src/events/mod.rs` — In-game events.  

**Verified Connections:**  
- **plato-torch:** No direct connection found.  
- **fleet-simulator:** No direct connection.  
- **plato-ensign:** No direct connection.  
- **constraint-theory-core:** Imported in `src/world/geometry.rs`:  
  ```rust
  use constraint_theory_core::snapping;
  ```
  Used for geometric snapping in world building.

**Integration Points:**  
- Could emit training tiles via event system (not yet implemented).  
- Uses constraint-theory-core for spatial reasoning.  
- Standalone service on port 8081.

## 4. plato-ensign
**Repository:** `plato-ensign`  
**Primary Role:** Export room experience from plato-torch into LoRA adapters.  
**Key Files:**  
- `src/exporter.py` — Main export logic.  
- `src/lora_builder.py` — LoRA adapter creation.  
- `configs/export_config.yaml` — Export settings.  

**Verified Connections:**  
- **plato-torch:** Directly imported by plato-torch (as above). Called via `export_room_experience()` hook.  
- **fleet-simulator:** No direct connection.  
- **holodeck-rust:** No direct connection.  

**Integration Points:**  
- Triggered from plato-torch room lifecycle.  
- Reads tile batches from plato-torch memory.  
- Outputs LoRA adapters to `output/adapters/`.
