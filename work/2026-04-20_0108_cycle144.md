# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 144  
**Phase:** 4 — Build & Test  
**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

---

## 1. **plato-torch** → **fleet-simulator**
**Connection:** ✅ **Connected via `fleet-simulator` as a training data source.**  
- `plato-torch/rooms/deadband_room.py` imports `fleet_simulator` and uses `FleetSimulator` class to generate simulation episodes.  
- The room calls `simulator.generate_episode()` to produce training episodes, which are then formatted into tiles.  
- This is a **direct dependency** — plato-torch cannot run DeadbandRoom without fleet-simulator installed.  
**Integration Gap:** None — this is a working, documented integration.

---

## 2. **plato-torch** → **holodeck-rust**
**Connection:** ❌ **Not directly connected.**  
- No import of `holodeck` found in plato-torch source files.  
- holodeck-rust is a standalone MUD server written in Rust, with sentiment-aware NPCs.  
- Potential integration point: `holodeck` could be used as an alternative training environment (like fleet-simulator) to generate interaction tiles.  
**Integration Gap:** **Major** — holodeck is currently isolated. No pipeline exists to feed holodeck interactions into plato-torch rooms.

---

## 3. **plato-torch** → **plato-ensign**
**Connection:** ✅ **Connected via export functionality.**  
- `plato-torch/ensign/exporter.py` provides `export_room_to_lora()` which takes a trained room and exports a LoRA adapter.  
- The ensign system reads room state and training tiles to produce a specialized model.  
- This is an **optional post-training step** — not required for room operation.  
**Integration Gap:** None — export pathway exists and is documented.

---

## 4. **fleet-simulator** → **holodeck-rust**
**Connection:** ❌ **No connection found.**  
- fleet-simulator is a Python-based fleet interaction simulator.  
- holodeck-rust is a Rust MUD server.  
- No shared protocols or data exchange observed in code.  
**Integration Gap:** **Major** — these are two separate simulation environments with no interoperability.

---

## 5. **fleet-simulator** → **plato-ensign**
**Connection:** ⚠️ **Indirect via plato-torch.**  
- fleet-simulator generates data for plato-torch rooms.  
- plato-ensign can export those trained rooms.  
- No direct fleet-simulator → plato-ensign pipeline.  
**Integration Gap:** **Moderate** — ensign export depends on plato-torch as intermediary.

---

## 6. **holodeck-rust** → **plato-ensign**
**Connection:** ❌ **No connection found.**  
- holodeck interactions are not currently captured as training tiles.  
- No exporter exists for holodeck session data.  
**Integration Gap:** **Major** — valuable NPC sentiment data is not being used for ensign training.
