# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 148  
**Phase:** 4 — Build & Test  
**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current state of integration between the four core technologies: **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

## 1. Verified Connections (What IS Connected)

### plato-torch ↔ fleet-simulator
- **Connection:** Direct integration via `fleet-simulator` as a data source.
- **Evidence:** `plato-torch/rooms/deadband_room.py` imports `fleet_simulator` and uses `FleetSimulator` class to generate training episodes.
- **Data Flow:** `fleet-simulator` simulates fleet interactions → produces episode data (state, action, reward) → `DeadbandRoom` ingests this as training tiles.
- **Purpose:** Generates synthetic training data for Deadband Protocol (P0/P1/P2) learning.

### plato-torch ↔ plato-ensign
- **Connection:** Direct export pipeline.
- **Evidence:** `plato-ensign/export.py` imports `torch` and `peft`; designed to load a trained `DeadbandRoom` model and export its adapter weights as a LoRA.
- **Data Flow:** Trained room model (`.pth` checkpoint) → `plato-ensign` extracts LoRA → saves as `.safetensors` file.
- **Purpose:** Turns room experience into portable specialist models (ensigns).

### fleet-simulator → holodeck-rust (Planned)
- **Connection:** Conceptual link documented.
- **Evidence:** `fleet-simulator/README.md` states: "Can feed simulated agent interactions into holodeck-rust for sentiment-aware NPC training."
- **Current State:** No observed import or direct API call in code. This is a documented future integration point.

## 2. Integration Gaps (What Is NOT Connected)

### holodeck-rust ↔ plato-torch
- **Gap:** No direct connection observed.
- **Evidence:** `holodeck-rust/src/` shows no Python bindings or HTTP API for `plato-torch` to call. `plato-torch` does not import or reference `holodeck-rust`.
- **Implication:** Sentiment-aware NPC interactions from holodeck are not currently feeding into room training.

### holodeck-rust ↔ plato-ensign
- **Gap:** No connection.
- **Evidence:** `plato-ensign` only handles `plato-torch` model checkpoints. No hooks for Rust-based MUD state.
- **Implication:** NPC behavior patterns cannot be exported as ensigns.

### fleet-simulator ↔ plato-ensign (Direct)
- **Gap:** No direct export from simulator to ensign.
- **Evidence:** `fleet-simulator` outputs episode dicts, not model weights. `plato-ensign` expects a trained PyTorch model.
- **Implication:** Simulation data must pass through a `plato-torch` room first to become an ensign.
