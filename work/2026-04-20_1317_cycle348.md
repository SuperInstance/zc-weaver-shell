## 1. plato-torch → fleet-simulator  
**Connection:** **Strong** (direct import, shared data flow)  
- **plato-torch/src/rooms/deadband_room.py** imports `from fleet_simulator import FleetSimulator`  
- **plato-torch/src/rooms/deadband_room.py** line 47: `self.simulator = FleetSimulator()`  
- **plato-torch/src/rooms/deadband_room.py** line 89: `simulation_data = self.simulator.run_scenario(scenario)`  
- **Data flow:** DeadbandRoom generates training scenarios → fleet-simulator runs them → returns simulation results → used to create training tiles.  
- **Integration gap:** None observed. This is a working, direct dependency.

---

## 2. plato-torch → holodeck-rust  
**Connection:** **Weak** (configuration reference only, no direct import)  
- **plato-torch/configs/holodeck_config.yaml** exists and defines MUD server settings (host, port, NPC sentiment thresholds).  
- **plato-torch/src/rooms/deadband_room.py** does **not** import holodeck-rust.  
- **plato-torch/src/rooms/deadband_room.py** line 112: `# TODO: Connect to holodeck for NPC interactions` (commented out).  
- **Integration gap:** No live connection. Configuration is present but unused. Potential channel: via REST API (holodeck-rust exposes HTTP endpoints).

---

## 3. plato-torch → plato-ensign  
**Connection:** **Moderate** (export pathway, but not automatic)  
- **plato-ensign/export_lora.py** imports `from plato_torch.rooms.base import BaseRoom`  
- **plato-torch/src/rooms/deadband_room.py** has method `export_training_data()` that writes `.pt` files to `output/tiles/`.  
- **plato-ensign/scripts/train_lora.py** reads from `output/tiles/` directory.  
- **Integration gap:** Manual step required. Room must explicitly call `export_training_data()`, then separate script `train_lora.py` runs. No automatic trigger.

---

## 4. fleet-simulator → holodeck-rust  
**Connection:** **None** (no imports, no config references)  
- **fleet-simulator/src/lib.rs** has no mention of "holodeck".  
- **fleet-simulator/Cargo.toml** does not list holodeck-rust as dependency.  
- **Integration gap:** Complete separation. Fleet-simulator generates synthetic fleet data; holodeck-rust manages NPC interactions. No shared data structures.

---

## 5. fleet-simulator → plato-ensign  
**Connection:** **Indirect via plato-torch**  
- Fleet-simulator data flows into plato-torch rooms → becomes training tiles → plato-ensign can consume those tiles.  
- No direct link between fleet-simulator and plato-ensign.

---

## 6. holodeck-rust → plato-ensign  
**Connection:** **None**  
- **holodeck-rust/src/main.rs** exports NPC interaction logs as JSON.  
- **plato-ensign** expects `.pt` tensor files. Format mismatch.  
- **Integration gap:** Data format incompatibility. Would require a transformer (JSON → tensor) to bridge.
