# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 250

**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets, self-training rooms, tile processing.

### Verified Connections:
- **To plato-ensign:** Direct import.  
  `plato-torch/src/rooms/deadband_room.py` imports `from plato_ensign import EnsignExporter`.  
  The `DeadbandRoom` class has an `export_to_ensign()` method that calls `EnsignExporter.export(room_state)`.
  
- **To fleet-simulator:** Indirect via data format.  
  `plato-torch/src/data/tile_schema.py` defines `TrainingTile` — a Pydantic model with fields `agent_id`, `timestamp`, `content`, `metadata`.  
  `fleet-simulator/src/simulation/tile_generator.rs` produces JSON that matches this schema. No direct Rust→Python import, but schema alignment suggests intentional interoperability.

- **To holodeck-rust:** No direct import found.  
  `plato-torch/` does not reference `holodeck-rust` in Python code or configs.

### Integration Points:
- `DeadbandRoom` → `EnsignExporter` is live.
- Tile schema is shared with fleet-simulator (implied).
- Room presets are isolated from holodeck.

---

## 2. fleet-simulator
**Location:** `fleet-simulator/`  
**Purpose:** Generate training data via multi-agent simulation.

### Verified Connections:
- **To plato-torch:** Schema alignment only.  
  `fleet-simulator/src/simulation/tile_generator.rs` creates `SimulationTile` struct with fields `agent_id: String`, `timestamp: DateTime<Utc>`, `content: String`, `metadata: serde_json::Value`.  
  This mirrors `plato-torch`'s `TrainingTile`. No direct import of Python modules.

- **To holodeck-rust:** No import found.  
  `fleet-simulator/Cargo.toml` does not list `holodeck` as a dependency.

- **To plato-ensign:** No import found.

### Integration Points:
- Tile data format is compatible with plato-torch (JSON).
- No runtime connection to holodeck or ensign.

## 3. holodeck-rust
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs.

### Verified Connections:
- **To plato-torch:** None.  
  `holodeck-rust/src/main.rs` and `src/npc/` show no Python interop.

- **To fleet-simulator:** None.  
  `Cargo.toml` has no `fleet-simulator` dependency.

- **To plato-ensign:** None.

### Integration Points:

## 4. plato-ensign
**Location:** `plato-ensign/`  
**Purpose:** Export room experience to LoRA adapters.

### Verified Connections:
- **To plato-torch:** Direct import by plato-torch (as above).  
  `plato-ensign/src/exporter.py` defines `EnsignExporter` class.  
  `plato-torch` calls it; ensign does not import torch.

- **To fleet-simulator:** None.

- **To holodeck-rust:** None.

### Integration Points:
- Exporter is called from `DeadbandRoom`.  
- No hooks for holodeck or simulator.

## 5. GhostInjector
**Location:** `plato-torch/src/utils/ghost_injector.py`  
**Purpose:** Inject "ghost tiles" (dead agents' lessons) into rooms.

### Verified Connections:
- **To holodeck-rust:** None currently.  
  `GhostInjector` reads from `memory/` directory and injects into `DeadbandRoom`.  
  No reference to holodeck in code.

### Integration Gap:
GhostInjector could feed historical failure patterns into holodeck NPCs (sentiment-aware learning). Not wired.

## 6. plato-relay
**Location:** `plato-relay/`  
**Purpose:** Relay tiles between rooms and external services.

### Verified Connections (inspected):
- `plato-relay/src/relay.py` imports `from plato_torch.rooms.deadband_room import DeadbandRoom`.  
- Listens for `room_updated` events, forwards tiles via WebSocket.

### Integration Gap:
`DeadbandRoom` does not yet emit events to plato-relay (no event registration in current `deadband_room.py`).

## Summary: What’s Connected vs. What’s Not

### Connected:
1. **plato-torch → plato-ensign** — Direct export path.
2. **plato-torch ↔ fleet-simulator** — Shared tile schema (implicit).
3. **plato-torch → plato-relay** — Import exists, but event flow not activated.

### Not Connected:
1. **holodeck-rust ↔ any** — Isolated.
2. **fleet-simulator → plato-ensign** — No export path from sim data.
3. **GhostInjector → holodeck-rust** — No injection point.
4. **DeadbandRoom → plato-relay** — Events not wired.
