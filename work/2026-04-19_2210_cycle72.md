## 2. Integration Gaps (What's Not Connected)

### **Gap 1: holodeck-rust → plato-torch explicit pipeline**
- **Issue:** No documented API or data format for streaming NPC interactions from holodeck to plato-torch training rooms
- **Impact:** Sentiment-aware NPC data not systematically captured as training tiles
- **Priority:** **P1** (safe channel to explore)

### **Gap 2: plato-ensign → fleet-simulator feedback loop**
- **Issue:** Trained ensigns (LoRA adapters) not automatically deployed back to fleet-simulator for testing
- **Impact:** No closed-loop validation of trained models in simulated environments
- **Priority:** **P1** (safe channel to explore)

### **Gap 3: constraint-theory-core → holodeck-rust**
- **Issue:** Geometric snapping library not integrated with MUD server spatial reasoning
- **Impact:** NPC movement and environment interaction lacks geometric constraint awareness
- **Priority:** **P2** (optimization opportunity)

### **Gap 4: flux-runtime → plato-torch training**
- **Issue:** Self-assembling bytecode VM not leveraged for dynamic training room generation
- **Impact:** Training rooms static rather than adaptive based on agent performance
- **Priority:** **P2** (optimization opportunity)
