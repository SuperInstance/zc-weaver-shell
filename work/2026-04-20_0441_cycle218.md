## 1. Verified Connections (Based on Code)

### plato-torch → fleet-simulator
- **Connection:** Strong, direct integration.
- **Evidence:** `plato-torch/src/training_rooms/deadband_room.py` imports `fleet_simulator` and uses `FleetSimulator` class.
- **Data Flow:** DeadbandRoom initializes a FleetSimulator instance, passes agent actions to it, and receives simulation state (positions, collisions) for training.
- **Purpose:** Generates synthetic fleet navigation data for P0/P1/P2 protocol training.

### plato-torch → plato-ensign
- **Connection:** Optional export pipeline.
- **Evidence:** `plato-torch/src/export/ensign_exporter.py` exists, reads room training logs, creates LoRA adapters.
- **Data Flow:** Training tiles → room logs → ensign_exporter → LoRA adapter (.safetensors).
- **Purpose:** Specialized model fine‑tunes from room experience.

### holodeck-rust → (none directly)
- **Connection:** Currently isolated in codebase.
- **Evidence:** `holodeck-rust/src/` shows no imports of plato-torch, fleet-simulator, or plato-ensign. It’s a standalone MUD server with sentiment‑aware NPCs.
- **Note:** May be integrated via external orchestration (not in source).

### fleet-simulator → (none downstream)
- **Connection:** Serves plato-torch only.
- **Evidence:** `fleet-simulator/src/lib.rs` exports simulator API; no imports of other fleet components.
- **Purpose:** Pure simulation engine.

### plato-ensign → (none upstream except plato-torch)
- **Connection:** Depends on plato-torch logs.
- **Evidence:** `plato-ensign/src/main.rs` reads `training_logs/` directory (expected from plato-torch).
- **Purpose:** Post‑training model export.

## 2. Integration Gaps (What’s Not Connected)

### Gap 1: holodeck-rust ↔ plato-torch
- **Issue:** No shared data pipeline. Holodeck NPC interactions are not feeding into PLATO training rooms.
- **Opportunity:** Connect holodeck event stream to DeadbandRoom or a new “SentimentRoom” to train on social navigation.

### Gap 2: fleet-simulator → holodeck-rust
- **Issue:** Simulation geometry (constraint‑theory‑core) not used for holodeck spatial reasoning.
- **Opportunity:** Use fleet‑simulator’s collision/position math for holodeck room layouts or NPC pathing.

### Gap 3: plato-ensign → holodeck-rust
- **Issue:** Ensign LoRA adapters not loaded into holodeck NPC AI.
- **Opportunity:** Hot‑swap NPC behavior models with ensign exports for adaptive NPCs.

### Gap 4: Ghost tiles → active pipelines
- **Issue:** GhostInjector (P0 knowledge from dead agents) is not wired into any current training loop.
- **Opportunity:** Inject ghost tiles as negative examples (P0) into DeadbandRoom or a dedicated “GhostRoom”.
