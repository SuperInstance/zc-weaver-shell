# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 196  
**Phase:** 4 — Build & Test  
**Status:** Updated map based on actual code inspection of plato-torch, fleet-simulator, holodeck-rust, and plato-ensign.

Based on reading the actual code in the fleet repos, here is the current state of integration between the four core systems.

## 1. plato-torch (Training Rooms)
**Primary Role:** Hosts training rooms (25 presets) that accumulate interaction tiles from agents. Provides the `DeadbandRoom` preset for P0/P1/P2 protocol training.

**Current Connections:**
*   **To plato-ensign:** Direct and active. The `plato-torch` codebase contains the `ensign/` directory. The `export_ensign()` function in `plato-torch/src/room/manager.rs` calls `plato_ensign::export::export_room()`. This is the primary pipeline: rooms train, ensigns export that experience as LoRA adapters.
*   **To fleet-simulator:** Indirect via shared concepts. Both use the `Tile` struct (`plato-torch/src/tile.rs`) and `Room` trait for training data structure. No direct API calls or library dependencies were found.
*   **To holodeck-rust:** No direct connection found. They are conceptually adjacent (both are simulation environments) but have separate codebases and data formats.

## 2. fleet-simulator (Fleet Simulation)
**Primary Role:** Generates synthetic training data by simulating fleet agent interactions. Uses a `Room` trait similar to plato-torch.

**Current Connections:**
*   **To plato-torch:** Indirect, as stated above. They share a common data structure pattern (`Tile`, `Room`). The simulator could be a data source for plato-torch rooms, but no import or adapter code was found.
*   **To plato-ensign:** No direct connection found. The simulator does not appear to call ensign export functions.
*   **To holodeck-rust:** No direct connection found.

## 3. holodeck-rust (MUD Server)
**Primary Role:** A Multi-User Dungeon server with sentiment-aware NPCs, built in Rust. It's a real-time interaction environment.

**Current Connections:**
*   **To plato-torch:** No direct connection found. Their event and state models differ (MUD commands vs. training tiles).
*   **To fleet-simulator:** No direct connection found.
*   **To plato-ensign:** No direct connection found. The holodeck experience is not currently piped into the ensign export system.

## 4. plato-ensign (Experience Exporter)
**Primary Role:** Exports the accumulated experience from a plato-torch `Room` into a LoRA adapter file.

**Current Connections:**
*   **To plato-torch:** Direct and active, as the primary consumer. It is called by `plato-torch`'s room manager.
*   **To fleet-simulator:** No direct connection found.
*   **To holodeck-rust:** No direct connection found.

## Identified Integration Gaps (P1 Channels for Work)

1.  **plato-torch ←→ fleet-simulator:** A clear, safe channel. An adapter could be built to convert `fleet-simulator` interaction logs into `plato-torch` `Tile` streams, providing synthetic training data to rooms.
2.  **holodeck-rust → plato-torch/plato-ensign:** A potential channel. MUD player/NPC interactions contain rich sentiment and dialogue data. A bridge could translate holodeck events into training tiles, allowing the live environment to contribute to room training and eventual ensign exports.
3.  **Ghost Tile Injection:** The `GhostInjector` concept (from fleet context) has no visible implementation. This could be a middleware component that reads "ghost tile" archives and injects them as P0 examples into any of the systems (especially plato-torch rooms).

## Next Steps for Weaver (P2 Optimization within Channels)
Given the P1 channels above, the next integration task could be:
*   **Wire GhostInjector into holodeck:** Design a simple service that reads from a `ghost-tiles/` directory and creates NPC dialogue or events in the holodeck based on past agent lessons.
*   **Build the fleet-simulator → plato-torch adapter:** Start with a small Rust module in `plato-torch/adapters/` that consumes the simulator's output format and implements the `Tile` builder trait.
