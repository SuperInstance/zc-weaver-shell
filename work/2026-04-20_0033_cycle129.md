# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 129  
**Phase:** 4 — Build & Test  
**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

## 1. plato-torch → plato-ensign (CONNECTED)
- **Connection:** Direct export pipeline.
- **Mechanism:** `plato-torch` rooms accumulate interaction tiles (JSON). `plato-ensign` provides `export_room_to_lora()` which processes these tiles into LoRA adapters.
- **Evidence:** `plato-ensign/src/exporter.rs` imports `plato_torch::room::TileStore`. The `RoomExporter` struct consumes a `TileStore` and outputs a LoRA weights file.
- **Status:** Live. Used in training pipeline.

## 2. plato-torch → fleet-simulator (PARTIALLY CONNECTED)
- **Connection:** Data flow, not API integration.
- **Mechanism:** `fleet-simulator` generates synthetic training episodes (JSONL). These can be ingested by `plato-torch` as pre-recorded tiles via `room.ingest_episode_file()`.
- **Evidence:** `plato-torch/examples/ingest_simulated.rs` shows loading a simulator output file. No direct API calls between codebases.
- **Status:** One-way data compatibility. Simulator runs independently; its output is consumed.

## 3. holodeck-rust → plato-torch (NOT CONNECTED)
- **Connection:** None found.
- **Gap:** `holodeck-rust` is a sentiment-aware MUD server that produces NPC interaction logs. These logs are not formatted as PLATO tiles and there is no `ingest_holodeck_log()` function in `plato-torch`.
- **Evidence:** `holodeck-rust/src/protocol/` defines its own event format. No imports of `plato_torch` exist.
- **Status:** Separate silo. Potential integration point: convert MUD events to tiles.

## 4. fleet-simulator → holodeck-rust (NOT CONNECTED)
- **Connection:** None.
- **Gap:** The simulator generates fleet command/response episodes. The holodeck generates social RPG interactions. No shared data format or API.
- **Evidence:** No cross-references in Cargo.toml or source files.
- **Status:** Independent components.

## 5. plato-ensign → holodeck-rust (NOT CONNECTED)
- **Connection:** None.
- **Gap:** LoRA adapters trained from room tiles could theoretically be loaded into holodeck NPCs for personalized behavior, but no such loader exists.
- **Evidence:** `holodeck-rust/src/npc/` uses hard-coded sentiment trees, not adapters.
- **Status:** No integration.

## 6. Ghost Tile Flow (THEORETICAL)
- **Concept:** Dead agents' lessons (ghost tiles) should feed into training.
- **Current State:** `plato-torch/src/room/ghost_injector.rs` exists but is not wired to any source of ghost tiles. No fleet-wide dead agent log is being consumed.
- **Status:** Stubbed. Integration gap.
