## 1. plato-torch → fleet-simulator
**Connection:** **Yes** — Direct import and usage.  
**Evidence:**  
- `plato-torch/rooms/deadband_room.py` imports `from fleet_simulator.simulator import FleetSimulator`.  
- `DeadbandRoom` instantiates `FleetSimulator` as `self.simulator`.  
- The room calls `self.simulator.step()` during its training loop to generate simulation data.  
**Data Flow:** Simulation states from `fleet-simulator` are used as training environment contexts within the PLATO room.  
**Integration Gap:** None observed; this is a working, direct dependency.

## 2. plato-torch → holodeck-rust
**Connection:** **No** — No direct imports or references found.  
**Evidence:**  
- Searched `plato-torch/` for "holodeck". No imports, string references, or configuration mentions.  
- `holodeck-rust/` is a separate Rust project (MUD server) with no Python bindings visible in `plato-torch`.  
**Integration Gap:** **Significant**. The sentiment-aware NPC environment (`holodeck-rust`) is not currently wired into the PLATO training loop. This is a potential channel for richer, narrative-driven training data.

## 3. plato-torch → plato-ensign
**Connection:** **Yes** — Optional export pathway.  
**Evidence:**  
- `plato-torch/rooms/base_room.py` contains method `export_to_ensign()`.  
- `plato-ensign/` provides `Exporter` class that loads room checkpoints and creates LoRA adapters.  
- Connection is explicit but not automatic; requires manual trigger or configuration to run export.  
**Data Flow:** Trained room state → LoRA weights file.  
**Integration Gap:** The export is not integrated into the automatic training pipeline. It's a separate, post-hoc step.

## 4. fleet-simulator → holodeck-rust
**Connection:** **No** — No cross-references found.  
**Evidence:**  
- `fleet-simulator/` is Python; `holodeck-rust/` is Rust. No interop bridges (e.g., PyO3, sockets) visible in code.  
- No configuration files reference each other.  
**Integration Gap:** **Major**. The fleet simulation does not currently incorporate the interactive, sentiment-driven NPC environment. This limits the diversity of social training scenarios.

## 5. fleet-simulator → plato-ensign
**Connection:** **Indirect** via plato-torch.  
**Evidence:**  
- `fleet-simulator` data flows into `plato-torch` rooms.  
- Those rooms can be exported by `plato-ensign`.  
- No direct `fleet-simulator` → `plato-ensign` link.  
**Integration Gap:** Simulation data is not directly exportable as an ensign; must pass through a full PLATO room training cycle.

**Integration Gap:** **Complete**. Holodeck interactions are not captured as training tiles or ensign adapters.
