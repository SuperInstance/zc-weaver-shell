# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 171  
**Phase:** 4 — Build & Test  
**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

## 1. Verified Connections (What's Connected)

### plato-torch ↔ fleet-simulator
- **Connection:** Direct integration via `fleet-simulator` as a training data source.
- **Evidence:** `plato-torch/src/rooms/deadband_room.py` imports `FleetSimulator` class.
- **Flow:** `DeadbandRoom` can instantiate a `FleetSimulator` to generate synthetic fleet interaction data for training.
- **Data Format:** Simulator produces `Tile` objects (state-action-reward sequences) that feed directly into room training loops.
- **Status:** **Connected and functional.** This is a core training pipeline.

### plato-torch ↔ plato-ensign
- **Connection:** Direct export pipeline.
- **Evidence:** `plato-torch/src/export/ensign_exporter.py` exists and is referenced in room configurations.
- **Flow:** After a room trains on accumulated tiles, `ensign_exporter` can package the room's trained adapter (LoRA weights) and metadata into a `.ensign` file.
- **Trigger:** Manual or automated export via room API endpoint `/export-ensign`.
- **Status:** **Connected and functional.** Export is an optional post-training step.

### fleet-simulator → holodeck-rust (Indirect)
- **Connection:** Data format alignment, not runtime integration.
- **Evidence:** Both systems use similar `Tile` structures (state, action, reward, metadata).
- **Flow:** Tiles generated by `fleet-simulator` could theoretically be replayed into `holodeck-rust` for sentiment-aware NPC training, but no direct API call exists.
- **Status:** **Potentially compatible via shared data schema.** No active runtime link.

## 2. Integration Gaps (What's Not Connected)

### holodeck-rust ↔ plato-torch
- **Gap:** No direct connection.
- **Detail:** `holodeck-rust` is a standalone MUD server with sentiment-aware NPCs. It does not import or call `plato-torch` rooms.
- **Missed Opportunity:** NPC interaction logs from holodeck could be rich training tiles for a dedicated "holodeck room" in plato-torch.
- **Status:** **Disconnected.**

### holodeck-rust ↔ plato-ensign
- **Gap:** No export path.
- **Detail:** `holodeck-rust` has no mechanism to export its trained NPC sentiment models as `.ensign` adapters.
- **Missed Opportunity:** NPC personality LoRAs cannot be shared or deployed elsewhere.
- **Status:** **Disconnected.**

### fleet-simulator ↔ plato-ensign (Direct)
- **Gap:** Simulator cannot export its own ensign.
- **Detail:** `fleet-simulator` generates training data but is not itself a trainable "room." Its internal policy could be exported, but no exporter is implemented.
- **Status:** **Disconnected.** Simulator is a data source, not a training endpoint.

### plato-ensign → holodeck-rust (Deployment)
- **Gap:** No loader for `.ensign` files into holodeck.
- **Detail:** `holodeck-rust` does not have an import mechanism to load a pre-trained LoRA adapter (e.g., to instantiate an NPC with a specific personality).
- **Missed Opportunity:** Cannot deploy fleet-trained specialist ensigns as NPCs in the MUD.
- **Status:** **Disconnected.**
