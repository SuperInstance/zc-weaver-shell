## 1. plato-torch

**Key files inspected:**
- `plato-torch/rooms/deadband_room.py`
- `plato-torch/rooms/__init__.py`
- `plato-torch/training/trainer.py`
- `plato-torch/config/room_configs.py`

**Connections found:**
- **plato-ensign**: Direct import. `deadband_room.py` imports `from plato_ensign import export_lora`. The `DeadbandRoom` class includes an `export_ensign()` method that calls `export_lora()` with room state.
- **fleet-simulator**: No direct import found in plato-torch source. However, `room_configs.py` contains a config entry `"fleet_sim_scenario": "basic_maneuver"`, suggesting planned or configuration-level integration.
- **holodeck-rust**: No Python import. No Rust bindings visible in the plato-torch codebase.

**Integration points:**
- Room training loops generate interaction tiles (JSON).
- Tiles are sent via HTTP POST to a `PLATO_RELAY_URL` (configurable).
- The `DeadbandRoom` can export a LoRA adapter via plato-ensign after training.

## 5. Integration Gaps (What's Not Connected)

| From | To | Gap Description |
|------|----|-----------------|
| plato-torch | holodeck-rust | No client to connect to holodeck WebSocket. NPC interactions not captured as tiles. |
| holodeck-rust | plato-torch | No tile emission. Interactions stay in memory or stdout. |
| fleet-simulator | plato-torch | Logs are not automatically formatted or sent as tiles. Manual pipeline required. |
| plato-torch | fleet-simulator | No simulation control from training rooms. |
| holodeck-rust | plato-ensign | No sentiment memory export to LoRA. |
| fleet-simulator | holodeck-rust | No shared agent state or environment. |

## 6. Immediate Integration Opportunities (P1 Safe Channels)

1. **plato-torch → holodeck-rust client**  
   - Build a simple async WebSocket client in Python (`websockets` library).  
   - Add `HolodeckConnector` class to plato-torch that joins a holodeck session and logs interactions as tiles.

2. **fleet-simulator log → plato-torch tile converter**  
   - Write a Rust binary or Python script that reads `output/scenario_*.json`, maps fields to tile schema, and POSTs to PLATO_RELAY_URL.

3. **holodeck-rust tile emission**  
   - Modify `holodeck-rust/src/server/mod.rs` to emit a JSON tile per interaction to a configurable HTTP endpoint (same relay as plato-torch).

4. **GhostInjector → holodeck** (from backlog)  
   - Ghost tiles (P0 knowledge) could be loaded into holodeck NPC memory as initial sentiment constraints.
