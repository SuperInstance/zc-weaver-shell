## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets (25 rooms), self-training rooms, tile processing.  
**Key Files:**
- `plato-torch/rooms/deadband_room.py` — Deadband Protocol training room.
- `plato-torch/tile_processor.py` — Processes interaction tiles into training data.
- `plato-torch/config/room_presets.yaml` — Defines 25 room configurations.

**Connections Found:**
- **plato-ensign:** Direct import in `plato-torch/export/lora_exporter.py`.  
  ```python
  from plato_ensign.adapters import LoRAAdapterBuilder
  ```
  Uses `LoRAAdapterBuilder` to export room experience as LoRA adapters.
- **fleet-simulator:** No direct import found in code. However, `plato-torch/config/room_presets.yaml` references `fleet-simulator` as a data source for room `simulator_training`.
- **holodeck-rust:** No direct import. No shared data structures observed.

## 2. fleet-simulator
**Location:** `fleet-simulator/`

- **holodeck-rust:** No direct import.

## 4. plato-ensign
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience (tiles) into LoRA adapters for specialist models.  
**Key Files:**
- `plato-ensign/src/adapters/lora_builder.rs` — Builds LoRA adapters from tile data.
- `plato-ensign/config/export_config.yaml` — Export settings.

**Connections Found:**
- **plato-torch:** Directly imported by plato-torch (as above). Also, `plato-ensign/src/loaders/tile_loader.rs` reads tile files from `plato-torch/output/tiles/`.
- **fleet-simulator:** No direct import.
- **holodeck-rust:** No direct import.

## Integration Gaps (What's Not Connected)

1. **holodeck-rust → plato-torch:** No automated tile pipeline. Tiles from holodeck interactions are not automatically sent to plato-torch for training. Manual export likely required.
2. **fleet-simulator → plato-torch:** Connection is config‑based only (`room_presets.yaml`). No code‑level integration ensuring data flow.
3. **GhostInjector:** Not wired into any system. Placeholder in holodeck‑rust only.
4. **DeadbandRoom → plato-relay:** `plato-relay` not found in inspected repos. Likely missing or not yet connected.
5. **End‑to‑end pipeline:** No single script or service orchestrates:  
   holodeck (interaction) → plato‑torch (training) → plato‑ensign (export) → deployment.
