## 1. plato-torch (Training Rooms)

**Key Files:**  
- `plato-torch/src/rooms/deadband_room.py` — Primary training room preset.  
- `plato-torch/src/rooms/__init__.py` — Exports 25 room presets.  
- `plato-torch/src/training/tile_processor.py` — Processes interaction tiles into training data.

**Connections Found:**  
- **To plato-ensign:** Direct export path. `deadband_room.py` includes `export_to_lora()` method that calls `plato_ensign.adapters.lora_exporter`. Verified via import: `from plato_ensign.adapters import lora_exporter`.  
- **To fleet-simulator:** No direct imports. However, `tile_processor.py` references `fleet_simulator.data_formats` in a commented-out section labeled "future integration". Not currently active.  
- **To holodeck-rust:** No Python imports. Configuration file `plato-torch/config/holodeck_bridge.yaml` exists but is unused in code.

**Integration Gap:**  
- Plato-torch rooms generate tiles but do not automatically send them to holodeck or fleet-simulator. Manual bridge required.

## 2. plato-ensign (LoRA Adapter Export)

**Key Files:**  
- `plato-ensign/src/adapters/lora_exporter.py` — Exports room experience to LoRA.  
- `plato-ensign/src/core/room_loader.py` — Loads room state from plato-torch.

**Connections Found:**  
- **From plato-torch:** Strong. `lora_exporter.py` is imported by `deadband_room.py`. Room state passed via `RoomExperience` dataclass.  
- **To fleet-simulator:** None. No shared data structures or imports.  
- **To holodeck-rust:** None.

**Integration Gap:**  
- Exported LoRA adapters are not automatically deployed to holodeck NPCs or fleet-simulator agents.

---

## 3. holodeck-rust (Sentiment-Aware MUD)

**Key Files:**  
- `holodeck-rust/src/npc/sentiment_engine.rs` — NPC sentiment system.  
- `holodeck-rust/src/world/room_manager.rs` — Manages MUD rooms.  
- `holodeck-rust/config/npc_config.toml` — NPC configuration.

**Connections Found:**  
- **To plato-torch:** Configuration bridge file exists (`holodeck_bridge.yaml`) but no Rust code references it.  
- **To plato-ensign:** None.  
- **To fleet-simulator:** None.

**Integration Gap:**  
- NPCs do not load trained LoRA adapters from plato-ensign. Sentiment engine uses static rules, not learned behavior.

## 4. fleet-simulator (Fleet Training Data)

**Key Files:**  
- `fleet-simulator/src/agents/deadband_agent.py` — Implements Deadband Protocol agents.  
- `fleet-simulator/src/data/tile_importer.py` — Imports tiles from external sources.

**Connections Found:**  
- **To plato-torch:** Commented-out import in `tile_importer.py`. Not active.  
- **To plato-ensign:** None.  
- **To holodeck-rust:** None.

**Integration Gap:**  
- Simulator agents do not receive real-time tiles from plato-torch rooms. Training data is static or synthetic.

---

## 5. Ghost Tiles & Mirror Play

**Location:** `fleet-knowledge/ghost_tiles/` — Contains JSON files of dead agents' lessons.  
**Current Usage:**  
- Read by `plato-torch/src/training/ghost_injector.py` during room initialization.  
- Injects P0 (negative space) knowledge into new training sessions.

**Integration Gap:**  
- Ghost tiles are not injected into holodeck NPCs or fleet-simulator agents. Only used in plato-torch rooms.

## 6. cudaclaw & flux-runtime

**cudaclaw:** GPU-resident agent runtime. No direct imports found in the four inspected repos.  
**flux-runtime:** Self-assembling bytecode VM. Referenced in `plato-torch/docs/ARCHITECTURE.md` as "future execution layer". Not integrated.
