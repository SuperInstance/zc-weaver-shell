## 1. plato-torch → fleet-simulator  
**Connection:** **Yes** — Direct import.  
**Evidence:**  
- `plato-torch/src/training/rooms/deadband_room.py` imports `from fleet_simulator.simulator import FleetSimulator`.  
- `deadband_room.py` line 47: `self.simulator = FleetSimulator(config_path="fleet_simulator/configs/basic_fleet.json")`.  
- The room uses the simulator to generate training episodes (P0/P1/P2 state transitions).  

**Data Flow:**  
1. Room calls `simulator.generate_episode(scenario="deadband")`.  
2. Simulator returns a list of `(state, action, reward)` tuples.  
3. Room converts these into training tiles (`.pt` files).  

**Integration Gap:**  
- The simulator config path is hardcoded. No environment variable or config injection.  
- Simulator output format is assumed; no validation adapter exists.

## 3. plato-torch → plato-ensign  
**Connection:** **Yes** — Ensign is an export module.  
**Evidence:**  
- `plato-torch/src/export/ensign_exporter.py` exists.  
- It imports `from plato_ensign.lora_adapter import LoRAAdapter`.  
- `deadband_room.py` line 89: `if export_ensign: self.export_to_ensign(tiles)`.  

**Data Flow:**  
1. Room accumulates tiles in `self.tile_buffer`.  
2. On export, calls `ensign_exporter.convert(tiles, output_path="adapters/")`.  
3. Exporter creates a LoRA adapter (`.safetensors` file) via `plato-ensign`.  

**Integration Gap:**  
- Export is optional (`export_ensign=False` by default).  
- No automatic trigger; manual or scheduled export needed.
