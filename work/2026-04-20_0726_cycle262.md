## 6. Integration Gaps (P1 Safe Channels)

1. **fleet-simulator → plato-torch**  
   - Gap: Simulation tiles are written to disk but not automatically ingested.
   - Safe Channel: Add a `SimulationTileLoader` in `plato-torch` that watches `fleet-simulator/output/tiles/`.

2. **holodeck-rust → plato-torch**  
   - Gap: NPC interactions are not captured as tiles.
   - Safe Channel: Extend `holodeck-rust` with a tile logging module that writes to shared storage.

3. **plato-torch → holodeck-rust**  
   - Gap: Trained ensigns not deployed to NPCs.
   - Safe Channel: Create `EnsignNPC` plugin for holodeck that loads LoRA adapters.

4. **Closed-loop training**  
   - Gap: No automatic cycle from simulation → training → deployment → simulation.
   - Safe Channel: Orchestrator service that runs simulator, trains room, exports ensign, updates holodeck.
