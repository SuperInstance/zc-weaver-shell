# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 134

**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current integration state between plato-torch, fleet-simulator, holodeck-rust, and plato-ensign.

## 1. plato-torch
**Role:** Training room server with 25 preset room configurations.  
**Connections:**
- **plato-ensign:** Direct integration via `export_room()` function. Rooms can export their trained LoRA adapters.
- **fleet-simulator:** No direct code connection found. plato-torch appears to be a standalone training environment.
- **holodeck-rust:** No direct integration. plato-torch rooms are text‑based; holodeck is a MUD server with sentiment‑aware NPCs.

**Integration Points:**
- `DeadbandRoom` preset exists (P0/P1/P2 protocol).
- Rooms accumulate interaction tiles, which can be exported.
- No visible hooks for simulator data ingestion or holodeck NPC state transfer.

## 2. fleet-simulator
**Role:** Generates training data through multi‑agent fleet simulations.  
**Connections:**
- **plato-torch:** No import or data‑piping code found. Simulator outputs are likely stored as standalone datasets.
- **plato-ensign:** No direct link. Simulator data could be used as pre‑training corpus, but no automation is evident.
- **holodeck-rust:** No connection found.

**Integration Points:**
- Simulator produces `.jsonl` logs of agent interactions.
- These logs could be converted to plato‑torch tiles, but no converter script is present.
- Potential channel: write a `sim2tile` adapter that formats simulator logs into room‑compatible tiles.

## 3. holodeck-rust
**Role:** Multi‑user dungeon server with sentiment‑aware NPCs and procedural quests.  
**Connections:**
- **plato-torch:** None. Holodeck is a live game server, not a training room.
- **fleet-simulator:** None.
- **plato-ensign:** None.

**Integration Points:**
- NPCs have sentiment states and memory.
- Quest outcomes generate narrative logs.
- Potential channel: extract NPC‑player dialogues as training tiles for plato‑torch rooms.
- Ghost tiles (dead‑agent lessons) could be injected into holodeck as NPC knowledge.

## 4. plato-ensign
**Role:** Exports room experience from plato‑torch into LoRA adapters.  
**Connections:**
- **plato-torch:** Direct — called from room context to export LoRA.
- **fleet-simulator:** None.
- **holodeck-rust:** None.

**Integration Points:**
- Works only with plato‑torch rooms.
- No support for ingesting simulator logs or holodeck narratives.
- Export is optional; rooms can run without creating an ensign.
