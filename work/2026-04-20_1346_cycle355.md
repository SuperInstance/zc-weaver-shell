## Integration Gaps Identified

### P0: What's NOT Connected (Negative Space)
1. **holodeck-rust ↔ plato-torch:** No bridge between Rust MUD and Python training rooms.
2. **fleet-simulator ↔ plato-torch:** No direct data pipeline for simulated training tiles.
3. **holodeck-rust ↔ plato-ensign:** No sentiment data export to LoRA adapters.
4. **fleet-simulator ↔ holodeck-rust:** No shared agent behavior models.

### P1: Safe Channels (Where Integration CAN Happen)
1. **Tile Format Standard:** All systems could adopt shared tile schema (JSON).
2. **REST/GRPC Bridges:** Python↔Rust interoperability layers.
3. **Shared Configuration:** Common YAML/TOML configs for room parameters.
4. **Message Queue:** Redis/RabbitMQ for cross-system communication.

### P2: Optimization Paths (Best Integration Points)
1. **Priority 1:** Create `tile_bridge.py` in plato-torch to ingest fleet-simulator scenarios.

3. **Priority 3:** Extend plato-ensign to accept holodeck sentiment data.
4. **Priority 4:** Standardize tile schema across all four systems.
