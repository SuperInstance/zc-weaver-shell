## 2. Identified Integration Gaps (What's Not Connected)

### **Gap 1: holodeck-rust → plato-torch tile pipeline**
- **Problem:** Holodeck NPC interactions are not harvested as training tiles.  
- **Impact:** Lost source of rich, sentiment-labeled dialogue data.  
- **Action:** Build `holodeck-tile-adapter` (Rust) to parse MUD logs → tile JSON.

### **Gap 2: plato-ensign → flux-runtime deployment**
- **Problem:** Ensigns export to LoRA, but flux-runtime (self-assembling bytecode VM) has no loader for them.  
- **Impact:** Cannot run ensigns in flux-runtime for lightweight deployment.  
- **Action:** Extend flux-runtime with `load_lora_adapter()` function.

### **Gap 3: DeadbandRoom → plato-relay**
- **Problem:** DeadbandRoom (plato-torch preset) lacks real-time relay to other fleet components.  
- **Impact:** Training rooms operate in isolation; no live fleet feedback.  
- **Action:** Integrate `plato-relay` WebSocket client into DeadbandRoom.

### **Gap 4: GhostInjector → holodeck-rust**
- **Problem:** Ghost tiles (dead agents' lessons) are not injected into holodeck as NPC knowledge.  
- **Impact:** Holodeck NPCs cannot learn from historical fleet failures.  
- **Action:** Create `GhostInjector` service that reads ghost tiles → updates holodeck NPC sentiment DB.

### **Gap 5: fleet-simulator → constraint-theory-core (unverified)**
- **Problem:** Connection assumed but not confirmed.  
- **Impact:** If missing, simulation lacks geometric snapping for spatial tasks.  
- **Action:** Check `fleet-simulator/Cargo.toml` for `constraint-theory-core` dependency.
