## Integration Gaps (P0/P1 Analysis)

### P0: Negative Space (What NOT to do)
- **Don't** create circular dependencies between plato-torch and holodeck-rust.
- **Don't** hardcode paths that break when repos move.
- **Don't** assume all components are running (file-based fallbacks needed).

### P1: Safe Channels (Where integration CAN happen)
1. **plato-torch ↔ fleet-simulator:** Already has file-based channel. Could add lightweight HTTP API for dynamic scenario requests.
2. **plato-torch ↔ holodeck-rust:** File channel exists. Could add WebSocket for live NPC updates.
3. **GhostInjector integration:** `plato-torch/rooms/ghost_injector.py` exists but not wired to holodeck. Safe channel: extend NPC sentiment engine with ghost tile data.

### P2: Optimization Opportunities
1. Replace file polling with event-driven updates.
2. Create shared data bus (Redis or in-memory) for fleet components.
3. Standardize agent state format across simulator, holodeck, and rooms.
