## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**  
- `plato-torch/src/rooms/deadband_room.py` — Deadband Protocol implementation.  
- `plato-torch/src/rooms/__init__.py` — Room registry.  
- `plato-torch/src/tile_server.py` — Receives and stores training tiles.  

**Connections:**  
- **To plato-ensign:** Exports room experience via `export_to_lora()` method in `deadband_room.py`.  
- **To fleet-simulator:** No direct import found. Potential integration via shared tile format.  
- **To holodeck-rust:** No direct import found.  

**Integration Points:**  
- `DeadbandRoom` inherits from `BaseRoom`.  
- `Tile` data structure (JSON) defined in `tile_server.py`.  
- Room configs in `plato-torch/configs/` reference `plato-ensign` export paths.

## 2. plato-ensign (LoRA Adapter Export)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `plato-ensign/src/exporter.py` — Main export logic.  
- `plato-ensign/src/lora_builder.py` — Builds LoRA from tile data.  

**Connections:**  
- **From plato-torch:** Called via `export_to_lora()` in `deadband_room.py`.  
- **To fleet-simulator:** No direct import found.  
- **To holodeck-rust:** No direct import found.  

**Integration Points:**  
- Imports `Tile` class from `plato-torch` (confirmed via `from plato_torch.tile_server import Tile`).  
- Outputs `.safetensors` files to `plato-ensign/outputs/`.

## 3. fleet-simulator (Fleet Simulation)
**Location:** `fleet-simulator/`  
**Purpose:** Generates training data via multi-agent simulation.  
**Key Files:**  
- `fleet-simulator/src/simulator/core.py` — Main simulation loop.  
- `fleet-simulator/src/agents/zeroclaw.py` — Zeroclaw agent implementation.  

**Connections:**  
- **To plato-torch:** No direct import found. Could produce tiles in compatible format.  
- **To plato-ensign:** No direct import found.  
- **To holodeck-rust:** No direct import found.  

**Integration Points:**  
- Uses `SimulationTile` class (similar structure to `Tile` but not identical).  
- Config in `fleet-simulator/config/sim_config.yaml` has `output_format: json`.

## Integration Gaps (P0 Negative Space)
1. **fleet-simulator → plato-torch:** Simulation tiles not ingested by tile server.  
2. **holodeck-rust → plato-torch:** NPC interactions not captured as training tiles.  
3. **GhostInjector:** Not found in codebase — likely planned but not built.  
4. **Shared data schema:** `Tile`, `SimulationTile`, `Event` are similar but not unified.

## Safe Channels (P1 Opportunities)
1. **Adapter pattern:** Write `SimulationTile.to_tile()` method to bridge fleet-simulator with plato-torch.  
2. **Webhook bridge:** Add POST endpoint to `tile_server.py` to accept holodeck-rust events.  
3. **GhostInjector module:** Create `holodeck-rust/src/ghost_injector.rs` to load P0 tiles from dead agents.  
4. **Configuration alignment:** Ensure `output_format` in fleet-simulator matches `Tile` JSON schema.
