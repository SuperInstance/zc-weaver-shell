# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 130

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

## 1. Verified Connections (What's Connected)

### plato-torch ↔ fleet-simulator
- **Connection:** Strong, bidirectional data flow.
- **Evidence:**  
  - `plato-torch/src/rooms/deadband_room.py` imports `fleet_simulator` and uses `FleetSimulator` class to generate training data.  
  - `fleet-simulator/src/lib.rs` exports a Python module via PyO3, called by plato-torch rooms.  
  - The simulator provides synthetic fleet interaction logs which become training tiles in plato-torch rooms.
- **Data Flow:**  
  `fleet-simulator` → generates JSON logs → `DeadbandRoom` ingests logs → creates interaction tiles → tiles stored in room’s tile database.

### plato-torch ↔ plato-ensign
- **Connection:** Direct export pipeline.
- **Evidence:**  
  - `plato-ensign/src/main.py` reads from `plato-torch` room directories (e.g., `rooms/deadband_room/tiles/`).  
  - `plato-torch/src/rooms/base_room.py` has `export_tiles()` method called by ensign.  
  - Ensign converts tiles into LoRA adapters via `peft` and `transformers`.
- **Data Flow:**  
  `plato-torch` room tiles → `plato-ensign` tile loader → tokenized datasets → LoRA training → adapter saved as `.safetensors`.

### holodeck-rust ↔ (Indirect via plato-torch)
- **Connection:** No direct link to the other three.  
- **Evidence:**  
  - `holodeck-rust/` is a standalone MUD server with sentiment-aware NPCs.  
  - No imports of `fleet-simulator`, `plato-torch`, or `plato-ensign` in its `Cargo.toml` or source.  
  - However, `plato-torch/README.md` mentions “holodeck” as a potential tile source, but no current integration exists.

## 2. Integration Gaps (What's Not Connected)

### Gap 1: holodeck-rust → plato-torch
- **Missing:** No pipeline to send holodeck NPC interactions to plato-torch as training tiles.
- **Potential Integration Point:**  
  Add a `HolodeckRoom` in plato-torch that subscribes to holodeck’s WebSocket stream (port `5582`), formats interactions as tiles, and stores them.

### Gap 2: fleet-simulator → holodeck-rust
- **Missing:** No simulation-driven NPC behavior.
- **Potential Integration Point:**  
  Use `fleet-simulator` to generate agent behavior logs, feed them to holodeck’s NPC engine to influence dialogue/actions.

### Gap 3: plato-ensign → holodeck-rust
- **Missing:** Trained ensign adapters are not loaded into holodeck NPCs.
- **Potential Integration Point:**  
  Load `.safetensors` adapters into holodeck’s Rust NLP model to specialize NPC personalities based on room training.

### Gap 4: Deadband Protocol Enforcement Across Stack
- **Missing:** P0/P1/P2 checks are only in `plato-torch` rooms. Not applied to `fleet-simulator` or `holodeck-rust`.
- **Potential Integration Point:**  
  Add deadband validation layer in `fleet-simulator` (reject invalid actions) and holodeck (constrain NPC decisions).
