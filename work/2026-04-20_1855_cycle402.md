## 7. Integration Gaps (P1 Safe Channels)

1. **plato‑torch ← fleet‑simulator**  
   - Gap: No ingestion path for simulation logs as training tiles.  
   - Safe channel: Write a log‑to‑tile converter that outputs JSON matching `tile-schema.md`.

2. **plato‑torch ← holodeck‑rust**  
   - Gap: No tile stream from MUD interactions.  
   - Safe channel: Add a tile‑export endpoint to holodeck‑rust that POSTs to plato‑torch’s tile API.

3. **plato‑ensign → plato‑torch**  
   - Gap: Manual file copy of checkpoints.  
   - Safe channel: Extend plato‑ensign to fetch checkpoints via plato‑torch’s HTTP API (if exposed).

4. **Ghost tiles / Deadband Protocol**  
   - Gap: No component currently consumes ghost tiles (P0 negative‑space knowledge).  
   - Safe channel: Add a P0 tile filter in plato‑torch’s tile processor that loads ghost tiles from a known directory.
