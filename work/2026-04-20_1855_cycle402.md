**Cycle:** 402

**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch → fleet-simulator
**Connection:** **Yes** — Direct import and usage.  
**Evidence:**  
- `plato-torch/rooms/deadband_room.py` imports `from fleet_simulator.simulator import FleetSimulator`  
- `deadband_room.py` line 47: `self.simulator = FleetSimulator()`  
- Simulator is used to generate training episodes (line 89: `episode = self.simulator.run_episode()`)

**Integration Gap:**  
- Plato-torch only uses the simulator for episode generation.  
- No bidirectional feedback loop observed — simulator doesn't receive training outcomes.

## 2. plato-torch → holodeck-rust
**Connection:** **No** — No imports or direct references found.  
**Evidence:**  
- Searched `plato-torch/` for "holodeck" — zero matches.  
- No Rust bindings or API calls in Python code.  
- `holodeck-rust` is a separate MUD server (TCP port 8080).

**Integration Gap:**  
- Plato-torch rooms could send NPC training data to holodeck, but currently don't.  
- Potential channel: sentiment logs from holodeck could feed plato-torch training.

## 3. plato-torch → plato-ensign
**Connection:** **Yes** — Optional export pathway.  
**Evidence:**  
- `plato-torch/export/ensign_exporter.py` exists.  
- `deadband_room.py` line 120: `if self.config.get('export_ensign'): self.export_to_ensign()`  
- Exporter converts room experience tiles to LoRA adapters.

**Integration Gap:**  
- Export is config‑driven, not automatic.  
- No observed feedback from ensign back to plato-torch.

## 4. fleet-simulator → holodeck-rust
**Connection:** **No** — No references in either direction.  
**Evidence:**  
- `fleet-simulator/src/` contains no "holodeck" mentions.  
- `holodeck-rust/src/` contains no "fleet" or "simulator" mentions.  
- Different languages (Rust vs. Python) and no inter‑process communication observed.

**Integration Gap:**  
- Simulator could populate holodeck with simulated agents as NPCs.  
- Holodeck sentiment data could influence simulator agent behavior.

## 5. fleet-simulator → plato-ensign
**Connection:** **No** — No direct link.  
**Evidence:**  
- `fleet-simulator` does not import or call `plato-ensign`.  
- Simulator outputs episodes to plato‑torch only.

**Integration Gap:**  
- Simulator episodes could be exported directly as ensign adapters (bypassing plato‑torch).  
- Currently only indirect via plato‑torch.

## 6. holodeck-rust → plato-ensign
**Connection:** **No** — No observed linkage.  
**Evidence:**

- No exporter or adapter generation code present.

**Integration Gap:**  
- NPC sentiment patterns could be captured as ensign LoRAs.  
- Missing: sentiment‑to‑adapter pipeline.
