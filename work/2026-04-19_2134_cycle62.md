## 2. Identified Integration Gaps (P1 Safe Channels)

### **Gap 1: holodeck‑rust → plato‑torch tile ingestion**
- **What’s missing:** No defined API or data format for sending NPC interaction logs from `holodeck‑rust` to a `plato‑torch` room as training tiles.  
- **P1 Safe channel:** Define a simple JSON schema for “holodeck interaction tiles” and a REST endpoint on `plato‑torch` to accept them.  
- **Action:** Write a spec for `holodeck‑to‑plato` adapter.

### **Gap 2: fleet‑simulator → plato‑torch direct pipeline**
- **What’s missing:** Currently implied data flow; no automated ingestion of simulation‑generated scenarios into specific rooms.  
- **P1 Safe channel:** Create a `SimulationTileExporter` module in `fleet‑simulator` that formats output to match `plato‑torch` tile schema.  
- **Action:** Draft a minimal exporter script.

### **Gap 3: GhostInjector (ghost tiles) → holodeck‑rust**
- **What’s missing:** Ghost tiles (dead agents’ P0 lessons) are stored but not injected into `holodeck‑rust` NPC sentiment models.  
- **P1 Safe channel:** Add a `GhostLoader` to `holodeck‑rust` that reads from fleet‑knowledge/ghost‑tiles/ and influences NPC behavior.  
- **Action:** Design a loading interface.

### **Gap 4: DeadbandRoom → plato‑relay**
- **What’s missing:** `DeadbandRoom` (a `plato‑torch` preset) likely needs to broadcast its state or logs to the fleet’s messaging layer (`plato‑relay`).  
- **P1 Safe channel:** Implement a `RelayHook` in the DeadbandRoom class that publishes key events.  
- **Action:** Examine `DeadbandRoom` source to identify hook points.
