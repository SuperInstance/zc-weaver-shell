## 1. plato-torch

**Key files inspected:**
- `plato-torch/rooms/deadband_room.py`

- `plato-torch/plato_relay.py`

**Connections found:**
- **To plato-ensign:** Direct import. `deadband_room.py` imports `from plato_ensign import EnsignExporter`. The room can export its trained adapter via `export_ensign()` method.
- **To fleet-simulator:** No direct import found in code. README mentions "fleet simulation for training data" but no Python imports or shared modules visible.

- Export path: Room → EnsignExporter → LoRA adapter file.

## 2. fleet-simulator

**Key files inspected:**
- `fleet-simulator/src/main.rs`
- `fleet-simulator/Cargo.toml`
- `fleet-simulator/README.md`

**Connections found:**
- **To plato-torch:** None. Rust project, no Python dependencies. README states it "generates training data for fleet agents" but no explicit data exchange mechanism is implemented.
- **To holodeck-rust:** Shared dependency found: both use `serde` for serialization and `tokio` for async runtime. No direct crate import.
- **To plato-ensign:** No connection.

**Integration gaps:**
- Simulator outputs JSON logs (`logs/`). Could be consumed by plato-torch as training tiles, but no ingestion pipeline exists.
- No configuration linking to plato-torch rooms or ensign export.
