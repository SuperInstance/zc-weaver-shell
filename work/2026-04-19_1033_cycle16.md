- **Connection:** Training data source

- **Evidence:** FLEET-CONTEXT.md lists both as key technologies with complementary purposes  
- **Status:** Connected (data flow)

### holodeck-rust → plato-torch
- **Connection:** Tile generation source  
- **Mechanism:** `holodeck-rust` MUD server with sentiment-aware NPCs → produces interaction tiles → consumed by `plato-torch` training rooms  
- **Evidence:** Both listed in architecture section describing tile-based training  
- **Status:** Connected (tile pipeline)

### cudaclaw → plato-torch
- **Connection:** Runtime environment  
- **Mechanism:** `cudaclaw` GPU-resident agent runtime could host `plato-torch` inference/training  
- **Evidence:** Both are GPU-focused technologies in fleet stack

- **Priority:** P2 (optimization — if compatible)
