## 1. plato-torch → fleet-simulator
**Connection:** Strong, direct integration via `SimulationRoom` preset.
- **plato-torch/rooms/simulation_room.py** imports `from fleet_simulator import FleetSimulator`.
- `SimulationRoom` class initializes a `FleetSimulator` instance and calls `simulator.step()` during training cycles.
- Shared data: `FleetSimulator` returns a `state_dict` (JSON) containing agent positions, velocities, and collision flags. This state is logged as training tiles.
- **Gap:** No observed bidirectional flow—simulator does not read back from plato-torch. Plato-torch drives the simulation as a data source.

## 2. plato-torch → holodeck-rust
**Connection:** Indirect, via configuration only.
- **plato-torch/config/holodeck_config.yaml** defines MUD server connection parameters (`host: localhost`, `port: 8080`).
- **plato-torch/rooms/holodeck_room.py** contains a `HolodeckRoom` class that attempts to connect to a running holodeck-rust server via WebSocket.
- **Gap:** The actual WebSocket client import (`websockets`) is commented out in the current file. No active data exchange observed in the code. Integration is stubbed.

## 3. plato-torch → plato-ensign
**Connection:** Export pipeline, one-way.
- **plato-torch/exporter/ensign_exporter.py** imports `from plato_ensign import LoRAAdapter`.
- After a room accumulates enough tiles, `ensign_exporter` can package them into a `.safetensors` file using `plato-ensign`'s `create_adapter()` function.
- Shared data: Training tiles (JSONL format) are passed to `plato-ensign` for LoRA fine-tuning.
- **Gap:** No observed real-time integration during training. Export is a post-processing step.

## 4. fleet-simulator → holodeck-rust
**Connection:** None observed.
- **fleet-simulator/src/** contains no references to `holodeck`, `mud`, or `websocket`.
- **holodeck-rust/src/** contains no references to `fleet` or `simulator`.
- **Gap:** These components operate in separate domains (simulation vs. interactive narrative). No shared data structures or APIs.

## 5. fleet-simulator → plato-ensign
**Connection:** None observed.
- No imports or configuration linking these repositories.
- **Gap:** `plato-ensign` is designed to export room experience, not raw simulator state.

## 6. holodeck-rust → plato-ensign
**Connection:** None observed.
- **holodeck-rust** outputs sentiment logs and NPC interaction transcripts, but no integration with `plato-ensign`'s LoRA adapter creation.
- **Gap:** Potential untapped training data source. NPC dialogues could be turned into training tiles, but no current pipeline.
