## 1. plato-torch

**Location:** `plato-torch/`  
**Purpose:** Training room presets, self-training rooms, tile processing.

**Connections Found:**

- **To plato-ensign:**  
  - `plato-torch/src/ensign_exporter.py` imports `plato_ensign` and calls `export_room_to_lora()`.  
  - Exports room experience (tiles) as LoRA adapters.  
  - **Status:** Connected via explicit Python import.

- **To fleet-simulator:**  
  - `plato-torch/presets/DeadbandRoom.py` imports `fleet_simulator` and uses `FleetSimulation` class.  
  - Simulated fleet data is used as training input.  
  - **Status:** Connected via Python import.

- **To holodeck-rust:**  
  - No direct imports found in Python files.  
  - `plato-torch/config/holodeck_bridge.yaml` contains endpoint configuration (`http://localhost:8080/api/npc`).  
  - **Status:** Connected via configuration, not code import.

**Integration Gaps:**  
- No direct code coupling to holodeck-rust; relies on REST API.  
- No import of `constraint-theory-core`, `cudaclaw`, or `flux-runtime`.
