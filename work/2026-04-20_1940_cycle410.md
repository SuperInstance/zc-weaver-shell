## Repository Structure (Verified)
Based on file inspection:

### plato-torch
- **Purpose**: Training room presets, self-training rooms.
- **Key files**: `DeadbandRoom.py`, `GhostInjector.py`, `plato_relay.py`.
- **Dependencies**: `torch`, `transformers`, `datasets`, `peft` (for LoRA).

### plato-ensign
- **Purpose**: Exports room experience into LoRA adapters.
- **Key files**: `ensign_exporter.py`, `lora_adapter.py`.
- **Dependencies**: `peft`, `torch`, `safetensors`.
- **Imports from plato-torch**: `from plato_torch import export_room` (verified in code).

### holodeck-rust
- **Purpose**: MUD server with sentiment-aware NPCs.
- **Key files**: `src/server.rs`, `src/npc/sentiment.rs`.
- **Dependencies**: `tokio`, `serde`, `websocket`.
- **No direct imports** to/from plato-torch or plato-ensign in Rust source.

### fleet-simulator
- **Purpose**: Fleet simulation for training data.
- **Key files**: `simulator/core.py`, `agents/deadband_agent.py`.
- **Dependencies**: `numpy`, `pandas`, `torch`.
- **Imports from plato-torch**: `from plato_torch import DeadbandRoom` (verified).

## Verified Connections

### 1. plato-torch → plato-ensign (Strong Connection)
- **plato-ensign/ensign_exporter.py** imports `export_room` from `plato_torch`.
- Data flow: Room training logs → LoRA adapters.
- **Integration status**: Connected via Python imports and shared data structures (training logs, model weights).

### 2. plato-torch → fleet-simulator (Moderate Connection)
- **fleet-simulator/agents/deadband_agent.py** imports `DeadbandRoom` from `plato_torch`.
- Used for simulating agent behavior in training environments.
- **Integration status**: Connected via Python import; uses room as simulation environment.

### 3. fleet-simulator → plato-ensign (Weak Connection)
- No direct imports found.
- Potential indirect connection: simulation data could be fed into plato-torch rooms, then exported via plato-ensign.
- **Integration status**: Not directly connected; data flow possible but not implemented.

### 4. holodeck-rust → Any Python Component (No Direct Connection)
- Rust codebase does not import Python modules.
- No Python bindings (`pyo3`) found in source.
- **Integration status**: Isolated; integration would require API bridge (e.g., REST, WebSocket).

- **Risk**: Extra step required; possible data loss or transformation errors.
- **P0 Action**: Do not assume direct simulator-to-ensign pipeline.
