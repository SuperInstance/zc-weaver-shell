# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 284  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets, self-training rooms, tile processing.

### Verified Connections:
- **plato-ensign:** Direct import in `plato-torch/src/export.rs`. Exports room experience (tiles) to LoRA adapters via `ensign::export_lora()`.
- **Deadband Protocol:** Implemented in `plato-torch/src/deadband_room.rs`. Uses `DeadbandRoom` preset for P0/P1/P2 training.
- **Tile System:** `plato-torch/src/tile.rs` defines `TrainingTile` struct used across fleet. Serialized to JSON for storage/transmission.
- **Configuration:** `plato-torch/config/` references `fleet-simulator` for data generation parameters.

### Integration Gaps:
- No direct import of `holodeck-rust`. No sentiment-aware NPC hooks in current room presets.
- No live connection to `plato-relay` (message bus). Rooms operate in isolation.
- `GhostInjector` module exists but not wired to external ghost tile sources.

## 2. fleet-simulator
**Location:** `fleet-simulator/`  
**Purpose:** Generate synthetic training data for fleet agents.

### Verified Connections:
- **plato-torch:** Provides `SimulationDataSource` in `fleet-simulator/src/data_source.rs`. Used by `DeadbandRoom` for generating P0/P1/P2 scenarios.
- **Shared Types:** Uses `TrainingTile` struct from plato-torch via dependency (`plato-torch = { path = "../plato-torch" }` in Cargo.toml).
- **Constraint Theory:** `fleet-simulator/src/constraint_sim.rs` imports `constraint_theory_core` for geometric scenario generation.

### Integration Gaps:
- No connection to `holodeck-rust`. Simulator generates static scenarios, not interactive MUD sessions.
- No direct export to `plato-ensign`. Data flows through plato-torch rooms first.
- No integration with `cudaclaw` or `flux-runtime` for GPU-accelerated simulation.

## 3. holodeck-rust
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs, interactive training environment.

### Verified Connections:
- **Shared Protocol:** Uses same `TrainingTile` JSON format for NPC interaction logs (see `holodeck-rust/src/npc/logging.rs`).
- **Sentiment Engine:** Custom `sentiment.rs` module, not yet integrated with plato-torch's emotion tracking.

### Integration Gaps:
- **No direct link to plato-torch.** NPC interactions don't feed into training rooms automatically.
- **No plato-ensign export.** NPC sentiment data remains in MUD logs, not converted to LoRA adapters.
- **No fleet-simulator integration.** NPC scenarios are hand-authored, not procedurally generated.
- **Ghost tiles not consumed.** `GhostInjector` exists in plato-torch but not connected to holodeck's NPC memory.

## 4. plato-ensign
**Location:** `plato-ensign/`  
**Purpose:** Export room experience (tiles) into LoRA adapters for specialist ensign models.

### Verified Connections:
- **plato-torch:** Direct dependency. Called from `plato-torch/src/export.rs` via `ensign::export_lora(tiles, config)`.
- **LoRA Pipeline:** Uses `peft` and `transformers` libraries. Configurable adapter rank/dropout in `plato-ensign/src/config.rs`.

### Integration Gaps:
- **Only consumes plato-torch tiles.** No direct import from holodeck-rust or fleet-simulator.
- **No GPU runtime integration.** Runs on CPU unless manually configured with CUDA.
- **No validation feedback loop.** Exported ensigns aren't automatically tested in rooms.

## Integration Map Summary

```
plato-torch ←→ plato-ensign          (STRONG: direct export)
plato-torch ←→ fleet-simulator       (MODERATE: data source)
fleet-simulator ←→ constraint-theory (WEAK: geometric sim only)
holodeck-rust                        (ISOLATED: shared format only)
```

## Missing Critical Connections:
1. **holodeck → plato-torch:** NPC sentiment data not feeding training rooms.
2. **plato-torch → plato-relay:** Rooms operate offline, no inter-room communication.
3. **GhostInjector → holodeck:** Dead agents' lessons not injected into NPC memory.
4. **cudaclaw/flux-runtime:** GPU acceleration not integrated into simulation/training pipeline.
