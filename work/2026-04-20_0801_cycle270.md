## 1. plato-torch → fleet-simulator → holodeck-rust → plato-ensign: Verified Connections

### **plato-torch → fleet-simulator**
- **Connection:** Strong, direct integration via `fleet_simulator` Python module.
- **Evidence:**  
  - `plato-torch/rooms/deadband_room.py` imports `from fleet_simulator import FleetSimulator`.  
  - `plato-torch/rooms/deadband_room.py` line 47: `self.simulator = FleetSimulator(config_path="fleet_simulator/configs/deadband_scenario.json")`.  
  - `plato-torch/tests/test_deadband_room.py` uses `FleetSimulator` to generate training data.
- **Data Flow:**  
  `DeadbandRoom` calls `simulator.generate_scenario()` → returns JSON with agent states, decisions, outcomes → converted to training tiles.

### **fleet-simulator → holodeck-rust**
- **Connection:** Indirect, via shared scenario definitions.
- **Evidence:**  
  - `fleet-simulator/configs/deadband_scenario.json` defines agents, environments, and P0/P1/P2 decision trees.  
  - `holodeck-rust/src/scenarios/deadband_scenario.rs` contains a Rust struct `DeadbandScenario` with identical fields (`agents`, `environment`, `decision_trees`).  
  - No direct import or API call between the two codebases found.
- **Data Flow:**  
  Manual or tool-assisted translation of JSON scenario → Rust struct. Used for sentiment-aware NPC behavior in holodeck MUD.

### **holodeck-rust → plato-ensign**
- **Connection:** Weak, potential via tile export.
- **Evidence:**  
  - `holodeck-rust/src/npc/sentiment_engine.rs` logs NPC interactions to `interaction_log.json`.  
  - `plato-ensign/src/tile_loader.rs` can load JSON logs from `data/interaction_logs/`.  
  - No direct reference found in code; integration appears planned but not wired.
- **Data Flow:**  
  NPC interaction logs → JSON files → manually placed in `plato-ensign/data/` → loaded as training tiles for LoRA adapter creation.

### **plato-torch → plato-ensign**
- **Connection:** Direct, via tile export/import.
- **Evidence:**  
  - `plato-torch/rooms/base_room.py` has method `export_tiles()` that writes to `tiles/` directory.  
  - `plato-ensign/src/tile_loader.rs` reads from `tiles/` directory (supports `.json`, `.jsonl`, `.pt`).  
  - `plato-ensign/README.md` states: "Load tiles from plato-torch rooms and train LoRA adapters."
- **Data Flow:**  
  `DeadbandRoom.export_tiles()` → `tiles/deadband_*.jsonl` → `plato-ensign --tiles-dir tiles/` → trains `deadband_lora.safetensors`.

## 2. Integration Gaps (What’s Not Connected)

### **Gap 1: holodeck-rust → plato-torch (live)**
- **Issue:** No live connection between holodeck MUD interactions and plato-torch room training.
- **Evidence:**  
  `holodeck-rust` writes logs to disk; `plato-torch` reads from `fleet_simulator`, not holodeck logs.
- **Potential Integration:**  
  Add a `HolodeckLogLoader` in `plato-torch/data_loaders/` to stream NPC interactions into training tiles.

### **Gap 2: plato-ensign → holodeck-rust (feedback loop)**
- **Issue:** Trained LoRA adapters from `plato-ensign` are not deployed back to `holodeck-rust` NPCs.
- **Evidence:**  
  `holodeck-rust` uses hard-coded sentiment weights (`src/npc/weights.json`). No dynamic adapter loading.
- **Potential Integration:**  
  Add `AdapterLoader` in `holodeck-rust` to load `*.safetensors` and update NPC models at runtime.

### **Gap 3: fleet-simulator → plato-ensign (direct)**
- **Issue:** `fleet_simulator` can generate training data, but no direct export to `plato-ensign` tile format.
- **Evidence:**  
  `fleet_simulator` outputs raw JSON scenario data; requires `plato-torch` room to convert to tiles.
- **Potential Integration:**  
  Add `export_tiles()` method to `FleetSimulator` class to generate `.jsonl` tiles directly.

### **Gap 4: GhostInjector not wired**
- **Issue:** `GhostInjector` (P0 knowledge from dead agents) exists in `plato-torch/tools/ghost_injector.py` but is not connected to any room or simulator.
- **Evidence:**  
  Tool is standalone; not imported in `deadband_room.py` or `fleet_simulator`.
- **Potential Integration:**  
  Integrate into `DeadbandRoom` as a preprocessor to inject ghost tiles before training.
