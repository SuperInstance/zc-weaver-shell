# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 273  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**
- `presets/deadband_room.py` — Deadband Protocol training room.
- `presets/__init__.py` — Exports presets.
- `trainer/room_trainer.py` — Main training loop.
- `utils/tile_loader.py` — Loads interaction tiles.

**Connections:**
- **To plato-ensign:** `plato-torch/exporter/lora_exporter.py` imports `plato_ensign` and calls `export_to_lora()`.
- **To fleet-simulator:** No direct import found. Potential integration via shared tile format.
- **To holodeck-rust:** No direct import. Room presets could theoretically send NPC training data.

**Integration Points:**
- `tile_loader.py` expects `.tile` files (serialized interactions).
- `room_trainer.py` logs to `training_logs/` — could be consumed by other systems.

## 2. fleet-simulator (Fleet Simulation)
**Location:** `fleet-simulator/`  
**Purpose:** Generate training data via simulated fleet interactions.  
**Key Files:**
- `src/simulation_engine.rs` — Core simulator.
- `src/agents/mod.rs` — Simulated agent behaviors.
- `src/output/tile_writer.rs` — Writes `.tile` files.

**Connections:**
- **To plato-torch:** `tile_writer.rs` outputs `.tile` files in format readable by `plato-torch/tile_loader.py`.
- **To holodeck-rust:** No direct import found.
- **To plato-ensign:** No direct import.

**Integration Points:**
- `.tile` files written to `output/tiles/` — can be loaded into plato-torch rooms.
- Simulation configs (`config/`) could be extended to include room parameters.

## 3. holodeck-rust (Sentiment-Aware MUD)
**Location:** `holodeck-rust/`  
**Purpose:** Multi‑User Dungeon server with NPCs that learn from sentiment.  
**Key Files:**
- `src/npc/sentiment_engine.rs` — NPC sentiment modeling.
- `src/world/tile_map.rs` — World tile management (note: different tile concept).
- `src/server.rs` — WebSocket server.

**Connections:**
- **To plato-torch:** No direct import. NPC sentiment data could be exported as training tiles.
- **To fleet-simulator:** No direct import.
- **To plato-ensign:** No direct import.

**Integration Points:**
- `sentiment_engine.rs` logs NPC interactions — could be formatted as `.tile` files.
- WebSocket server (`:8080`) could be connected to a training room for live data.

## 4. plato-ensign (LoRA Adapter Export)
**Location:** `plato-ensign/`  
**Purpose:** Export room experience into LoRA adapters.  
**Key Files:**
- `src/lib.rs` — Provides `export_to_lora()` function.
- `src/adapters/lora_builder.rs` — Builds LoRA from training logs.

**Connections:**
- **From plato-torch:** `lora_exporter.py` calls `plato_ensign.export_to_lora()`.
- **To fleet-simulator:** No direct import.
- **To holodeck-rust:** No direct import.

**Integration Points:**
- Expects training logs from `plato-torch/training_logs/`.
- Outputs LoRA weights (`.safetensors`) — usable by inference systems.

## 6. Integration Gaps (What’s Not Connected)

1. **holodeck‑rust ↔ plato‑torch**  
   - NPC sentiment interactions not captured as training tiles.
   - No pipeline to feed live MUD data into a training room.

2. **fleet‑simulator ↔ holodeck‑rust**  
   - Simulated agents cannot interact with MUD NPCs.
   - No shared agent‑behavior definitions.

3. **plato‑ensign ↔ fleet‑simulator**  
   - Cannot export LoRA directly from simulation data (must go through a room).

4. **GhostInjector** (mentioned in backlog)  
   - Not found in any repo. Presumed tool to inject ghost tiles into training streams.

## 7. Recommended Integration Steps (P1 Safe Channels)

1. **Bridge holodeck‑rust to plato‑torch:**  
   - Add a `tile_exporter.rs` in holodeck that writes NPC interactions as `.tile` files.
   - Modify `plato-torch/tile_loader.py` to watch a directory for new tiles.

2. **Connect fleet‑simulator to holodeck‑rust:**  
   - Extend `fleet-simulator/agents/mod.rs` to communicate via WebSocket to holodeck.
   - Use shared protocol for agent actions.

3. **Direct simulation‑to‑ensign path:**  
   - Create a lightweight “simulation room” preset in plato‑torch that loads simulator tiles and triggers ensign export.

4. **Implement GhostInjector:**  
   - Tool to read `memory/` files and format them as P0 ghost tiles.
   - Inject into tile stream during room training.
