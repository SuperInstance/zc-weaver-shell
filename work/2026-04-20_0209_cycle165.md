# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 165  
**Phase:** 4 — Build & Test  
**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

---

## 1. **plato-torch** (Training Room Engine)
**Role:** Provides 25 preset training rooms (including `DeadbandRoom`). Handles tile ingestion, agent training loops, and room state management.  
**Key Files:** `rooms/deadband_room.py`, `core/tile_manager.py`, `core/room_factory.py`

### Current Connections:
- **→ plato-ensign:** Direct export path via `export_ensign()` method in `DeadbandRoom`. Rooms can serialize their trained adapter (LoRA) to disk.
- **→ fleet-simulator:** No direct code dependency found. However, both share a common `Tile` data structure (`fleet-simulator/src/tile.rs` vs `plato-torch/core/tile.py`). This suggests a *conceptual* link for training data flow.
- **→ holodeck-rust:** No direct integration found in code. Plato-torch rooms are Python-based; holodeck is a Rust MUD server.

### Integration Gaps:
- No live data pipeline from `fleet-simulator` (simulation tiles) into `plato-torch` rooms.
- No mechanism for `holodeck-rust` NPC interactions to generate tiles for plato-torch training.
- `DeadbandRoom` is isolated—does not broadcast or relay tiles to other services.

---

## 2. **fleet-simulator** (Multi-Agent Simulation)
**Role:** Simulates fleet agent interactions to generate training tiles. Written in Rust.  
**Key Files:** `src/tile.rs`, `src/simulation/engine.rs`, `src/agents/mod.rs`

### Current Connections:
- **→ plato-torch:** None. Tiles are generated but not sent to a room. The `Tile` struct in Rust mirrors the Python `Tile` class, but there is no exporter or network bridge.
- **→ holodeck-rust:** No direct link. Both are Rust projects but separate crates.
- **→ plato-ensign:** No connection. The simulator does not produce LoRA adapters.

### Integration Gaps:
- Tiles stay in-memory or log files. No automatic ingestion into a training room.
- Simulated agents do not interact with live holodeck NPCs.
- No feedback loop where trained ensigns improve simulator agents.

---

## 3. **holodeck-rust** (Sentiment-Aware MUD Server)
**Role:** Text-based multiplayer dungeon with NPCs that react to player sentiment.  
**Key Files:** `src/npc/sentiment.rs`, `src/world/room.rs`, `src/server/mod.rs`

### Current Connections:
- **→ plato-torch:** None. NPC dialogues and player interactions are not captured as tiles.
- **→ fleet-simulator:** None. The MUD world is static, not dynamically simulated.
- **→ plato-ensign:** None. No adapter export from NPC behavior.

### Integration Gaps:
- Rich NPC interactions are lost for training—no tile generation.
- No way to inject “ghost tiles” (P0 knowledge) into NPC dialogue trees.
- Standalone service; not part of the larger training pipeline.

---

## 4. **plato-ensign** (LoRA Adapter Exporter)
**Role:** Exports room training experience as lightweight LoRA adapters.  
**Key Files:** `ensign/exporter.py`, `ensign/lora_builder.py`

### Current Connections:
- **← plato-torch:** Called from `DeadbandRoom.export_ensign()`. Direct dependency.
- **→ fleet-simulator:** None. Ensigns are not used to guide simulations.
- **→ holodeck-rust:** None. NPCs do not load or use ensigns.

### Integration Gaps:
- Export is manual—no automatic triggering based on training milestones.
- Ensigns are saved to disk but not deployed to other services.
- No versioning or registry to track which ensigns are active in the fleet.

---

## 5. **Summary of Integration Points**

| From → To          | Connection Status | Data Flow |
|-------------------|-------------------|-----------|
| plato-torch → plato-ensign | ✅ Direct | Room → LoRA adapter |
| fleet-simulator → plato-torch | ❌ Missing | Sim tiles → training room |
| holodeck-rust → plato-torch | ❌ Missing | NPC interactions → tiles |
| plato-ensign → holodeck-rust | ❌ Missing | Adapter → NPC behavior |
