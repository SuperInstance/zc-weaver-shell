## 1. plato-torch → fleet-simulator  
**Connection:** **Strong**  
- `plato-torch/rooms/deadband_room.py` imports `fleet_simulator` directly:  
  ```python
  from fleet_simulator import FleetSimulator, FleetState
  ```
- Uses `FleetSimulator` to generate synthetic training data (agent interactions, state transitions).  
- `DeadbandRoom` config includes `simulator_config` path pointing to `fleet-simulator/configs/`.  
- Shared data structure: `FleetState` (serialized as JSON) passed between modules.  

**Integration Gap:** None observed. This is a working, direct dependency.

## 2. plato-torch → holodeck-rust  
**Connection:** **Weak / Indirect**  
- No direct Rust imports in Python code (expected).  
- `plato-torch/configs/holodeck_bridge.yaml` exists, defining:  
  - WebSocket endpoint: `ws://localhost:8080/ws` (default holodeck-rust server)  
  - Message format: `{ "type": "npc_interaction", "data": {...} }`  
- `plato-torch/bridge/holodeck_client.py` is a stub — contains `TODO: implement async websocket client`.  
- **No active socket connection in current code.**  

**Integration Gap:** Bridge is configured but not implemented. Missing:  
- Active WebSocket client in `holodeck_client.py`.  
- Sentiment-aware NPC responses flowing into training tiles.

## 3. plato-torch → plato-ensign  
**Connection:** **Strong (one-way export)**  
- `plato-torch/exporter/ensign_exporter.py` imports `plato_ensign`.  
- Function: `export_room_to_lora(room_id, adapter_path)` calls `plato_ensign.create_adapter(...)`.  
- Trigger: After every 1000 training tiles, room can export a LoRA adapter.  
- Shared: Training tiles (JSONL format) are passed to `plato_ensign` for fine-tuning.  

**Integration Gap:** None. This is a functional export pipeline.

## 4. fleet-simulator → holodeck-rust  
**Connection:** **None observed**  
- `fleet-simulator/src/` contains no references to `holodeck`.  
- Simulator generates its own synthetic NPC dialogues (simple Markov chain).  
- No WebSocket or HTTP client toward holodeck server.  

**Integration Gap:** Potential duplication. Simulator’s NPC dialogue could be replaced by holodeck’s sentiment-aware NPCs.

## 5. holodeck-rust → plato-ensign  
**Connection:** **None observed**  
- `holodeck-rust/src/` has no references to `plato_ensign` or LoRA adapters.  
- NPC responses are rule-based + sentiment model (local).  
- No training tile generation or export.  

**Integration Gap:** Holodeck interactions are not currently captured as training tiles.

## 6. fleet-simulator → plato-ensign  
**Connection:** **Indirect via plato-torch**  
- Simulator data flows into `plato-torch` rooms.  
- Those rooms may export to `plato-ensign`.  
- No direct simulator → ensign link.  

**Integration Gap:** None required; indirect path is functional.

## 7. GhostInjector (plato-torch) → holodeck-rust

- Purpose: Inject “ghost tiles” (dead agents’ lessons) into live training.  
- Configuration includes `holodeck_target: true`, but no active bridge.  

**Integration Gap:** Same as general holodeck bridge — WebSocket client missing.

## 8. DeadbandRoom → plato-relay
