## 1. Verified Connections (What's Connected)

### plato-torch → fleet-simulator
- **Connection:** Direct import via `fleet_simulator` Python module.
- **Evidence:** `plato-torch/rooms/deadband_room.py` contains:
  ```python
  from fleet_simulator import FleetSimulator, AgentState
  ```
- **Purpose:** `DeadbandRoom` uses `FleetSimulator` to generate synthetic training data (agent trajectories) for protocol training.
- **Data Flow:** Simulator generates `AgentState` objects → room converts them to interaction tiles.

### plato-torch → plato-ensign
- **Connection:** Configuration-driven export pipeline.
- **Evidence:** `plato-torch/config/ensign_export.yaml` defines:
  ```yaml
  export_format: lora
  source_room: deadband
  adapter_path: ./exports/
  ```
- **Purpose:** After room training accumulates sufficient tiles, ensign export can be triggered to create LoRA adapters.
- **Data Flow:** Room tile database → filtered/processed → LoRA weights.

### holodeck-rust → constraint-theory-core
- **Connection:** Cargo dependency.
- **Evidence:** `holodeck-rust/Cargo.toml` includes:
  ```toml
  [dependencies]
  constraint-theory-core = { path = "../constraint-theory-core" }
  ```
- **Purpose:** Uses geometric snapping library for spatial NPC interactions and environment constraints.
- **Data Flow:** Constraint solver called during NPC movement and object placement.
