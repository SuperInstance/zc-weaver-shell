## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**  
- `plato-torch/src/rooms/deadband_room.py` — Deadband Protocol implementation.  
- `plato-torch/src/rooms/__init__.py` — Room registry.  
- `plato-torch/src/tile_server.py` — Receives and validates training tiles.  

**Connections:**  
- **To plato-ensign:** Exports room experience via `export_lora()` in `deadband_room.py`.  
- **To fleet-simulator:** Not directly imported. No shared data structures observed.  
- **To holodeck-rust:** No direct imports.  

**Integration Points:**  
- `tile_server.py` listens for POST requests (port 8082). Accepts JSON tiles with `agent_id`, `room_id`, `content`.  
- Rooms store interaction history in `self.history`.  

## 2. plato-ensign (LoRA Export)  
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `plato-ensign/src/exporter.py` — `export_lora(room_history, config)`.  
- `plato-ensign/src/models/lora_adapter.py` — LoRA weight extraction.  

**Connections:**  
- **From plato-torch:** Called via `deadband_room.export_lora()`.  
- **To fleet-simulator:** No observed imports.  
- **To holodeck-rust:** No observed imports.  

**Integration Points:**  
- Expects `room_history` list of interaction dicts.  
- Outputs `.safetensors` file to `output/`.  

## 3. fleet-simulator (Fleet Simulation)  
**Location:** `fleet-simulator/`  
**Purpose:** Generates training data via simulated fleet interactions.  
**Key Files:**  
- `fleet-simulator/src/simulator/core.rs` — Simulates agent behaviors.  
- `fleet-simulator/src/data/tile_generator.rs` — Produces synthetic training tiles.  

**Connections:**  
- **To plato-torch:** No direct imports. Could feed tiles to `tile_server.py` via HTTP.  
- **To plato-ensign:** No observed imports.  
- **To holodeck-rust:** No observed imports.  

**Integration Points:**  
- `tile_generator.rs` creates JSON matching plato-torch tile schema.  
- Could POST to `localhost:8082/tile`.  

## 4. holodeck-rust (Sentiment-Aware MUD)

**Purpose:** MUD server with NPCs that react to sentiment.

- `holodeck-rust/src/npc/sentiment_engine.rs` — Sentiment analysis.  

**Connections:**  
- **To plato-torch:** No direct imports.  
- **To plato-ensign:** No observed imports.  
- **To fleet-simulator:** No observed imports.  

**Integration Points:**
