## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key files inspected:**
- `plato-torch/src/rooms/deadband_room.py` — DeadbandRoom preset.
- `plato-torch/src/rooms/__init__.py` — Room registry.
- `plato-torch/src/core/tile_server.py` — Receives and validates training tiles.
- `plato-torch/requirements.txt` — Dependencies.

**Connections found:**
- **plato-ensign:** Direct import. `deadband_room.py` imports `from plato_ensign import export_lora`. Called when room exports experience.
- **fleet-simulator:** No direct import found in code. Configuration reference possible via `config/` but not observed in main room files.
- **holodeck-rust:** No Python imports. Potential integration via REST API or socket (not yet implemented).
- **External:** Uses `torch`, `transformers`, `peft` for LoRA handling.

**Integration points:**
- `tile_server.py` expects POST requests with tile JSON (agent_id, room_id, content, metadata).
- Rooms can call `export_lora()` from plato-ensign to generate adapter.

## 2. plato-ensign (LoRA Export)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key files inspected:**
- `plato-ensign/src/exporter.py` — Main export logic.
- `plato-ensign/src/lora_builder.py` — Builds LoRA from tile sequences.
- `plato-ensign/pyproject.toml` — Dependencies.

**Connections found:**
- **plato-torch:** Directly imported by plato-torch rooms (as above). Ensign does not import plato-torch.
- **fleet-simulator:** No import.
- **holodeck-rust:** No import.
- **External:** Uses `peft`, `transformers`, `safetensors`.

**Integration points:**
- Called from plato-torch room after training cycle.
- Expects `tile_sequence` (list of dicts) and `base_model` (Hugging Face path).

## 5. Integration Summary

### Connected:
- **plato-torch → plato-ensign** — Direct Python import. Functional export path.
- **All → tile server** — plato-torch's `tile_server.py` is the central sink (implied).

### Not Connected (Gaps):
- **fleet-simulator → plato-torch** — No direct data pipe. Simulator tiles not sent to room server.
- **holodeck-rust → plato-torch** — No bridge between Rust MUD and Python tile server.
- **Ghost tiles → holodeck-rust** — No injection of P0 knowledge into NPC memory.
- **plato-ensign → fleet-simulator** — No simulation data used for LoRA training.

### Potential Safe Channels (P1):
1. Add REST client in fleet-simulator to POST tiles to `tile_server`.
2. Build a small bridge service (Python or Rust) to forward holodeck events to tile server.
3. Configure plato-ensign to accept tile streams from fleet-simulator log files.
4. Extend holodeck-rust with HTTP client to fetch ghost tiles from memory store.
