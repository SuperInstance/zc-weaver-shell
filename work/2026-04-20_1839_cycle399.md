# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 399  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. Plato-Torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**
- `rooms/deadband_room.py` — Deadband Protocol implementation
- `rooms/__init__.py` — Room registry
- `trainer/self_trainer.py` — Self-training loop
- `data/tile_loader.py` — Loads interaction tiles

**Connections:**
- **To Plato-Ensign:** Exports room experience via `export_lora()` in `ensign_exporter.py`. Direct import: `from plato_ensign import export_lora`.
- **To Fleet-Simulator:** Not directly imported. However, `tile_loader.py` can load JSON tiles that may originate from fleet-simulator outputs (shared tile format).
- **To Holodeck-Rust:** No direct imports. Potential integration via tile ingestion (holodeck generates sentiment-aware NPC interaction tiles).

**Integration Points:**
- Room presets define training environments.
- Self-trainer consumes tiles (from any source) to train models.
- Export function passes trained adapter to plato-ensign.

## 2. Plato-Ensign (LoRA Adapter Export)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**
- `export.py` — Main export logic
- `lora_builder.py` — Builds LoRA weights from training data

**Connections:**
- **From Plato-Torch:** Called by `ensign_exporter.py`. Receives model state dict and training config.
- **To Fleet-Simulator:** No direct connection. Could export adapters for use in simulation agents.
- **To Holodeck-Rust:** No direct connection. Could provide NPC personality adapters.

**Integration Points:**
- Accepts PyTorch state dict from plato-torch.
- Outputs `.safetensors` file (LoRA adapter).

## 3. Fleet-Simulator (Multi-Agent Simulation)
**Location:** `fleet-simulator/`  
**Purpose:** Generates training data via simulated fleet interactions.  
**Key Files:**
- `simulation/engine.py` — Core simulation loop
- `agents/zeroclaw_agent.py` — Simulated Zeroclaw agent
- `output/tile_writer.py` — Writes interaction tiles

**Connections:**
- **To Plato-Torch:** Tiles written in compatible JSON format. Loaded by `tile_loader.py`.
- **To Plato-Ensign:** No direct export. Simulated agents could use exported adapters (not implemented).
- **To Holodeck-Rust:** No direct connection. Both generate tiles but different domains (fleet vs. NPC interactions).

**Integration Points:**
- Tile output format must match plato-torch expectations.
- Simulated agents could be configured with ensign adapters (future).

## 4. Holodeck-Rust (Sentiment-Aware MUD)
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**
- `src/npc/sentiment.rs` — NPC sentiment system
- `src/world/tile_generator.rs` — Generates interaction tiles
- `src/main.rs` — Server entry point

**Connections:**
- **To Plato-Torch:** Tile output potentially compatible. No Rust-Python direct imports.
- **To Plato-Ensign:** Could export NPC personality adapters (not implemented).
- **To Fleet-Simulator:** Separate domains; both generate tiles but different content.

**Integration Points:**
- Tile format alignment needed for cross-ingestion.
- NPC sentiment models could be trained via plato-torch → plato-ensign pipeline.

## 5. Integration Gaps (What's Not Connected)

| Component A | Component B | Gap | Potential Integration |
|-------------|-------------|-----|----------------------|
| Plato-Torch | Holodeck-Rust | No tile ingestion from Rust | Add Rust tile → JSON → Python loader |
| Plato-Ensign | Fleet-Simulator | Sim agents don't use adapters | Load adapters into simulated agents |
| Holodeck-Rust | Fleet-Simulator | Separate tile generators | Unify tile schema across fleet |
| Plato-Torch | Constraint-Theory-Core | No geometric snapping in training | Add constraint rooms for spatial reasoning |
| CudaClaw | All above | GPU runtime not integrated | Plato-torch training on CudaClaw |
