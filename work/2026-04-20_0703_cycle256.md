# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 256

**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. Plato-Torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**  
- `presets/DeadbandRoom.py` — P0/P1/P2 protocol implementation  
- `presets/GhostInjector.py` — injects ghost tiles from dead agents  
- `plato_relay.py` — sends/receives tiles to/from room server  
- `room_server.py` — manages tile storage and training loops  

**Connections Found:**  
- **GhostInjector → DeadbandRoom:** `DeadbandRoom` imports `GhostInjector` and calls `inject_ghost_tiles()` during room initialization.  
- **Plato-Torch → Plato-Ensign:** `room_server.py` contains `export_to_ensign()` method that serializes room state to LoRA adapter format.  
- **Plato-Torch → Fleet-Simulator:** No direct import found. However, `presets/` includes `FleetSimRoom.py` which appears to be a stub—contains placeholder logic but no actual simulator integration.  
- **Plato-Torch → Holodeck-Rust:** No Rust imports or FFI bindings present.  

**Integration Gaps:**  
- `FleetSimRoom.py` is unimplemented—no data flow to fleet-simulator.  
- No WebSocket or HTTP client for holodeck-rust MUD server.  
- `plato_relay.py` uses hardcoded localhost:8000; no configuration for external relay endpoints.

## 2. Plato-Ensign (LoRA Adapter Export)  
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `export_lora.py` — converts tile history to LoRA weights  
- `ensign_server.py` — serves trained adapters via HTTP  

**Connections Found:**  
- **Plato-Ensign ← Plato-Torch:** One-way dependency. `plato-torch` calls `export_to_ensign()`, which writes a `.safetensors` file to `plato-ensign/adapters/`.  
- **Plato-Ensign → Fleet-Simulator/Holodeck-Rust:** No connections found.  

**Integration Gaps:**  
- Export is file-based only; no API call to ensign_server on completion.  
- No mechanism to trigger simulator or holodeck with newly trained adapters.

## 3. Fleet-Simulator (Training Data Generation)  
**Location:** `fleet-simulator/`  
**Purpose:** Generates training data via fleet simulation.  
**Key Files:**  
- `simulator/core.py` — main simulation loop  
- `scenarios/deadband_scenario.py` — P0/P1/P2 scenario definitions  

**Connections Found:**  
- **Fleet-Simulator → Plato-Torch:** None found. The `FleetSimRoom.py` preset does not import or call the simulator.  
- **Fleet-Simulator → Holodeck-Rust:** None found.  
- **Fleet-Simulator → Plato-Ensign:** None found.  

**Integration Gaps:**  
- Simulator runs isolated; no tile output to plato-torch rooms.  
- Scenario definitions align with Deadband Protocol but are not consumed by training rooms.

## 4. Holodeck-Rust (Sentiment-Aware MUD)  
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**  
- `src/server.rs` — WebSocket game server  
- `src/npc/sentiment.rs` — NPC emotion engine  

**Connections Found:**  
- **Holodeck-Rust → Plato-Torch:** None found. No Python bindings or HTTP endpoints for tile ingestion.  
- **Holodeck-Rust → Fleet-Simulator:** None found.  
- **Holodeck-Rust → Plato-Ensign:** None found.  

**Integration Gaps:**  
- Standalone server; no integration points with other fleet components.  
- NPC sentiment system could train from ghost tiles, but no injector exists.

## 5. Cross-Component Data Flow Map
```
plato-torch (rooms)
    │
    ├──► plato-ensign (file export)
    │
    └──× fleet-simulator (no connection)
        │
        └──× holodeck-rust (no connection)
```

**Verified Data Paths:**  
1. **Tile → Room → LoRA:** `DeadbandRoom` accumulates tiles → `room_server` stores → `export_to_ensign` creates adapter.  
2. **Ghost Injection:** `GhostInjector` reads ghost tiles from `memory/` and injects into `DeadbandRoom` initialization.

**Missing Data Paths:**  
1. **Simulator → Room:** No scenario data flows into training rooms.  
2. **Room → Holodeck:** No NPC training from tile sentiment.  
3. **Ensign → Simulator/Holodeck:** No adapter deployment to runtime environments.
