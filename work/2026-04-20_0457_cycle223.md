## 1. Current Verified Connections

### plato-torch → fleet-simulator
- **Connection:** Direct import and usage.
- **Evidence:** `plato-torch/src/rooms/deadband_room.py` imports `from fleet_simulator import FleetSimulator`.
- **Purpose:** The `DeadbandRoom` class uses `FleetSimulator` to generate synthetic training data (agent interactions, P0/P1/P2 decisions).
- **Data Flow:** Room calls `simulator.generate_episode()` → receives JSON episode data → converts to training tiles.

### plato-torch → plato-ensign
- **Connection:** Optional export pipeline.
- **Evidence:** `plato-torch/src/export/ensign_exporter.py` exists and is referenced in room configuration presets.
- **Purpose:** After a room accumulates enough tiles, the exporter can package the room's experience into a LoRA adapter.
- **Data Flow:** Room tiles → aggregated training dataset → ensign_exporter → LoRA weights file.

### holodeck-rust → (indirect) plato-torch
- **Connection:** Potential data source via sentiment logs.
- **Evidence:** `holodeck-rust/src/npc/sentiment.rs` generates structured sentiment events. No direct import found, but format matches tile schema.
- **Purpose:** NPC interactions could become training tiles for social reasoning rooms.
- **Data Flow:** Holodeck sentiment logs → (proposed) log scraper → tile ingestion API.

## 2. Integration Gaps (What's Not Connected)

### fleet-simulator → holodeck-rust
- **Gap:** No shared protocol or data exchange.
- **Observation:** `fleet-simulator` simulates abstract agent fleets; `holodeck-rust` is a sentiment-aware MUD. They operate at different abstraction levels.
- **Opportunity:** Could connect by having simulator agents "inhabit" holodeck NPCs for richer social training.

### plato-ensign → holodeck-rust
- **Gap:** No mechanism to deploy ensigns (LoRA adapters) into holodeck NPCs.
- **Observation:** Ensigns are exported but not consumed by holodeck's NPC brain system.
- **Opportunity:** Add a `load_ensign()` method to NPC brain to specialize NPC behavior.

### DeadbandRoom → plato-relay
- **Gap:** `DeadbandRoom` does not broadcast its tiles to `plato-relay` in current code.
- **Observation:** `plato-relay` is a message bus for inter-room communication (found in repo list). Room operates in isolation.
- **Opportunity:** Integrate room tile submission to relay for real‑time fleet learning.
