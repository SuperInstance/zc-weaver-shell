# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 137

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

---

## 1. plato-torch → plato-ensign (CONNECTED)
**Connection Type:** Direct export pipeline  
**Evidence:**  
- `plato-torch/src/rooms/deadband_room.py` includes `export_to_ensign()` method  
- `plato-ensign/src/export.rs` imports `RoomExperience` struct from plato-torch  
- `plato-ensign/Cargo.toml` has dependency: `plato-torch = { path = "../plato-torch" }`  
- Export creates LoRA adapters from accumulated training tiles

**Integration Status:** ✅ **Live** — Room experience can be exported as ensign models

---

## 2. plato-torch → fleet-simulator (PARTIAL)
**Connection Type:** Data source  
**Evidence:**  
- `fleet-simulator/src/data/ingest.rs` has `ingest_training_tiles()` function  
- Function accepts `Vec<TrainingTile>` which matches `plato-torch/src/tiles.rs`  
- No direct import of plato-torch crate in fleet-simulator  
- Uses serialized JSON format for tile transfer

**Integration Status:** ⚠️ **Indirect** — Data format compatible but no runtime dependency

---

## 3. fleet-simulator → holodeck-rust (NOT CONNECTED)
**Connection Type:** None found  
**Evidence:**  
- `holodeck-rust/src/npc/` has no imports from fleet-simulator  
- `fleet-simulator/src/output/` generates JSON but no holodeck-specific format  
- Both repos exist in fleet but no cross-references in code

**Integration Gap:** NPCs in holodeck don't receive simulated fleet data

---

## 4. holodeck-rust → plato-torch (ONE-WAY)
**Connection Type:** Tile generation  
**Evidence:**  
- `holodeck-rust/src/sentiment/mod.rs` has `generate_training_tile()` function  
- Outputs `TrainingTile` struct that matches plato-torch format  
- No import of plato-torch — defines own struct  
- `plato-torch/src/ingest/` has `ingest_external_tile()` but no holodeck-specific handler

**Integration Status:** ⚠️ **Format-compatible** — Tiles can be ingested but no direct integration

---

## 5. plato-ensign → holodeck-rust (NOT CONNECTED)
**Connection Type:** None  
**Evidence:**  
- `holodeck-rust/Cargo.toml` has no plato-ensign dependency  
- Ensign models (LoRA adapters) not loaded by NPC sentiment system  
- No model inference endpoints in holodeck

**Integration Gap:** Trained ensigns not deployed to holodeck NPCs

---

## 6. fleet-simulator → plato-ensign (NOT CONNECTED)
**Connection Type:** None  
**Evidence:**  
- Simulation data doesn't feed ensign training directly  
- `plato-ensign/src/train.rs` only accepts `RoomExperience` from plato-torch  
- No ingestion path for simulated interaction data

**Integration Gap:** Simulated fleet behavior not captured in ensign models
