## 6. Integration Gaps (What's Not Connected)

1. **fleet-simulator → plato-torch**  
   - Simulated tiles not ingested by training rooms.  
   - Potential: Pipe `sim_tiles/` into `plato-torch/tiles/` with a sync script.

2. **holodeck-rust → plato-torch**  
   - NPC sentiment data not used for training.  
   - Potential: Send sentiment events as training tile metadata.

3. **GhostInjector (unlocated) → holodeck-rust**  
   - Ghost tiles (dead agents' lessons) not injected into MUD.  
   - Potential: Implement a `GhostInjector` service that reads ghost tiles and modifies NPC dialogue.

4. **DeadbandRoom → plato-relay (unlocated)**  
   - Room state not broadcast to other fleet components.  
   - Potential: Integrate a message queue (Redis/ZMQ) for real-time updates.

5. **End-to-end pipeline**  
   - No single pipeline from simulation → training → export → deployment.  
   - Potential: Orchestrate with a shell script or Python pipeline.
