## 1. plato-torch

**Location:** `fleet-knowledge/plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.

**Key files inspected:**
- `presets/DeadbandRoom.py` — Defines `DeadbandRoom` class.
- `presets/__init__.py` — Exports `DeadbandRoom`.
- `README.md` — Describes room presets and training loop.

**Connections found:**
- **plato-ensign:** `DeadbandRoom` includes a method `export_to_ensign()` that serializes room state and calls `plato_ensign.create_adapter()` (import statement present).
- **fleet-simulator:** No direct import found in plato-torch files. Training data appears to be generated internally via `self.generate_training_tiles()`.
- **holodeck-rust:** No Rust imports or FFI bindings detected. Plato-torch is pure Python/PyTorch.
- **cudaclaw:** Not referenced.
- **flux-runtime:** Not referenced.

**Integration notes:**  
Plato-torch is a training front-end. It produces tiles and can export to plato-ensign. It does not appear to consume external simulation data.

- `src/export/mod.rs` — Data export utilities.
- `Cargo.toml` — Dependencies.

**Connections found:**
- **plato-torch:** No direct link. Simulator exports to JSON (`simulation_output.json`). Could be read by Python training scripts, but no import found.
- **plato-ensign:** Not referenced.

- **constraint-theory-core:** Listed as a dependency in `Cargo.toml`. Used for geometric snapping in simulation.
- **cudaclaw:** Not referenced.
- **flux-runtime:** Not referenced.

**Integration notes:**  
Fleet-simulator is a standalone Rust binary that produces simulation data. It uses constraint-theory-core for geometry. Output is not automatically fed into plato-torch.

## Integration Summary

### Connected:
1. **plato-torch → plato-ensign** — Direct Python-to-Rust bridge for LoRA export.
2. **fleet-simulator → constraint-theory-core** — Geometric dependency.
3. **holodeck-rust → flux-runtime** — NPC bytecode execution.

### Not Connected (Gaps):
1. **fleet-simulator → plato-torch** — Simulation data not automatically ingested for training.
2. **holodeck-rust → plato-torch** — NPC interactions not captured as training tiles.
3. **plato-ensign → fleet-simulator/holodeck** — LoRA adapters only from plato-torch rooms.
4. **cudaclaw → any** — GPU runtime isolated.
