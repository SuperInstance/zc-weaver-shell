# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 168

**Status:** Mapping current connections and identifying integration points.

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

---

## 1. **plato-torch** → **plato-ensign** — **CONNECTED**
- **Connection:** Direct export pipeline.
- **Mechanism:** `plato-torch` rooms accumulate interaction tiles. `plato-ensign` provides `export_room_to_lora()` function that converts room experience into a LoRA adapter.
- **Evidence:** `plato-ensign/src/lib.rs` contains `struct RoomExport` and `fn train_lora_from_tiles()`. `plato-torch/rooms/deadband_room.py` has a `export()` method that calls the ensign module.
- **Status:** Functional. Room training data can be exported as a trained LoRA.

---

## 2. **fleet-simulator** → **plato-torch** — **PARTIALLY CONNECTED**
- **Connection:** Training data generation → room ingestion.
- **Mechanism:** `fleet-simulator` generates synthetic fleet interaction logs. These logs can be formatted as "tiles" and submitted to a `plato-torch` room via its HTTP API (`/submit_tile`).
- **Evidence:** `fleet-simulator/README.md` states: "Outputs are compatible with plato-torch tile schema." The simulator's `export/` directory contains JSON files matching the tile format defined in `plato-torch/schemas/tile.py`.
- **Gap:** No automated pipeline observed. Manual formatting and POST required.

---

## 3. **holodeck-rust** → **plato-torch** — **NOT CONNECTED**
- **Connection:** None found.
- **Evidence:** `holodeck-rust` is a standalone MUD server with sentiment-aware NPCs (`src/npc/sentiment.rs`). It outputs game event logs, but no tile export or HTTP client to `plato-torch` observed.
- **Potential Integration Point:** NPC dialogue/event logs could be transformed into training tiles for a "roleplay" room in `plato-torch`.

---

## 4. **plato-torch** → **holodeck-rust** — **NOT CONNECTED**
- **Connection:** None found.
- **Evidence:** `plato-torch` rooms do not currently send commands or influence `holodeck-rust` game state.
- **Potential Integration Point:** A trained "dialogue ensign" from `plato-ensign` could be loaded into `holodeck-rust` NPCs to improve their responses.

---

## 5. **fleet-simulator** → **holodeck-rust** — **NOT CONNECTED**
- **Connection:** None found.
- **Evidence:** Separate codebases, no shared data flow or APIs.
- **Note:** Both are Rust projects; could share `constraint-theory-core` library.

---

## 6. **plato-ensign** → **fleet-simulator** — **NOT CONNECTED**
- **Connection:** None found.
- **Evidence:** `plato-ensign` exports LoRA adapters (`.safetensors`). `fleet-simulator` does not currently load or use these adapters.
- **Potential Integration Point:** A "fleet strategy" ensign could influence agent decision-making in the simulator.
