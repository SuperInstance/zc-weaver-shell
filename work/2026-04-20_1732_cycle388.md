## 1. plato-torch → fleet-simulator  
**Connection:** **Yes** — Direct import.  
**Evidence:**  
- `plato-torch/src/training/rooms/deadband_room.py` imports `from fleet_simulator.simulator import FleetSimulator`.  
- `deadband_room.py` line 47: `self.simulator = FleetSimulator(config_path="fleet_simulator/configs/basic_fleet.json")`.  
- Simulator provides synthetic agent interaction logs used as training data tiles.  

**Data Flow:**  
`fleet_simulator` → generates JSON logs → `deadband_room` ingests as `self.memory_buffer` → tiles submitted to PLATO room server.  

**Integration Gap:**  
- No observed export of trained room state back to simulator for adaptive simulation.  
- Simulator config path is hardcoded; no runtime configuration from room parameters.  

## 2. plato-torch → plato-ensign  
**Connection:** **Yes** — Optional export pipeline.  
**Evidence:**  
- `plato-torch/src/training/export/ensign_exporter.py` exists.  
- `ensign_exporter.py` imports `from plato_ensign.lora_adapter import LoRAAdapter`.  
- `deadband_room.py` has method `export_to_ensign()` that calls `ensign_exporter.export(room_state, output_path)`.  

**Data Flow:**  
Room training loop → accumulates interaction tiles → `export_to_ensign()` → creates LoRA adapter weights file (`.safetensors`).  

**Integration Gap:**  
- Export is manual/invoked; not automatic after training cycles.  
- No observed feedback loop where ensign adapters are reloaded into `plato-torch` for further training.
