## 1. Plato-Torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**
- `presets/DeadbandRoom.py` — Implements Deadband Protocol training loop.
- `presets/__init__.py` — Exports `DeadbandRoom`, `MirrorRoom`, `GhostRoom`.
- `trainer/room_trainer.py` — Main training loop, imports `torch`, `transformers`.
- `utils/tile_loader.py` — Loads interaction tiles from disk.

**Connections:**
- **To Plato-Ensign:** `presets/DeadbandRoom.py` line 47: `from plato_ensign import export_lora`. Direct call to `export_lora(room_state)` after training epochs.
- **To Fleet-Simulator:** None found in imports. However, `data/simulated_tiles/` directory exists with placeholder JSON files suggesting intended ingestion.
- **To Holodeck-Rust:** No Python imports. Configuration file `config/holodeck_endpoints.yaml` contains `rust_server_url: "http://localhost:8080"` but no active HTTP client in code.
- **Internal:** Uses `torch`, `transformers`, `datasets` (Hugging Face). Room states saved as `.pt` checkpoints.

## 2. Plato-Ensign (LoRA Exporter)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**
- `export_lora.py` — Main export function.
- `lora_builder.py` — Builds LoRA layers from room state.
- `config/model_configs.yaml` — Maps room types to base models (e.g., `DeadbandRoom: "meta-llama/Llama-3.2-1B"`).

**Connections:**
- **From Plato-Torch:** Called as library function. `export_lora` expects a `room_state` dict with `tiles` and `model_type`.
- **To Fleet-Simulator/Holodeck-Rust:** No direct connections.
- **Internal:** Uses `peft` (Parameter-Efficient Fine-Tuning) and `transformers`. Outputs `.safetensors` files to `exports/`.

## 3. Fleet-Simulator (Training Data Generator)
**Location:** `fleet-simulator/`  
**Purpose:** Generates synthetic training data for fleet agents.  
**Key Files:**
- `simulate_agents.py` — Simulates multi-agent interactions.
- `output/tile_generator.py` — Formats output as interaction tiles.
- `config/scenarios.yaml` — Defines scenarios (e.g., `deadband_scenario`, `mirror_play`).

**Connections:**
- **To Plato-Torch:** No direct imports. Outputs JSON tiles to `output/tiles/`. Plato-torch’s `tile_loader.py` can load JSON but no automatic ingestion observed.
- **To Holodeck-Rust:** `simulate_agents.py` line 89: `from holodeck_rust.client import send_npc_update` (commented out). Configuration includes `enable_holodeck: false`.
- **Internal:** Uses `numpy`, `pandas`. Simulated agent logic in `agents/`.

## 4. Holodeck-Rust (Sentiment-Aware MUD Server)
**Location:** `holodeck-rust/`  
**Purpose:** Multi-User Dungeon server with sentiment-aware NPCs.  
**Key Files:**
- `src/main.rs` — Actix-web server with WebSocket endpoints.
- `src/npc/sentiment.rs` — Calculates NPC sentiment from dialogue.
- `src/api/tile_ingest.rs` — POST endpoint `/ingest_tile` (unimplemented stub).

**Connections:**
- **From Fleet-Simulator:** Commented-out client import. No active link.
- **To Plato-Torch:** Configuration referenced but no active HTTP client in Python code.
- **Internal:** Uses `actix-web`, `serde`, `tokio`. Exposes WebSocket (`/ws`) for real-time interaction.
