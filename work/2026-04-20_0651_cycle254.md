# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 254  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch
**Location:** `fleet-repos/plato-torch/`  
**Purpose:** Training room presets, self-training rooms, tile processing.

### Verified Connections:
- **plato-ensign:** Direct import in `plato-torch/src/export.rs`:
  ```rust
  use plato_ensign::{LoRAAdapter, export_room_experience};
  ```
  Exports room training data to LoRA adapters via `export_room_experience()`.
- **Deadband Protocol:** Implemented in `plato-torch/src/deadband_room.rs` as `DeadbandRoom` preset.
- **Tile Processing:** Reads/writes `.tile` files in `plato-torch/data/tiles/`. Shared format with fleet-simulator.
- **Configuration:** `plato-torch/config/rooms.toml` references `holodeck-rust` endpoint for NPC sentiment data.

### Integration Gaps:
- No direct import of `fleet-simulator` or `holodeck-rust` in source code.
- Configuration-based link to holodeck, but no compile-time dependency.

## 2. fleet-simulator
**Location:** `fleet-repos/fleet-simulator/`  
**Purpose:** Generate training data via multi-agent fleet simulations.

### Verified Connections:
- **Tile Format:** Uses same `.tile` schema as plato-torch (see `fleet-simulator/src/tile.rs`).
- **Output Directory:** Writes to `fleet-simulator/output/tiles/`, which can be ingested by plato-torch rooms.
- **Agent Definitions:** `fleet-simulator/src/agents/` includes `zeroclaw_agent.rs` matching our architecture.

### Integration Gaps:
- No direct import of plato-torch, holodeck-rust, or plato-ensign.
- Simulation tiles are not automatically forwarded to plato-torch; manual copy required.
- No sentiment integration from holodeck-rust.

## 3. holodeck-rust
**Location:** `fleet-repos/holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs for immersive training.

### Verified Connections:
- **plato-torch:** Referenced in `holodeck-rust/config/server.toml`:
  ```toml
  [training]
  tile_export_path = "../plato-torch/data/tiles/holodeck/"
  ```
- **Sentiment Engine:** `holodeck-rust/src/sentiment/` provides `SentimentNPC` struct.
- **WebSocket API:** Exposes port 8081 for real-time interaction.

### Integration Gaps:
- No direct import of plato-torch or fleet-simulator.
- Tile export is one-way (holodeck → plato-torch) via file system.
- No integration with plato-ensign for LoRA export of NPC interactions.

## 4. plato-ensign
**Location:** `fleet-repos/plato-ensign/`  
**Purpose:** Export room experience into LoRA adapters.

### Verified Connections:
- **plato-torch:** Directly imported by plato-torch as a library.
- **LoRA Format:** `plato-ensign/src/lora.rs` defines `LoRAAdapter` struct.
- **Export API:** `export_room_experience(room_id: String) -> Result<LoRAAdapter>`.

### Integration Gaps:
- Only called from plato-torch; not integrated with fleet-simulator or holodeck-rust.
- No automatic triggering; requires explicit room export command.

## 5. Integration Summary

### Connected:
- **plato-torch ↔ plato-ensign:** Strong library dependency, direct API calls.
- **holodeck-rust → plato-torch:** Configuration-driven file export.
- **fleet-simulator → plato-torch:** Shared tile format, manual file transfer possible.

### Not Connected:
- **fleet-simulator ↔ holodeck-rust:** No shared data flow.
- **fleet-simulator ↔ plato-ensign:** No direct export path.
- **holodeck-rust ↔ plato-ensign:** NPC interactions not exported as LoRA.
- **DeadbandRoom ↔ plato-relay:** Not found in code (plato-relay not in inspected repos).
