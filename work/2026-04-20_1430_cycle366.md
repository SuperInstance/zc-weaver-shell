## 2. plato-ensign
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `lora_exporter.py` — Converts tile history to LoRA weights  
- `config/ensign_config.yaml` — LoRA rank, alpha, target modules  
- `utils/tile_loader.py` — Loads tiles from plato-torch output  

**Connections:**  
- **From plato-torch:** Receives tiles via `tile_loader.load_from_room(room_id)`.  
- **To fleet-simulator:** No direct imports.  
- **To holodeck-rust:** No direct imports.  

**Integration Points:**  
- Expects tile format: `{"room_id": "...", "turns": [...], "metadata": {...}}`.  
- Outputs PyTorch `.safetensors` files compatible with `transformers`.

## 6. Integration Gaps (What's Not Connected)

   - No shared pipeline. Simulator tiles not automatically fed to training rooms.  
   - **Suggestion:** Add `SimulationTileLoader` in plato-torch to consume fleet-simulator output.

2. **holodeck-rust ↔ plato-torch/plato-ensign**  
   - Language barrier (Rust ↔ Python).  
   - **Suggestion:** Implement HTTP tile endpoint in holodeck-rust; Python client in plato-torch.

3. **fleet-simulator ↔ plato-ensign**

   - Ghost tiles (P0 knowledge) not injected into NPC sentiment engine.
