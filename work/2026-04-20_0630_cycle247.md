## 1. Plato-Torch (Training Rooms)
**Repo:** `plato-torch`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**  
- `rooms/deadband_room.py` — Deadband Protocol implementation  
- `rooms/__init__.py` — Room registry  
- `tile_server.py` — Accepts training tiles via HTTP POST  
- `models/room_model.py` — Base room class  

**Connections:**  
- **To Plato-Ensign:** Exports room experience via `export_lora()` method in `deadband_room.py`.  
- **To Fleet-Simulator:** Not directly imported. Potential integration via shared tile format.  
- **To Holodeck-Rust:** No direct imports.  

**Integration Points:**  
- `tile_server.py` listens on port 8080 for tile submissions (JSON).  
- Rooms can be loaded via preset names (e.g., `deadband`).  
- Room state can be serialized to disk for ensign export.

## 2. Plato-Ensign (LoRA Adapter Export)  
**Repo:** `plato-ensign`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `export.py` — Main export script  
- `lora_builder.py` — Builds LoRA from room checkpoints  
- `configs/room_config.yaml` — Export configuration  

**Connections:**  
- **From Plato-Torch:** Imports `DeadbandRoom` from `plato_torch.rooms`.  
- **To Fleet-Simulator:** No direct imports.  
- **To Holodeck-Rust:** No direct imports.  

**Integration Points:**  
- Loads room checkpoint from `plato-torch/checkpoints/`.  
- Configurable via YAML (output path, adapter rank, alpha).  
- Can be triggered via CLI: `python export.py --room deadband`.

## 3. Fleet-Simulator (Training Data Generation)  
**Repo:** `fleet-simulator`  
**Purpose:** Fleet simulation for training data.  
**Key Files:**  
- `simulator/core.py` — Main simulation loop  
- `agents/zeroclaw.py` — Zeroclaw agent implementation  
- `output/tile_generator.py` — Generates training tiles  

**Connections:**  
- **To Plato-Torch:** Tile generator outputs JSON matching `plato-torch` tile schema.  
- **To Plato-Ensign:** No direct imports.  
- **To Holodeck-Rust:** No direct imports.  

**Integration Points:**  
- Tiles can be POSTed to `plato-torch` tile server.  
- Simulation runs produce `tile_batch.json` files.  
- Agent behaviors are configurable via `config/sim_config.yaml`.

## 4. Holodeck-Rust (Sentiment-Aware MUD)  
**Repo:** `holodeck-rust`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**  
- `src/server.rs` — WebSocket game server  
- `src/npc/sentiment.rs` — NPC sentiment engine  
- `src/world/rooms.rs` — Room definitions  

**Connections:**  
- **To Plato-Torch:** No direct imports.  
- **To Plato-Ensign:** No direct imports.  
- **To Fleet-Simulator:** No direct imports.  

**Integration Points:**  
- Exposes WebSocket endpoint on port 8081.  
- NPCs can be configured with emotion profiles (JSON).  
- Room definitions are separate from `plato-torch` rooms.
