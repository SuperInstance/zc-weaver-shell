## 1. Verified Connections (What's Connected)

### plato-torch ↔ fleet-simulator
- **Connection:** Direct integration via `fleet-simulator` as a training data source.
- **Evidence:** `plato-torch/rooms/deadband_room.py` imports `fleet_simulator` and uses `FleetSimulatorEnvironment` to generate simulation-based training episodes.
- **Data Flow:** Fleet simulation states → training tiles → room experience.
- **Purpose:** Provides synthetic, high-volume training data for Deadband Protocol rooms.

### plato-torch ↔ plato-ensign
- **Connection:** Built-in export pipeline.
- **Evidence:** `plato-torch/export/ensign_exporter.py` reads room experience (tiles) and converts them into LoRA adapters via `plato-ensign`.
- **Data Flow:** Room tiles → LoRA training dataset → trained ensign (specialist model).
- **Purpose:** Optional export of room knowledge into portable, fine-tuned models.

### holodeck-rust ↔ (Indirect) plato-torch
- **Connection:** Potential integration via sentiment-aware NPC interactions as tile sources.
- **Evidence:** `holodeck-rust/src/npc/sentiment.rs` defines NPC emotional states that could generate interaction logs. No direct import of `plato-torch` observed, but architecture suggests NPC logs could be formatted as tiles.
- **Data Flow:** NPC interactions → interaction logs → (possible) tile conversion.
- **Purpose:** Provides human-like, sentiment-driven training scenarios.

## 2. Integration Gaps (What's Not Connected)

### plato-ensign ↔ fleet-simulator
- **Gap:** No direct connection observed. Ensign exporter pulls from room tiles, not directly from simulator raw data.
- **Implication:** Ensigns trained only on room-curated data, not raw simulation episodes. May miss low-level simulation patterns.

### holodeck-rust ↔ plato-ensign
- **Gap:** No export path from holodeck interactions to ensign adapters.
- **Implication:** NPC sentiment knowledge remains siloed in holodeck; not captured as portable specialist models.

### holodeck-rust ↔ fleet-simulator
- **Gap:** No shared state or communication. Separate environments.
- **Implication:** Fleet simulation lacks NPC-driven social scenarios; holodeck lacks fleet coordination simulations.

### DeadbandRoom ↔ plato-relay
- **Gap:** `DeadbandRoom` operates locally; no observed integration with `plato-relay` (if exists) for distributed tile sharing.
- **Implication:** Room experience may not be shared across fleet instances in real-time.
