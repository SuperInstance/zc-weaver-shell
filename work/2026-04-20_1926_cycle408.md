## Integration Map Summary

### Connected:
1. **plato-torch ↔ plato-ensign** — Bidirectional:  
   - plato-torch imports `export_lora` from plato-ensign.  
   - plato-ensign imports `BaseRoom` from plato-torch.  
   - **Strong integration:** Room training → LoRA export pipeline.

2. **fleet-simulator → plato-torch** — One-way data flow:  
   - Simulator writes trajectories to plato-torch data directory.  
   - **Weak integration:** Configuration-based, not runtime.

3. **holodeck-rust → constraint-theory-core** — Library dependency:  
   - Uses geometric snapping library.  
   - **Modular integration:** External crate.

### Not Connected:

   - Gap: Sentiment-aware NPCs not used in training rooms.

2. **holodeck-rust ↔ fleet-simulator** — No references.  
   - Gap: Simulation not driving MUD environments.

3. **holodeck-rust ↔ plato-ensign** — No references.  
   - Gap: NPC interactions not exported as specialist models.

4. **fleet-simulator ↔ plato-ensign** — No references.  
   - Gap: Simulation trajectories not directly exported to LoRA.
