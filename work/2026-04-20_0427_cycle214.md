## 1. Repository Structure & Dependencies

Based on reading actual repository files:

### **plato-torch** (`plato-torch/`)
- **Purpose:** Training room presets and self-training infrastructure.
- **Key files:**
  - `presets/DeadbandRoom.py` – Implements Deadband Protocol training.
  - `presets/GhostInjector.py` – Injects ghost tiles from dead agents.
  - `presets/MirrorPlay.py` – Implements screen-to-screen iteration.
  - `plato_relay/` – Contains relay server for inter-room communication.
- **Dependencies:** `torch`, `transformers`, `fastapi`, `pydantic`. No internal fleet imports found in `requirements.txt` or `setup.py`.

### **plato-ensign** (`plato-ensign/`)
- **Purpose:** Exports room experience into LoRA adapters.
- **Key files:**
  - `export_lora.py` – Main export script.
  - `config/room_config.yaml` – References `plato-torch` preset names.
- **Dependencies:** `peft`, `datasets`, `transformers`. No direct imports of other fleet repos in code.

### **holodeck-rust** (`holodeck-rust/`)
- **Purpose:** MUD server with sentiment-aware NPCs.
- **Key files:**
  - `src/npc/sentiment.rs` – Implements sentiment system.
  - `src/world/mod.rs` – World state management.
- **Dependencies:** `serde`, `tokio`, `sqlx`. No imports of `plato-torch` or `fleet-simulator`.

### **fleet-simulator** (`fleet-simulator/`)
- **Purpose:** Generates training data through fleet simulation.
- **Key files:**
  - `simulate.py` – Main simulation loop.
  - `agents/zeroclaw_agent.py` – Simulates Zeroclaw behavior.
- **Dependencies:** `numpy`, `pandas`. No imports of other fleet repos.

## 3. Integration Gaps (What’s Not Connected)

| Gap | Description | Potential Integration Point |
|-----|-------------|-----------------------------|
| **plato-torch → fleet-simulator** | No import of simulation data in plato-torch presets. | Could feed simulated agent interactions into `DeadbandRoom` as training tiles. |
| **holodeck-rust → plato-torch** | No sentiment data flows to training rooms. | Sentiment logs could become training tiles for emotion-aware ensigns. |
| **plato-ensign → holodeck-rust** | No LoRA adapters are loaded into holodeck NPCs. | Could load ensign exports to specialize NPC behavior. |
| **fleet-simulator → plato-ensign** | Simulation outputs are not formatted as room experience for export. | Could convert simulation traces to `room_config.yaml` compatible format. |
| **GhostInjector → holodeck-rust** | Ghost tiles are not injected into holodeck world state. | Could map ghost tiles to NPC memory or event triggers. |
