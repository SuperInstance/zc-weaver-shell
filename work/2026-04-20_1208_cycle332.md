## 1. plato-torch
**Repository:** `plato-torch`  
**Primary Role:** Training room server with 25 preset rooms, self-training capabilities.  
**Key Files:**  
- `rooms/deadband_room.py` ‚Äî Deadband Protocol implementation  
- `rooms/__init__.py` ‚Äî Room registry  
- `server.py` ‚Äî Main training server  
- `tile_store.py` ‚Äî Stores interaction tiles  

**Connections Found:**  
- **plato-ensign:** Direct import in `rooms/deadband_room.py`  
  ```python
  from plato_ensign import export_room_experience
  ```
  Used to export room experience to LoRA adapters.  
- **fleet-simulator:** No direct import found in code. Configuration reference in `configs/training_presets.yaml` mentions `fleet-simulator` as a data source for scenario generation.  
- **holodeck-rust:** No direct import. Not referenced in configuration or code.  
- **Shared Data:** Uses `Tile` dataclass (defined in `tile_store.py`) which contains `agent_id`, `room_id`, `content`, `timestamp`. This structure is referenced by plato-ensign.

## 2. fleet-simulator
**Repository:** `fleet-simulator`  
**Primary Role:** Generates training data through multi-agent fleet simulations.  
**Key Files:**  
- `simulator/core.py` ‚Äî Main simulation loop  
- `agents/zeroclaw_agent.py` ‚Äî Zeroclaw agent implementation  
- `scenarios/deadband_scenario.py` ‚Äî Deadband Protocol scenarios  

**Connections Found:**  
- **plato-torch:** No direct import. Outputs simulation logs in JSON format that match `Tile` structure. Logs are written to `output/tiles/` directory.  
- **plato-ensign:** No direct import.  
- **holodeck-rust:** No direct import or reference.  
- **Shared Data:** Generates JSON files with fields: `agent_id`, `scenario_id`, `action`, `timestamp`, `reward`. Structure similar but not identical to plato-torch `Tile`.

## 3. holodeck-rust

**Key Files:**

- `src/npc/sentiment.rs` ‚Äî Sentiment-aware NPC system

**Connections Found:**  
- **plato-torch:** No direct import or reference.  
- **fleet-simulator:** No direct import or reference.  
- **plato-ensign:** No direct import or reference.  
- **Shared Data:** Uses internal `Event` struct with fields: `player_id`, `room_id`, `command`, `response`, `sentiment_score`. No shared data structures with other repositories.

## 4. plato-ensign
**Repository:** `plato-ensign`  
**Primary Role:** Exports room experience from plato-torch into LoRA adapters for specialist models.  
**Key Files:**  
- `exporter.py` ‚Äî Main export pipeline  
- `lora_adapter.py` ‚Äî LoRA adapter generation  
- `config.yaml` ‚Äî Export configuration  

**Connections Found:**  
- **plato-torch:** Direct import and dependency. `exporter.py` imports `from plato_torch.tile_store import load_tiles`.  
- **fleet-simulator:** No direct import. Configuration has optional `simulation_data_path` parameter.  
- **holodeck-rust:** No direct import or reference.  
- **Shared Data:** Directly consumes `Tile` objects from plato-torch. Outputs `.safetensors` files with LoRA weights.
