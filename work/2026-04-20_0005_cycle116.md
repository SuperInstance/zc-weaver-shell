## 1. Verified Connections (What's Connected)

### **plato-torch ↔ plato-ensign**
- **Connection:** Direct and functional.
- **Mechanism:** `plato-torch` exports training room experience (interaction tiles) via `plato-ensign`'s `export_room()` function.
- **Evidence:**  
  - `plato-ensign/src/lib.rs` defines `export_room(room_id: &str, output_path: &str) -> Result<()>`.  
  - `plato-torch/rooms/deadband_room.py` includes `self.ensign_exporter` attribute and calls `export_to_lora()` after training cycles.
- **Data Flow:** Tiles (JSON) → Room training → LoRA adapter (`.safetensors`).

### **plato-torch ↔ fleet-simulator**
- **Connection:** Indirect via shared tile format.
- **Mechanism:** `fleet-simulator` generates synthetic training tiles in the same JSON schema used by `plato-torch` rooms.
- **Evidence:**  
  - `fleet-simulator/src/tile_generator.rs` produces `Tile { id, agent_id, content, metadata }`.  
  - `plato-torch/tile_server.py` accepts tiles via POST `/submit_tile` with identical structure.
- **Integration Point:** Tiles from simulator can be ingested by any plato-torch room for training.

### **holodeck-rust ↔ (No Direct Integration Yet)**
- **Observation:** `holodeck-rust` is currently a standalone MUD server with sentiment-aware NPCs.  
- **No imports or references** to plato-torch, fleet-simulator, or plato-ensign in its `Cargo.toml` or source files.
- **Potential Link:** Its `NPC` struct has `sentiment: f32` and `memory: Vec<String>`—could be a tile consumer or generator.

## 2. Integration Gaps (What's Not Connected)

### **Gap 1: holodeck-rust → plato-torch**
- **Missing:** No pipeline to send NPC interactions or player dialogues to plato-torch as training tiles.
- **Opportunity:** NPC dialogues could become tiles for sentiment-training rooms.

### **Gap 2: fleet-simulator → holodeck-rust**
- **Missing:** No simulated agents acting inside the MUD.
- **Opportunity:** Use fleet-simulator to generate NPC behavior trees or quest lines.

### **Gap 3: plato-ensign → holodeck-rust**
- **Missing:** No loading of trained LoRA adapters into NPC AI.
- **Opportunity:** Equip NPCs with specialized ensigns (e.g., "merchant ensign", "guard ensign").

### **Gap 4: Deadband Protocol ↔ holodeck-rust**
- **Missing:** The MUD doesn't implement P0/P1/P2 decision layers.
- **Opportunity:** NPCs could use deadband logic for dynamic sentiment shifts.
