## 1. plato-torch
**Location:** `plato-torch/`

## 2. fleet-simulator
**Location:** `fleet-simulator/`  
**Purpose:** Generate training data through multi-agent simulation.  
**Key files:**
- `src/simulation/engine.rs` ŌĆö Core simulation loop.
- `src/agents/mod.rs` ŌĆö Agent definitions.
- `src/output/tiles.rs` ŌĆö Exports simulation events as tiles.

**Connections observed:**
- **To plato-torch:** `tiles.rs` defines `struct SimTile` with fields matching `TrainingTile` (e.g., `agent_id`, `timestamp`, `content`). No direct RustŌåÆPython import, but tile format alignment suggests data flow.
- **To holodeck-rust:** No direct import. Both are Rust projects; could share agent models.
- **To plato-ensign:** No direct link.

**Integration points:**
- Tile output path configurable ŌĆö could write to directory monitored by plato-torch.
- Agent behavior profiles might align with holodeck NPC sentiment models.

## 3. holodeck-rust

**Integration points:**

## 6. Integration Gaps (What's Not Connected)
