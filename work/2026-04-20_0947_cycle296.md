## 1. plato-torch (Training Rooms)  
**Location:** `plato-torch/`  
**Primary Role:** Hosts 25 training room presets, including `DeadbandRoom`. Manages tile ingestion, training loops, and room state.  
**Key Files:**  
- `presets/deadband_room.py` — Implements `DeadbandRoom` class with P0/P1/P2 protocol logic.  
- `core/tile_processor.py` — Processes incoming training tiles (JSON).  
- `core/room_server.py` — HTTP server for tile submission and room state queries.  

**Connections Found:**  
1. **plato-ensign** — `presets/deadband_room.py` imports `from plato_ensign import export_lora`. Direct export path exists.  
2. **fleet-simulator** — No direct imports. However, `tile_processor.py` expects tiles with `simulation_data` field (optional).  
3. **holodeck-rust** — No imports. No shared data structures observed.  
4. **External** — Uses `torch`, `transformers`, `fastapi`.  

**Integration Gap:**  
- No live data feed from `fleet-simulator`. Tiles must be manually formatted with simulation results.  
- No bidirectional link to `holodeck-rust` for sentiment-aware NPC interactions.  

## 2. fleet-simulator (Fleet Simulation)  
**Location:** `fleet-simulator/`  
**Primary Role:** Generates training data via multi-agent fleet simulations. Outputs episodic logs.  
**Key Files:**  
- `src/simulation/engine.rs` — Core simulation loop.  
- `src/output/json_logger.rs` — Exports simulation episodes as JSON.  
- `configs/fleet_scenarios.yaml` — Scenario definitions.  

**Connections Found:**  
1. **plato-torch** — No direct imports. Output JSON must be manually transformed into tiles.  
2. **holodeck-rust** — No imports.  
3. **plato-ensign** — No direct link.  

**Integration Gap:**  
- Simulation outputs are not automatically ingested by `plato-torch`. Manual tile crafting required.  
- No real-time simulation state feeding into training rooms.  

## 3. holodeck-rust (Sentiment-Aware MUD)  
**Location:** `holodeck-rust/`  
**Primary Role:** Multi-User Dungeon server with NPCs that have sentiment memory.  
**Key Files:**  
- `src/npc/sentiment.rs` — Tracks NPC emotional state.  
- `src/protocol/mod.rs` — Network protocol for client connections.  
- `config/world.yaml` — World layout and NPC placements.  

**Connections Found:**  
1. **plato-torch** — None.  
2. **fleet-simulator** — None.  
3. **plato-ensign** — None.  

**Integration Gap:**  
- NPC sentiment data is isolated. No export path to training tiles.  
- No mechanism to inject `GhostInjector` data (from dead agents) into NPC memory.  

## 4. plato-ensign (LoRA Exporter)  
**Location:** `plato-ensign/`  
**Primary Role:** Exports room experience from `plato-torch` into LoRA adapters.  
**Key Files:**  
- `src/exporter.py` — `export_lora(room_state, output_path)` function.  
- `src/adapters/lora_manager.py` — Manages LoRA merging and saving.  

**Connections Found:**  
1. **plato-torch** — Direct import in `deadband_room.py`. Called when room reaches export threshold.  
2. **fleet-simulator** — None.  
3. **holodeck-rust** — None.  

**Integration Gap:**  
- Only triggered from `plato-torch`. No input from other systems.  
- LoRA adapters are not automatically deployed back to `holodeck-rust` NPCs or `fleet-simulator` agents.  

## 5. GhostInjector (Dead Agent Lessons)

**Integration Gap:**
