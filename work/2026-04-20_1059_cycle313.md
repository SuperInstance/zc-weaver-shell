**Cycle:** 313

## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**
- `rooms/deadband_room.py` — Deadband Protocol training room.
- `rooms/__init__.py` — Exports room classes.
- `trainer/self_trainer.py` — Self‑training loop.
- `config/room_configs.yaml` — Room hyperparameters.

**Connections Out:**
- **To plato‑ensign:** `deadband_room.py` imports `from plato_ensign import export_lora` (line 47). Room can export its experience as a LoRA adapter.
- **To fleet‑simulator:** No direct import found. However, `room_configs.yaml` contains a `data_source` field that *could* point to simulator outputs (not currently configured).
- **To holodeck‑rust:** No Python imports. Potential integration via shared tile format (JSON) over network.

**Integration Gap:** Rooms are not wired to receive live training data from fleet‑simulator or holodeck. They currently rely on static datasets.

## 2. fleet‑simulator (Fleet Simulation)
**Location:** `fleet‑simulator/`  
**Purpose:** Generate training data from simulated fleet interactions.  
**Key Files:**
- `src/simulator/core.rs` — Main simulation engine.
- `src/output/json_exporter.rs` — Exports simulation traces as JSON.
- `config/sim_config.toml` — Simulation parameters.

**Connections Out:**
- **To plato‑torch:** None. The simulator writes JSON files to `output/`. No automatic ingestion by plato‑torch observed.
- **To holodeck‑rust:** None in code. Could share agent behavior models conceptually.
- **To plato‑ensign:** No direct link.

**Integration Gap:** Simulator runs independently. Its output is not consumed by any training room. A manual or automated pipeline is missing.

## 3. holodeck‑rust (Sentiment‑Aware MUD)
**Location:** `holodeck‑rust/`  
**Purpose:** Multi‑user dungeon with NPCs that react to player sentiment.  
**Key Files:**
- `src/server/mod.rs` — WebSocket server.
- `src/npc/sentiment_engine.rs` — NPC emotion model.
- `src/world/tile_map.rs` — World representation.

**Connections Out:**
- **To plato‑torch:** None. However, `tile_map.rs` defines a `Tile` struct that could align with plato‑torch’s tile format (needs verification).
- **To fleet‑simulator:** No imports.
- **To plato‑ensign:** No imports.

**Integration Gap:** Holodeck is a standalone game server. Its interactions are not captured as training tiles for plato‑torch rooms.

## 4. plato‑ensign (LoRA Exporter)
**Location:** `plato‑ensign/`  
**Purpose:** Export room experience into LoRA adapters.  
**Key Files:**
- `export_lora.py` — Main export function.
- `config/adapter_config.yaml` — LoRA hyperparameters.

**Connections Out:**
- **To plato‑torch:** Called by `deadband_room.py` after training. One‑way dependency.
- **To fleet‑simulator:** None.
- **To holodeck‑rust:** None.

**Integration Gap:** Only works with plato‑torch rooms. Cannot export experiences from holodeck or simulator directly.

## 5. GhostInjector & DeadbandRoom
**GhostInjector** (`plato‑torch/ghost_injector.py`) reads “ghost tiles” (dead agents’ lessons) and injects them into rooms as P0 negative examples.  
**Current Status:** Injects into `DeadbandRoom` via `add_ghost_tiles()` method.  
**Integration Gap:** Not connected to holodeck. Holodeck NPCs could benefit from ghost‑tile avoidance learning.

**DeadbandRoom** (`plato‑torch/rooms/deadband_room.py`) implements the three‑phase protocol.  
**Current Status:** Exports to plato‑ensign, accepts ghost tiles.  
**Integration Gap:** Not connected to plato‑relay (messaging bus) or live data sources.

## 6. cudaclaw & flux‑runtime
**cudaclaw** (`cudaclaw/`): GPU‑resident agent runtime. Not directly imported by any of the four repos above.  
**flux‑runtime** (`flux‑runtime/`): Self‑assembling bytecode VM. No imports found in plato‑torch, fleet‑simulator, holodeck‑rust, or plato‑ensign.

## Integration Gaps (P1 Safe Channels)
1. **fleet‑simulator → plato‑torch:** No pipeline. Simulator JSON output not consumed.
2. **holodeck‑rust → plato‑torch:** No tile sharing. NPC interactions lost for training.
3. **plato‑torch → plato‑relay:** DeadbandRoom not publishing to messaging bus.
4. **GhostInjector → holodeck‑rust:** Ghost tiles not injected into NPC sentiment engine.
5. **cudaclaw / flux‑runtime:** Not integrated with any of the four core repos.

2. Add a WebSocket hook in holodeck‑rust to stream interaction tiles to a plato‑torch room.
3. Connect DeadbandRoom to plato‑relay (if plato‑relay exists in repo).
4. Extend GhostInjector to write avoidance rules into holodeck’s NPC decision tables.
