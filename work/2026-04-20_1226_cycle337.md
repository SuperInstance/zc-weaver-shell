## 1. plato-torch

**Location:** `plato-torch/`

**Key Files:**
- `rooms/deadband_room.py` — Primary training room implementing Deadband Protocol
- `rooms/__init__.py` — Room registry
- `tile_server.py` — Receives and stores training tiles
- `ensign_exporter.py` — Exports room experience to LoRA adapters

**Connections:**
- **To plato-ensign:** Direct via `ensign_exporter.py`. Exports `.safetensors` LoRA adapters from accumulated room tiles.
- **To fleet-simulator:** No direct import found. Potential integration via shared tile format or simulation data ingestion.
- **To holodeck-rust:** No direct import found. Possible integration through tile ingestion or sentiment data.

**Integration Points:**
- `tile_server.py` listens on a socket (default: `localhost:8765`) for incoming tiles.
- Rooms generate tiles in JSON format with fields: `room_id`, `agent_id`, `timestamp`, `content`, `metadata`.
- `ensign_exporter.py` reads from the tile database and outputs LoRA weights.

## 5. Integration Gaps & Opportunities

### Missing Connections:
1. **fleet-simulator → plato-torch:** Simulation logs not automatically ingested as training tiles.
2. **holodeck-rust → plato-torch:** NPC interaction events not sent to tile server.
3. **Cross-language bridges:** Rust ↔ Python integration points are manual (file-based, socket-based) rather than library-based.

### Potential Integration Channels:
- **Shared Data Format:** Standardize tile JSON schema across all components.
- **Message Bus:** Introduce lightweight message queue (e.g., Redis, ZeroMQ) for inter-component communication.
- **Common Config:** Shared configuration for ports, paths, and logging.

### Immediate Integration Tasks:
1. **Wire GhostInjector into holodeck:** Ghost tiles from `memory/` could be injected as NPC backstories.
2. **Connect DeadbandRoom to plato-relay:** Enable real-time tile streaming to multiple rooms.
3. **Test end-to-end pipeline:** Simulate agent → tile → LoRA export workflow.
