## 1. Plato-Torch
**Repo:** `plato-torch`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:** `DeadbandRoom.py`, `presets/`, `plato_relay.py`, `GhostInjector.py`

### Verified Connections:
- **To Plato-Ensign:** Direct import in `DeadbandRoom.py`:
  ```python
  from plato_ensign import export_room_experience
  ```
  Used to export room experience as LoRA adapters after training.
- **To Fleet-Simulator:** No direct import found. However, `DeadbandRoom` includes a `simulation_mode` flag and references a `FleetSimulator` class in comments, suggesting planned integration.
- **To Holodeck-Rust:** No direct import. `GhostInjector.py` mentions "sentiment-aware NPCs" but no Rust bindings or API calls observed.

### Integration Points:
- `plato_relay.py` provides WebSocket server for tile submission.
- `GhostInjector` reads `memory/` files to inject ghost tiles (dead agents' lessons).
- Presets define room behaviors and training objectives.

## 2. Plato-Ensign
**Repo:** `plato-ensign`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:** `export.py`, `lora_builder.py`

### Verified Connections:
- **From Plato-Torch:** Imported by `DeadbandRoom` as shown above.
- **To Fleet-Simulator/Holodeck-Rust:** No imports or references found.
- **Internal:** Uses PyTorch and Hugging Face `peft` library for LoRA fine-tuning.

### Integration Points:
- Expects a trained room state (model weights + interaction tiles) from `plato-torch`.
- Outputs `.safetensors` files for deployment.

## 3. Fleet-Simulator

- **To Holodeck-Rust:** No references.
- **To Plato-Ensign:** No references.

### Integration Points:

- Logs could be converted to tiles for training.
