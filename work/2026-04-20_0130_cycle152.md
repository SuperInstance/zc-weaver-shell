Based on reading the actual files in the fleet repos, here is the current state of integration between the four core technologies.

## 1. plato-torch (Training Rooms)
**Connections:**
- **plato-ensign:** Direct integration. `plato-torch/rooms/DeadbandRoom.py` exports training data via `export_to_ensign()` method. Creates `.pt` files in `ensign/export/` directory.
- **fleet-simulator:** Indirect connection. `plato-torch/rooms/` includes `FleetSimRoom.py` which imports `fleet_simulator` module. Training data can be generated from simulation runs.
- **holodeck-rust:** No direct connection found in code. No imports or API calls to holodeck in plato-torch files.

**Integration Points:**
- Room presets (25 total) generate interaction tiles
- Tiles stored in `rooms/tiles/` directory
- `plato-relay` service can forward tiles to other systems (mentioned but not implemented)

## 2. plato-ensign (LoRA Export)
