## 1. Verified Connections (What's Connected)

### plato-torch → plato-ensign (Strong, Direct)
- **Connection:** `plato-torch/src/rooms/` exports training data via `plato-ensign/src/export/`.
- **Mechanism:** `Room` class in plato-torch has `export_to_ensign()` method that serializes interaction tiles into LoRA-compatible format.
- **Evidence:** `plato-ensign/tests/test_export.py` imports `DeadbandRoom` from plato-torch and validates tile conversion.
- **Data Flow:** Tiles (JSON) → LoRA adapters (safetensors).

### fleet-simulator → plato-torch (One-way, Data Feed)
- **Connection:** `fleet-simulator/src/scenarios/` generates training episodes that can be loaded into plato-torch rooms.
- **Mechanism:** `SimulationEpisode` class has `to_tiles()` method that produces tile format matching plato-torch's `TileBuffer`.
- **Evidence:** `plato-torch/examples/load_simulated_episodes.py` shows ingestion path.
- **Data Flow:** Simulated agent interactions → training tiles → room training.

### holodeck-rust → (plato-torch) (Planned, Not Direct)
- **Connection:** Design documents mention "sentiment-aware NPC interactions as training data", but no import/export code exists.
- **Mechanism:** Hypothetical: holodeck MUD server could log interactions in tile-compatible JSON.
- **Evidence:** `holodeck-rust/README.md` states "future integration with PLATO training system".
- **Current Status:** No active code connection. Integration is a design intention.

## 2. Integration Gaps (What's Not Connected)

### plato-ensign → holodeck-rust (Missing)
- **Gap:** No mechanism to deploy trained ensigns (LoRA adapters) into holodeck NPCs.
- **Impact:** Holodeck NPCs cannot benefit from room-trained behavioral adapters.
- **Potential Bridge:** Could create `EnsightLoader` in holodeck to apply LoRA to NPC dialogue models.

### fleet-simulator → holodeck-rust (Missing)
- **Gap:** Simulated fleet scenarios cannot drive holodeck MUD events.
- **Impact:** Cannot use simulator to generate live role-playing scenarios.
- **Potential Bridge:** Could adapt `SimulationEpisode` to produce MUD command sequences.

### plato-torch → constraint-theory-core (Missing)
- **Gap:** Geometric constraint solving not integrated into training rooms.
- **Impact:** Rooms cannot train on spatial reasoning tasks.
- **Potential Bridge:** Could create `ConstraintRoom` preset that imports `constraint-theory-core` Rust bindings.

### cudaclaw → plato-torch (Partial)
- **Gap:** GPU-resident runtime not directly used for room inference.
- **Current:** plato-torch uses standard PyTorch CUDA, not cudaclaw's persistent agent runtime.
- **Potential Bridge:** Could replace `Room.inference()` backend with cudaclaw kernel calls.
