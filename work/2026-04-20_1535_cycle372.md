## 5. Integration Summary

### Connected:
- **plato-torch ↔ plato-ensign** — Direct Python import. Functional pipeline: tiles → LoRA adapters.

### Not Connected (Gaps):
- **plato-torch ↔ fleet-simulator** — No imports. Potential integration: simulator tiles → plato-torch ingestion API.  
- **plato-torch ↔ holodeck-rust** — Language barrier (Python/Rust). Potential integration: REST API bridge.  
- **plato-ensign ↔ fleet-simulator** — No connection. Simulated data not directly exported to LoRA.  
- **plato-ensign ↔ holodeck-rust** — No connection.  
- **fleet-simulator ↔ holodeck-rust** — No connection.  

### Shared Data Structure:
- `TrainingTile` (plato-torch) is likely the common format, but only plato-ensign uses it directly.  
- Other repos may have their own tile-like structures (e.g., `InteractionLog` in holodeck-rust).
