## 1. plato-torch → fleet-simulator
**Connection:** Strong, direct integration via `fleet_simulator` Python package.
- `plato-torch/rooms/deadband_room.py` imports `from fleet_simulator import FleetSimulator`.
- `DeadbandRoom` instantiates `FleetSimulator` as `self.simulator`.
- Room passes agent actions to simulator via `self.simulator.step(actions)`.
- Simulator returns observations, rewards, done flags.
- Shared data structure: `actions` dict mapping agent IDs to action vectors.
**Gap:** None observed. This is a working, coupled integration.

## 2. plato-torch → plato-ensign
**Connection:** Indirect, via file export.
- `plato-torch/rooms/base_room.py` has `export_to_ensign()` method.
- Method writes room state (interaction tiles) to `.pt` files in `exports/` directory.
- `plato-ensign` reads these `.pt` files to train LoRA adapters.
- No runtime API calls; file‑based handoff.
**Gap:** No direct Python import. Relies on filesystem synchronization.

## 3. holodeck-rust → plato-torch
**Connection:** None observed in code.
- `holodeck-rust` is a standalone MUD server written in Rust.
- `plato-torch` is Python/PyTorch training framework.
- No imports of `holodeck` in `plato-torch` source.
- No shared configuration files referenced.
**Gap:** No integration. These are separate systems.

## 4. holodeck-rust → fleet-simulator
**Connection:** None observed.
- `fleet-simulator` is Python‑based.
- `holodeck-rust` has no Python bindings in current code.
- No shared network ports or IPC mechanisms visible.
**Gap:** Complete separation.

## 5. fleet-simulator → plato-ensign
**Connection:** None observed.
- `fleet-simulator` generates simulation trajectories.
- `plato-ensign` trains on exported room tiles.
- No direct data flow; tiles must pass through a `plato-torch` room first.
**Gap:** No direct pipeline. Requires plato-torch as intermediary.

## 6. plato-ensign → holodeck-rust
**Connection:** None observed.
- `plato-ensign` outputs LoRA adapters (`.safetensors`).
- `holodeck-rust` has no visible mechanism to load PyTorch adapters.
- No configuration referencing trained ensigns.
**Gap:** No integration.

## 7. GhostInjector (plato-torch) → holodeck-rust

- No Rust bindings or network client to `holodeck-rust`.
**Gap:** Not wired. GhostInjector only works within plato-torch rooms.

## 8. DeadbandRoom → plato-relay
