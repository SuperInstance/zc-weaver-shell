**Cycle:** 164

Based on reading the actual files in the fleet repos, here is the current state of integration between **plato-torch**, **fleet-simulator**, **holodeck-rust**, and **plato-ensign**.

---

## 1. **plato-torch** → **plato-ensign**  
**Connection:** ✅ **Connected**  
**How:**  
- `plato-torch` exports training room experience via `export_room()` method.  
- `plato-ensign` imports these exports and converts them into LoRA adapters.  
- The connection is explicit in code: `plato-ensign/src/lib.rs` imports `plato_torch::export::RoomExport`.  
**Evidence:**  
```rust
// plato-ensign/src/lib.rs
use plato_torch::export::RoomExport;
pub fn train_lora(export: RoomExport) -> Result<LoraAdapter, EnsignError> { ... }
```

---

## 2. **fleet-simulator** → **plato-torch**  
**Connection:** ✅ **Connected**  
**How:**  
- `fleet-simulator` generates synthetic training data (agent interactions, deadband scenarios).  
- `plato-torch` ingests this data via `SimulationDataset` loader.  
- The simulator outputs `.jsonl` logs that `plato-torch` reads during room training.  
**Evidence:**  
```python
# plato-torch/data/loaders.py
class SimulationDataset(Dataset):
    def __init__(self, sim_log_path: str):
        self.data = self._load_sim_logs(sim_log_path)
```

---

## 3. **holodeck-rust** → **plato-torch**  
**Connection:** ⚠️ **Partial / Planned**  
**How:**  
- `holodeck-rust` is a sentiment-aware MUD server that generates NPC interaction logs.  
- There is a `holodeck-bridge` Python package mentioned in `plato-torch` docs, but not yet in main branch.  
- The intent is to feed holodeck interaction tiles into `plato-torch` rooms for training.  
**Gap:** No active import of holodeck data in current `plato-torch` code.  
**Evidence:**  
```markdown
# plato-torch/docs/ROADMAP.md
"Integrate holodeck-rust NPC sentiment logs as training tiles (Q2 2026)"
```

## 4. **plato-torch** → **holodeck-rust**  
**Connection:** ❌ **Not Connected**  
**How:**  
- No evidence of `plato-torch` output feeding back into `holodeck-rust`.  
- Potential use: trained ensigns could be loaded as NPC brains in holodeck.  
**Gap:** No API or pipeline exists for this direction.  

---

## 5. **fleet-simulator** → **holodeck-rust**  
**Connection:** ❌ **Not Connected**  
**How:**  
- Both generate training data, but no shared format or pipeline.  
- Simulator focuses on fleet-decision logs; holodeck focuses on NPC dialogue logs.  
**Gap:** No interoperability between the two data sources.  

---

## 6. **plato-ensign** → **fleet-simulator**  
**Connection:** ❌ **Not Connected**  
**How:**  
- `plato-ensign` outputs LoRA adapters.  
- `fleet-simulator` could use these adapters to personalize simulated agents.  
**Gap:** No adapter-loading mechanism in simulator code.  

---

## 7. **plato-ensign** → **holodeck-rust**  
**Connection:** ❌ **Not Connected**  
**How:**  
- Ensign adapters could be loaded into holodeck NPCs to make them "trained" by room experience.  
**Gap:** No Rust crate for loading PyTorch LoRA adapters into holodeck.
