## 1. plato-torch
**Repository:** `plato-torch`  
**Purpose:** Training room presets and self-training rooms.  
**Key Files:**  
- `rooms/deadband_room.py` — Implements Deadband Protocol training loop.  
- `rooms/__init__.py` — Exports room classes.  
- `trainers/self_trainer.py` — Self‑training logic.  

**Connections:**  
- **To plato-ensign:** `plato-torch/rooms/base_room.py` imports `from plato_ensign.export import export_lora`. This is a direct integration point for exporting room experience to LoRA adapters.  
- **To fleet-simulator:** No import found in code. However, `plato-torch/README.md` mentions “simulated fleet data” as a possible input source. No live code linkage observed.  
- **To holodeck-rust:** No import or reference found.  

**Integration Gaps:**  
- No direct data pipeline from `fleet-simulator` to `plato-torch` training loops.  
- No hook for `holodeck-rust` sentiment‑aware NPC interactions to feed into room training.

## 2. fleet-simulator
**Repository:** `fleet-simulator`  
**Purpose:** Generate training data from simulated fleet interactions.  
**Key Files:**  
- `simulator/core.py` — Main simulation engine.  
- `output/formatters.py` — Formats simulation traces as training tiles.  

**Connections:**  
- **To plato-torch:** None found in code. The simulator writes JSONL files to disk; rooms would need to read those files explicitly.  
- **To plato-ensign:** No import found.  
- **To holodeck-rust:** No import found.  

**Integration Gaps:**  
- Simulator output is not automatically ingested by any training room.  
- No real‑time streaming of simulation events to other components.

**Integration Gaps:**

## 4. plato-ensign
**Repository:** `plato-ensign`  
**Purpose:** Export room experience into LoRA adapters.  
**Key Files:**  
- `export/lora_exporter.py` — Converts room checkpoints to LoRA.  
- `utils/merge.py` — Merges multiple LoRA adapters.  

**Connections:**  
- **To plato-torch:** Direct import as noted above.  
- **To fleet-simulator:** None.  
- **To holodeck-rust:** None.  

**Integration Gaps:**  
- Only works with `plato-torch` room checkpoints; cannot export from other data sources (e.g., simulator traces, NPC logs).
