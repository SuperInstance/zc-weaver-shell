## 1. plato-torch (Training Rooms)

**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**  
- `rooms/deadband_room.py` — Deadband Protocol training room.  
- `rooms/__init__.py` — Exports room classes.  
- `trainer/self_train.py` — Self‑training loop.  
- `config/room_configs.yaml` — Room hyperparameters.

**Connections Outbound:**  
- **plato‑ensign:** `deadband_room.py` imports `from plato_ensign import export_lora` (line 47). Room calls `export_lora()` after training to create a LoRA adapter.  
- **fleet‑simulator:** No direct import found. However, `room_configs.yaml` contains a `simulator_data_path` key pointing to `../fleet-simulator/output/`. This is a configuration‑level dependency.  
- **holodeck‑rust:** No import or config reference found.  
- **cudaclaw:** No import found.  
- **flux‑runtime:** No import found.

**Integration Gaps:**  
- No live data feed from fleet‑simulator; uses static file path.  
- No hook to push trained models to holodeck for NPC sentiment tuning.  
- No GPU‑resident runtime (cudaclaw) integration for accelerated training.

## 2. fleet‑simulator (Fleet Simulation)

**Location:** `fleet-simulator/`  
**Purpose:** Generate training data via multi‑agent simulations.  
**Key Files:**  
- `src/simulator/core.rs` — Core simulation engine.  
- `src/agents/mod.rs` — Agent definitions.  
- `output/` — Directory for generated trajectories.

**Connections Outbound:**  
- **plato‑torch:** None. The simulator writes JSONL files to `output/`. Plato‑torch reads them via the config path. One‑way data flow.  
- **plato‑ensign:** No import found.  
- **holodeck‑rust:** No import found.  
- **constraint‑theory‑core:** `Cargo.toml` includes `constraint-theory-core = { path = "../constraint-theory-core" }`. Used for geometric snapping in agent movement.  
- **cudaclaw/flux‑runtime:** No references.

**Integration Gaps:**  
- Simulator does not trigger plato‑torch training automatically.  
- No real‑time streaming to rooms; batch file‑based only.  
- No feedback loop from trained ensigns back into simulation.

   *Safe channel:* Add a model‑server endpoint in holodeck that loads ensign LoRAs.

2. **fleet‑simulator → plato‑torch:** Batch file dependency only.  
   *Safe channel:* Implement a lightweight gRPC stream from simulator to room.

3. **GhostInjector → holodeck‑rust:** Feature exists but not wired.  
   *Safe channel:* Enable the `ghost_injector` feature and add a memory‑loading system.

4. **cudaclaw → plato‑torch:** GPU acceleration unused.  
   *Safe channel:* Wrap cudaclaw as a PyTorch custom operator for training steps.

5. **plato‑ensign → holodeck‑rust/fleet‑simulator:** Export only from rooms.  
   *Safe channel:* Add exporter hooks in holodeck (NPC memory) and simulator (agent policies).

6. **flux‑runtime → any:** Isolated VM.  
   *Safe channel:* Define a bytecode interface for agent migration between services.
