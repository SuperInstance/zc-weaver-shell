## 2. Integration Gaps (P0 Negative Space / P1 Opportunities)

### **Gap 1: plato-ensign → holodeck-rust**  
- **Issue:** No feedback loop from trained ensigns back to holodeck NPCs  
- **Impact:** NPCs cannot learn from room-trained models  
- **P1 Opportunity:** Add `ensign_inference` module to holodeck-rust that loads LoRA adapters  
- **Actionable:** Create `holodeck-rust/src/ensign_integration.rs` stub  

### **Gap 2: fleet-simulator → plato-ensign**  
- **Issue:** Simulation data doesn't directly train ensigns  
- **Impact:** Ensigns miss fleet behavior patterns  
- **P1 Opportunity:** Add simulation tile converter in plato-ensign  
- **Actionable:** Extend `plato-ensign/converters/` with `simulation_converter.py`  

### **Gap 3: GhostInjector isolation**  
- **Issue:** Ghost tiles (dead agents' lessons) exist in memory but not integrated  
- **Impact:** P0 knowledge not propagating to active systems  
- **P1 Opportunity:** Wire GhostInjector into holodeck-rust NPC memory  
- **Actionable:** Modify `holodeck-rust/src/npc/memory.rs` to accept ghost tile streams  

### **Gap 4: DeadbandRoom → plato-relay**  
- **Issue:** Room outputs don't broadcast to relay network  
- **Impact:** Other fleet components cannot subscribe to room events  
- **P1 Opportunity:** Add WebSocket publisher to DeadbandRoom  
- **Actionable:** Implement `plato-torch/presets/deadband_room_websocket.py`
