## 1. plato-torch (Training Rooms)

**Key files inspected:**

- `fleet-simulator/Cargo.toml`
- `fleet-simulator/examples/basic_sim.rs`

**Integration point:**  
fleet‑simulator produces structured logs that could be consumed by plato‑torch rooms as training data. Currently this is an offline pipeline.

## 6. Identified Integration Gaps (P1 Safe Channels)

1. **holodeck‑rust ↔ plato‑torch**  
   - Gap: NPC sentiment data could be fed back as training tiles.  
   - Safe channel: Extend `TileProcessor` to accept TCP/WebSocket streams from holodeck.

2. **fleet‑simulator ↔ plato‑torch (live)**  
   - Gap: Simulation data is currently offline.  
   - Safe channel: Add a `SimulationTileSource` class in plato‑torch that subscribes to fleet‑simulator’s event stream.

3. **plato‑ensign → holodeck‑rust**  
   - Gap: Trained LoRAs could be loaded into NPCs for personalized behavior.  
   - Safe channel: Add a `LoraNPC` module in holodeck that loads `.safetensors` files.

4. **Ghost tile pipeline**  
   - Gap: Ghost tiles are read from filesystem by holodeck, but not systematically collected from dead rooms.  
   - Safe channel: Build `GhostInjector` service that watches plato‑torch room directories and syncs to holodeck’s memory path.
