## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets, self-training rooms, tile ingestion.  
**Key Files:**
- `presets/DeadbandRoom.py` — Implements Deadband Protocol training loop.
- `presets/__init__.py` — Exports `DeadbandRoom`, `MirrorRoom`, `GhostRoom`.
- `tile_ingestor.py` — Ingests training tiles (JSON) into room state.
- `requirements.txt` — Lists `torch`, `transformers`, `peft` (for LoRA).

**Connections:**
- **plato-ensign:** Direct. `presets/DeadbandRoom.py` imports `from plato_ensign import export_lora` (line 47). Room can export its trained adapter.
- **fleet-simulator:** Indirect via data. `tile_ingestor.py` expects tile format `{"room": "...", "interaction": "...", "reward": float}`. Fleet-simulator could generate such tiles.
- **holodeck-rust:** No direct import. Potential integration via tile format alignment.

## 2. fleet-simulator
**Location:** `fleet-simulator/`  
**Purpose:** Generate training data via multi-agent simulation.  
**Key Files:**
- `src/simulation/engine.rs` — Core simulation loop.
- `src/agents/mod.rs` — Agent definitions (Oracle1, JetsonClaw1, Forgemaster).
- `src/output/tile_writer.rs` — Writes simulation events as tiles.
- `Cargo.toml` — Depends on `serde_json`, `tokio`.

**Connections:**
- **plato-torch:** Output tiles match plato-torch’s expected JSON format (`tile_ingestor.py`). No Rust→Python bridge observed; tiles likely written to shared storage (e.g., `fleet-knowledge/tiles/`).
- **holodeck-rust:** No direct import. Both are Rust projects but separate crates.
- **plato-ensign:** No direct link.
