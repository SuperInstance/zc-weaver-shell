## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets, self-training rooms, tile processing.  
**Key Files:**  
- `presets/DeadbandRoom.py` — Implements Deadband Protocol training loop.  
- `core/tile_processor.py` — Processes incoming tiles into training data.  
- `core/room_server.py` — Serves rooms, manages agent sessions.  

**Connections Found:**  
- **To plato-ensign:** `presets/DeadbandRoom.py` imports `from plato_ensign import export_lora`. Direct call to `export_lora()` when room exports an ensign. Verified.  
- **To fleet-simulator:** No direct imports found. However, `presets/DeadbandRoom.py` references "simulation_data" as a possible tile source in comments. No live integration observed.  
- **To holodeck-rust:** None. No Rust imports or API calls in Python files.  

**Integration Gaps:**  
- DeadbandRoom does not pull live simulation data from fleet-simulator.  
- No hook to inject ghost tiles from holodeck into the training loop.  

## 2. fleet-simulator
**Location:** `fleet-simulator/`  
**Purpose:** Generates training data via multi-agent fleet simulations.  
**Key Files:**  
- `src/simulation_engine.rs` — Core simulation logic.  
- `src/agents/mod.rs` — Defines agent behaviors.  
- `src/output/mod.rs` — Exports simulation results as JSON.  

**Connections Found:**  
- **To plato-torch:** None. Simulation outputs to JSON files (`data/`). No Python bindings or HTTP API observed.  
- **To holodeck-rust:** None. Separate codebase; no shared crates.  
- **To plato-ensign:** None.  

**Integration Gaps:**  
- Simulation data is file-based, not streamed to plato-torch rooms.  
- No mechanism to feed simulation events into holodeck as NPC sentiment triggers.  

## 3. holodeck-rust
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs, persistent world state.  
**Key Files:**  
- `src/npc/sentiment.rs` — NPC emotional state machine.  
- `src/world/events.rs` — In-game event system.  
- `src/api/mod.rs` — (Minimal) REST API endpoints.  

**Connections Found:**  
- **To plato-torch:** None. No Python client in holodeck.  
- **To fleet-simulator:** None.  
- **To plato-ensign:** None.  

**Integration Gaps:**  
- NPC sentiment system does not receive external data (e.g., simulation outcomes, ghost tiles).  
- No export of interaction logs to plato-torch for tile generation.  

## 4. plato-ensign
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `export_lora.py` — Main export function.  
- `train.py` — Fine-tuning logic.  
- `configs/` — LoRA configuration templates.  

**Connections Found:**  
- **From plato-torch:** Imported by DeadbandRoom for ensign export. One-way dependency.  
- **To fleet-simulator:** None.  
- **To holodeck-rust:** None.  

**Integration Gaps:**  
- Only triggered manually from plato-torch rooms. No automatic export based on simulation or holodeck events.
