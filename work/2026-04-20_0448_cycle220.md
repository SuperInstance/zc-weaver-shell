## 1. Verified Connections (What's Connected)

### plato-torch → plato-ensign
- **Connection:** Direct integration via `plato-torch/src/ensign_exporter.py`
- **Mechanism:** `EnsignExporter` class loads trained room checkpoints and exports them as LoRA adapters using `peft` library
- **Data Flow:** Room training tiles → PyTorch checkpoints → LoRA weights (.safetensors) + adapter config (adapter_config.json)
- **Evidence:** `plato-torch/README.md` states: "Export room experience to LoRA adapters via plato-ensign"

### plato-torch → fleet-simulator
- **Connection:** Indirect via shared tile format
- **Mechanism:** Both use `Tile` data structure defined in `plato-torch/src/data/tile.py`
- **Data Flow:** fleet-simulator generates synthetic training tiles in same format consumed by plato-torch rooms
- **Evidence:** `fleet-simulator/src/tile_generator.rs` imports `Tile` struct definition matching plato-torch format

### holodeck-rust → fleet-simulator
- **Connection:** Direct dependency in Cargo.toml
- **Mechanism:** `holodeck-rust/Cargo.toml` includes: `fleet-simulator = { path = "../fleet-simulator" }`
- **Data Flow:** Holodeck uses fleet-simulator's NPC behavior models and tile generation
- **Evidence:** `holodeck-rust/src/npc/mod.rs` imports `fleet_simulator::npc_ai`

## 2. Integration Gaps (What's Not Connected)

### plato-ensign → holodeck-rust
- **Gap:** No direct connection
- **Issue:** LoRA adapters exported by plato-ensign are not consumed by holodeck-rust NPCs
- **Evidence:** `holodeck-rust` Cargo.toml has no `plato-ensign` dependency; no adapter loading code in NPC modules

### fleet-simulator → plato-ensign
- **Gap:** One-way data flow only
- **Issue:** fleet-simulator generates tiles for plato-torch but cannot directly trigger ensign exports
- **Evidence:** No API calls from fleet-simulator to plato-ensign exporter service

### holodeck-rust → plato-torch
- **Gap:** No real-time integration
- **Issue:** Holodeck NPC interactions don't feed directly into plato-torch training rooms
- **Evidence:** No WebSocket or HTTP connection between holodeck server and plato-torch room manager

### constraint-theory-core → Any other component
- **Gap:** Isolated library
- **Issue:** Geometric snapping engine not integrated with training or simulation pipelines
- **Evidence:** No imports of `constraint-theory-core` in plato-torch, fleet-simulator, or holodeck-rust
