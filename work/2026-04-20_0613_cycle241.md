## 1. plato-torch
**Repo:** `plato-torch`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key files inspected:**  
- `plato-torch/src/rooms/deadband_room.py`  
- `plato-torch/src/rooms/__init__.py`  
- `plato-torch/src/tile_server.py`  
- `plato-torch/requirements.txt`

**Connections found:**  
- **To plato-ensign:** Direct import. `deadband_room.py` imports `from plato_ensign import export_lora`. This is a functional hook—when a room’s training loop finishes, it can export a LoRA adapter via `plato-ensign`.  
- **To fleet-simulator:** No direct import found in code. However, `requirements.txt` includes `fleet-simulator` as an optional dependency. The room presets are designed to consume simulation-generated training data, but the link appears configuration-driven, not hard-coded.  
- **To holodeck-rust:** No Python imports. A `holodeck` config section appears in `deadband_room.yaml`, suggesting planned interoperability via socket or API, but no live code connection observed.

**Integration point:**  
`plato-torch` → `plato-ensign` is live. Rooms call `export_lora()` post-training.  
`plato-torch` → `fleet-simulator` is a soft link via data ingestion.  
`plato-torch` → `holodeck-rust` is config-only, not wired.

## 2. fleet-simulator
**Repo:** `fleet-simulator`  
**Purpose:** Fleet simulation for training data.  
**Key files inspected:**  
- `fleet-simulator/src/lib.rs`  
- `fleet-simulator/Cargo.toml`  
- `fleet-simulator/examples/generate_tiles.rs`

**Connections found:**  
- **To plato-torch:** No Rust bindings to Python. The simulator outputs tile data (JSON) that can be read by `plato-torch` rooms. This is a file/stream handoff, not a library integration.  
- **To holodeck-rust:** `Cargo.toml` does not list `holodeck-rust` as a dependency. No shared types observed.  
- **To plato-ensign:** No connection.

**Integration point:**  
`fleet-simulator` → `plato-torch` is a data pipeline (simulation tiles → room ingestion). No API.

## 4. plato-ensign
**Repo:** `plato-ensign`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key files inspected:**  
- `plato-ensign/src/lib.rs`  
- `plato-ensign/Cargo.toml`

**Integration point:**  
`plato-ensign` ← `plato-torch` is live (Python calls Rust).
