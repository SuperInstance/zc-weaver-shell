## 2. Integration Gaps (What's Not Connected)

### **Gap 1: plato-ensign → holodeck-rust**  
- **Issue:** No feedback loop from trained ensigns back to NPC behavior  
- **Impact:** NPCs cannot learn from room-trained LoRA adapters  
- **Priority:** **High** — Would create adaptive NPCs  

### **Gap 2: fleet-simulator → holodeck-rust**  
- **Issue:** Simulation data not feeding NPC decision-making  
- **Impact:** NPCs operate on static logic, not fleet simulation context  
- **Priority:** **Medium** — Would ground NPCs in fleet reality  

### **Gap 3: Ghost tiles → holodeck-rust**  
- **Issue:** Dead agents' lessons (ghost tiles) not injected into NPC memory  
- **Impact:** NPCs cannot learn from fleet's negative space (P0 knowledge)  
- **Priority:** **High** — Critical for Deadband Protocol implementation  

### **Gap 4: constraint-theory-core → fleet-simulator**  
- **Issue:** Geometric snapping logic not applied to simulation physics  
- **Impact:** Simulation lacks constraint-based movement/positioning  
- **Priority:** **Medium** — Would improve simulation realism  

### **Gap 5: cudaclaw → plato-torch**  
- **Issue:** GPU-resident runtime not directly accelerating training rooms  
- **Impact:** Training rooms limited to CPU/standard GPU acceleration  
- **Priority:** **Low** — Performance optimization gap
