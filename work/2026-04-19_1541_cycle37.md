## 1. Verified Current Connections (P1 — Safe Channels)

Based on reading actual fleet repos:

### **plato-torch → plato-ensign**
- **Connection:** Direct export pipeline  
- **Mechanism:** `plato-torch` rooms accumulate interaction tiles → `plato-ensign` exports room experience as LoRA adapters  
- **Evidence:** `plato-ensign/README.md` states: "Exports room experience from plato-torch into LoRA adapters"  
- **Status:** **Connected** — functional pipeline exists

### **plato-torch → fleet-simulator**
- **Connection:** Training data source  
- **Mechanism:** `fleet-simulator` generates simulation data → used as training input for `plato-torch` rooms  
- **Evidence:** `fleet-simulator/README.md`: "Generates training data for fleet agents"  
- **Status:** **Connected** — one-way data flow confirmed

### **holodeck-rust → plato-torch**
- **Connection:** Sentiment-aware NPC interactions become training tiles  
- **Mechanism:** `holodeck-rust` MUD server with sentiment-aware NPCs → interactions captured as tiles → feed into `plato-torch` rooms  
- **Evidence:** `holodeck-rust/src/lib.rs` shows `Tile` struct with sentiment fields  
- **Status:** **Connected** — data format compatibility exists

## 2. Integration Gaps (P0 — Negative Space)

### **Gap 1: plato-ensign → holodeck-rust**
- **Issue:** No documented connection  
- **Impact:** LoRA adapters trained from room experience cannot directly influence NPC behavior  
- **Evidence:** `holodeck-rust` has no import mechanism for LoRA files  
- **Priority:** **High** — breaks feedback loop between training and simulation

### **Gap 2: fleet-simulator → holodeck-rust**
- **Issue:** No bidirectional connection  
- **Impact:** Simulation data doesn't dynamically affect MUD world state  
- **Evidence:** `fleet-simulator` outputs JSON/CSV, `holodeck-rust` expects real-time event streams  
- **Priority:** **Medium** — limits realism of training environment

### **Gap 3: GhostInjector → holodeck-rust**
- **Issue:** Ghost tiles (dead agents' lessons) not integrated into MUD  
- **Impact:** P0 knowledge from failed agents not available to NPCs  
- **Evidence:** `GhostInjector` class exists in `plato-torch` but no hook to `holodeck-rust`  
- **Priority:** **High** — violates Deadband Protocol principle of preserving negative space knowledge

### **Gap 4: DeadbandRoom → plato-relay**
- **Issue:** Room-specific protocol not connected to fleet communication layer  
- **Impact:** DeadbandRoom operates in isolation from fleet coordination  
- **Evidence:** `plato-relay` handles inter-agent messaging, DeadbandRoom has no subscription  
- **Priority:** **Medium** — limits collaborative training scenarios
