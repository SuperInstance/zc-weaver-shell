## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**
- `presets/deadband_room.py` — Deadband Protocol training room.
- `presets/__init__.py` — Exports presets.
- `room.py` — Base Room class.
- `trainer.py` — Training loop.

**Connections:**
- **plato-ensign:** Exports room experience to LoRA adapters.  
  - `plato-torch/presets/deadband_room.py` imports `from plato_ensign import export_lora`.  
  - `export_lora()` called after training in `DeadbandRoom.finalize()`.
- **fleet-simulator:** Provides training data.  
  - `plato-torch/trainer.py` imports `from fleet_simulator import ScenarioLoader`.  
  - `ScenarioLoader` loads `.json` scenarios for room training.
- **holodeck-rust:** Not directly imported. No Rust bindings in Python code.
- **cudaclaw:** GPU runtime for training.  
  - `plato-torch/trainer.py` has conditional import: `if CUDA_AVAILABLE: from cudaclaw import GPUTrainer`.

**Integration Gaps:**
- No direct link to `holodeck-rust`. Training rooms do not call MUD server.
- `plato-relay` not referenced in code (mentioned in tasks but not integrated).

## 2. fleet-simulator (Training Data)
**Location:** `fleet-simulator/`  
**Purpose:** Fleet simulation for training data.  
**Key Files:**
- `scenarios/` — JSON scenario files.
- `loader.py` — `ScenarioLoader` class.
- `simulator.py` — Core simulation engine.

**Connections:**
- **plato-torch:** Provides data.  
  - `fleet-simulator/loader.py` defines `ScenarioLoader`, imported by plato-torch.
  - Scenarios used in `DeadbandRoom` via `load_scenario()`.
- **holodeck-rust:** Potential integration via JSON export.  
  - `fleet-simulator/simulator.py` has `export_to_mud()` method (commented out).
- **plato-ensign:** No direct imports.

**Integration Gaps:**
- `export_to_mud()` not active. No live data flow to holodeck.
- Simulator runs standalone; not invoked by other services.

## 3. holodeck-rust (MUD Server)
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**
- `src/npc/sentiment.rs` — Sentiment system.
- `src/world/mod.rs` — World state.
- `Cargo.toml` — Dependencies.

**Connections:**
- **plato-torch:** None. No Python bindings.
- **fleet-simulator:** None. `export_to_mud()` not implemented.
- **plato-ensign:** None.
- **constraint-theory-core:** Used for spatial snapping.  
  - `Cargo.toml` includes `constraint-theory-core = "0.3"`.
  - `src/world/spatial.rs` imports `constraint_theory_core::snap`.

**Integration Gaps:**
- Isolated from Python training ecosystem.
- No ingestion of fleet-simulator scenarios.
- No connection to plato-torch rooms.

## 4. plato-ensign (LoRA Exporter)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**
- `exporter.py` — `export_lora()` function.
- `adapters/` — LoRA weight storage.
- `config.yaml` — Exporter settings.

**Connections:**
- **plato-torch:** Direct integration.  
  - `plato-ensign/exporter.py` imported by `deadband_room.py`.  
  - Called post-training to save adapter.
- **fleet-simulator:** None.
- **holodeck-rust:** None.
- **cudaclaw:** Optional GPU acceleration.  
  - `exporter.py` checks for `cudaclaw` and uses `GPULoraExporter` if available.

**Integration Gaps:**
- Only receives data from plato-torch rooms.
- No feedback loop to holodeck or simulator.

## 5. GhostInjector & plato-relay (Task References)
**GhostInjector:** Not found in repo search. Likely planned component for injecting ghost tiles into rooms.  
**plato-relay:** Not found in code. Possibly a message bus for room coordination (mentioned in tasks).
