## 1. plato-torch
**Repo:** `plato-torch`  
**Purpose:** Training room presets, self-training rooms.  
**Key Files:** `presets/DeadbandRoom.py`, `presets/__init__.py`, `trainer.py`, `config/room_config.yaml`.

### Verified Connections:
- **Imports:** `torch`, `transformers`, `datasets` (Hugging Face), `peft` (LoRA).  
- **Internal:** `DeadbandRoom` preset references `BaseRoom` class.  
- **Data Flow:** Room instances generate `interaction_tiles` (JSONL) saved to `output/tiles/`.  
- **Configuration:** `room_config.yaml` defines model paths, LoRA settings, tile storage paths.  
- **Integration Point:** Tiles are written to filesystem; no direct API call to other services observed.

### Integration Gaps:
- No import of `fleet-simulator` or `holodeck-rust` modules.  
- Tile output is file-based; no built-in HTTP or WebSocket push to `plato-relay` or `plato-ensign`.  
- Configuration does not reference external service URLs (e.g., holodeck endpoint).

## 2. fleet-simulator
**Repo:** `fleet-simulator`  
**Purpose:** Generate training data via multi-agent simulation.  
**Key Files:** `simulator/core.py`, `agents/zeroclaw_agent.py`, `config/sim_config.toml`.

### Verified Connections:
- **Imports:** `asyncio`, `aiohttp`, `pydantic`.  
- **Internal:** Simulates `ZeroclawAgent` interactions; outputs `simulation_logs.jsonl`.  
- **Data Flow:** Logs saved to `data/logs/`; format similar to plato-torch tiles but includes agent state metadata.  
- **Configuration:** `sim_config.toml` defines agent count, scenario, output paths.

### Integration Gaps:
- No import of `plato-torch` or `holodeck-rust`.  
- Output is standalone; no automatic ingestion by plato-torch rooms.  
- Simulator does not call plato-torch training routines.

## 3. holodeck-rust
**Repo:** `holodeck-rust`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:** `src/server.rs`, `src/npc/sentiment.rs`, `config/server.toml`.

### Verified Connections:
- **Imports:** `tokio`, `serde`, `websocket`.  
- **Internal:** WebSocket server on port `8080`; NPCs use sentiment engine.  
- **Data Flow:** Player interactions logged to `logs/holodeck.log` (JSON format).  
- **Configuration:** `server.toml` defines port, NPC count, sentiment model path.

### Integration Gaps:
- No Rust FFI bindings to Python (`plato-torch`).  
- Logs are separate format; not aligned with plato-torch tile schema.  
- No built-in HTTP endpoint to receive tiles from plato-torch.

## 4. plato-ensign
**Repo:** `plato-ensign`  
**Purpose:** Export room experience into LoRA adapters.  
**Key Files:** `ensign/exporter.py`, `ensign/lora_train.py`, `config/export_config.yaml`.

### Verified Connections:
- **Imports:** `torch`, `peft`, `datasets`.  
- **Internal:** Reads `interaction_tiles` from directory (configurable path).  
- **Data Flow:** Loads tiles, fine-tunes base model with LoRA, outputs adapter to `adapters/`.  
- **Configuration:** `export_config.yaml` references tile source path, base model, LoRA rank.

### Verified Integration with plato-torch:
- **Explicit Connection:** `export_config.yaml` contains `tile_source: "../plato-torch/output/tiles/"` (relative path example).  
- **Shared Schema:** Assumes tiles match plato-torch JSONL format.  
- **Gap:** Manual path configuration; no service discovery.

## Integration Map Summary

### Connected:
- **plato-torch → plato-ensign:** File-based tile flow (configured path).  
- **Shared Data Schema:** Interaction tiles (JSONL) are common currency.

### Not Connected (Gaps):
- **plato-torch ↔ fleet-simulator:** No import or data exchange.  
- **plato-torch ↔ holodeck-rust:** No direct connection; different languages (Python/Rust), separate logs.  
- **fleet-simulator ↔ holodeck-rust:** No interaction.  
- **plato-ensign ↔ fleet-simulator/holodeck-rust:** Only consumes plato-torch tiles.

### Missing Integration Points:
1. **Tile Ingestion Service:** No central service to collect tiles from all sources (plato-torch, fleet-simulator, holodeck-rust).  
2. **plato-relay:** Not yet inspected; likely intended as message bus.  
3. **GhostInjector:** Not found in current file tree; presumed to inject ghost tiles into holodeck.  
4. **DeadbandRoom to plato-relay:** No WebSocket/HTTP client in DeadbandRoom preset.
