## 2. Integration Gaps (P0 Negative Space / P1 Opportunities)

### **Gap 1: holodeck-rust → plato-torch tile pipeline**
- **Problem:** Holodeck generates rich NPC interaction logs, but no pipeline converts them to plato-torch training tiles.  
- **Risk:** Valuable sentiment-training data stranded.  
- **P1 Action:** Write a `holodeck2tile` adapter script in `plato-torch/importers/`.

### **Gap 2: plato-ensign → holodeck-rust feedback loop**
- **Problem:** Ensigns (LoRA adapters) trained on room experience could personalize holodeck NPCs, but no injection point exists.  
- **Risk:** Trained specialist models remain siloed.  
- **P1 Action:** Add a `GhostInjector` module to `holodeck-rust` that loads ensigns for NPC personality modulation.

### **Gap 3: fleet-simulator → plato-ensign direct training**
- **Problem:** Simulator data flows only to plato-torch rooms, not directly to ensign export.  
- **Risk:** Extra hop required; possible data degradation.  
- **P1 Action:** Extend `plato-ensign` to accept simulator logs directly (bypassing room intermediate).

### **Gap 4: DeadbandRoom ↔ plato-relay**
- **Problem:** `DeadbandRoom` (plato-torch preset) lacks real-time relay for multi-agent training.  
- **Risk:** Isolated room training limits collaborative learning.  
- **P1 Action:** Integrate `plato-relay` message bus into DeadbandRoom API.
