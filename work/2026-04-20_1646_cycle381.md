## 1. plato-torch
**Repo:** `plato-torch`  
**Purpose:** Training room presets and self-training rooms.  
**Key files:** `rooms/deadband_room.py`, `rooms/__init__.py`, `trainer/room_trainer.py`, `configs/room_configs.yaml`.

### Verified Connections:
- **To plato-ensign:**  
  - `plato-torch/rooms/base_room.py` imports `from plato_ensign.export import export_lora`.  
  - `plato-torch/trainer/room_trainer.py` calls `export_lora()` after training cycles.  
  - Config `room_configs.yaml` includes `ensign_export_path: "./exports"`.

- **To fleet-simulator:**  
  - `plato-torch/rooms/deadband_room.py` imports `from fleet_simulator.scenarios import DeadbandScenario`.  
  - Uses `DeadbandScenario.generate()` for synthetic training data.  
  - Shared data structure: `TrainingTile` (defined in `plato-torch/data/tile.py`) is consumed by simulator.

- **To holodeck-rust:**  
  - No direct imports found in Python files.  
  - Indirect: `plato-torch/configs/room_configs.yaml` references `holodeck_endpoint: "http://localhost:8080"` but no active HTTP client observed in code.

### Integration Gaps:
- **holodeck-rust connection is config-only.** No client implementation in `plato-torch` to send/receive NPC interactions.
- **DeadbandRoom** does not call `plato-relay` (missing import or config).
