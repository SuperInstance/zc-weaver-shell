## 2. Integration Gaps (What's Not Connected)

### **Gap 1: holodeck-rust → plato-torch**
- **Problem:** NPC interactions in holodeck are not captured as training tiles.
- **Impact:** Lost training data from rich, sentiment-driven conversations.
- **Action:** Wire a `TileInjector` from holodeck events to a DeadbandRoom.

### **Gap 2: plato-ensign → fleet-simulator**
- **Problem:** Ensigns (trained LoRAs) are not used to guide simulation.
- **Impact:** Simulation runs with base models only, not fleet-trained specialists.
- **Action:** Load ensign adapters into simulator's agent models.

### **Gap 3: DeadbandRoom ↔ plato-relay**
- **Problem:** Room training is isolated; no live tile sharing between rooms.
- **Impact:** Rooms train in silos; no cross-room knowledge transfer.
- **Action:** Connect DeadbandRoom to `plato-relay` (message bus) for tile broadcasting.

### **Gap 4: GhostInjector (concept) → holodeck**
- **Problem:** Ghost tiles (P0 lessons from dead agents) exist in memory but aren't injected into live experiences.
- **Impact:** NPCs can't learn from past agent failures.
- **Action:** Implement `GhostInjector` that reads `memory/` files and inserts warnings/constraints into holodeck dialogues.
