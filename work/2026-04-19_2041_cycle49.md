## 1. Verified Current Connections (What's Connected)

Based on reading the actual fleet-knowledge files and repository structures:

### **plato-torch ↔ fleet-simulator**
- **Connection:** Direct integration via training data pipeline
- **Mechanism:** `fleet-simulator` generates synthetic training scenarios (agent interactions, decision trees) that are formatted into "tiles"
- **Flow:** Simulation output → tile format → ingested by plato-torch training rooms
- **Evidence:** Both repos reference shared tile schema; fleet-simulator README states "output compatible with plato-torch training rooms"

### **plato-torch ↔ plato-ensign**
- **Connection:** Core export pipeline
- **Mechanism:** `plato-ensign` extracts LoRA adapters from trained plato-torch rooms
- **Flow:** Room training completes → ensign exports room experience as LoRA weights
- **Evidence:** plato-ensign described as "exports room experience into LoRA adapters" in FLEET-CONTEXT.md

### **holodeck-rust ↔ (Indirect fleet-simulator)**
- **Connection:** Thematic/architectural alignment, not yet direct data flow
- **Mechanism:** Both generate interactive agent environments; holodeck focuses on sentiment-aware NPCs, fleet-simulator on fleet-scale scenarios
- **Evidence:** Both listed under "Key Technologies" as complementary simulation systems

## 2. Identified Integration Gaps (What's Not Connected)

### **Gap 1: holodeck-rust → plato-torch training pipeline**
- **Current state:** holodeck generates NPC interactions but doesn't feed them as training tiles to plato-torch
- **Opportunity:** NPC dialogue/behavior patterns could become sentiment-training tiles
- **Action needed:** Tile-formatter bridge between holodeck output and plato-torch ingestion

### **Gap 2: fleet-simulator → holodeck-rust scenario injection**
- **Current state:** Separate simulation domains (fleet-scale vs. character-scale)
- **Opportunity:** Fleet scenarios could seed holodeck world events; NPC reactions could inform fleet simulation
- **Action needed:** Bidirectional event protocol between simulation systems

### **Gap 3: plato-ensign → cudaclaw runtime deployment**
- **Current state:** Ensigns export LoRA adapters but deployment path unclear
- **Opportunity:** Direct loading of ensign adapters into cudaclaw GPU-resident agents
- **Action needed:** CUDA-aware adapter loader in cudaclaw runtime

### **Gap 4: constraint-theory-core → holodeck-rust spatial reasoning**
- **Current state:** Geometric snapping library isolated from NPC navigation
- **Opportunity:** Constraint-based NPC movement and interaction boundaries
- **Action needed:** Rust FFI bridge or direct crate dependency
