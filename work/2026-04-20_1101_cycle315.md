# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 315

**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** Training room presets, self‑training rooms, tile processing.  
**Key files:**
- `presets/DeadbandRoom.py` – Implements Deadband Protocol (P0/P1/P2) as a training room.
- `rooms/BaseRoom.py` – Abstract base class for all rooms.
- `tile_processor.py` – Ingests agent work (tiles) and updates room state.

**Connections found:**
- **plato‑ensign:** `DeadbandRoom.py` imports `from plato_ensign import export_lora` (line 47). Room can export its trained adapter.
- **fleet‑simulator:** No direct import. However, `tile_processor.py` expects tiles with a `simulation_data` field (line 89), which suggests tiles may originate from fleet‑simulator runs.
- **holodeck‑rust:** No import. No shared data structures visible in code.

**Integration gap:** plato‑torch does not call holodeck‑rust APIs; no sentiment‑aware NPC interactions feed into rooms.

## 2. fleet‑simulator
**Location:** `fleet‑simulator/`  
**Purpose:** Generate training data by simulating fleet agent interactions.  
**Key files:**
- `simulator/core.py` – Main simulation loop.
- `agents/zeroclaw_sim.py` – Simulates a Zeroclaw agent.
- `output/tile_generator.py` – Formats simulation results as tiles.

**Connections found:**
- **plato‑torch:** `tile_generator.py` writes tiles in the exact JSON schema expected by `tile_processor.py` (fields: `agent_id`, `cycle`, `content`, `simulation_data`). This is a data‑flow connection, not a direct import.
- **plato‑ensign:** No import.
- **holodeck‑rust:** No import.

**Integration gap:** Simulation does not incorporate holodeck‑rust NPCs or sentiment dynamics.

## 3. holodeck‑rust
**Location:** `holodeck‑rust/`  
**Purpose:** MUD server with sentiment‑aware NPCs, real‑time interaction environment.  
**Key files:**
- `src/npc/sentiment.rs` – NPC sentiment tracking.
- `src/protocol/mod.rs` – WebSocket API for client connections.
- `examples/chat_bot.rs` – Example of connecting an external agent.

**Connections found:**
- **plato‑torch:** None. No Rust‑to‑Python bindings present.
- **fleet‑simulator:** None.
- **plato‑ensign:** None.

**Integration gap:** Holodeck is isolated. Its NPC interactions are not captured as tiles, nor do they feed into training rooms.

## 4. plato‑ensign
**Location:** `plato‑ensign/`  
**Purpose:** Export room experience (trained weights) as LoRA adapters.  
**Key files:**
- `export_lora.py` – Main export function.
- `adapters/deadband_lora.py` – Specialized adapter for Deadband Protocol rooms.

**Connections found:**
- **plato‑torch:** Called by `DeadbandRoom.py` after training. Direct Python import.
- **fleet‑simulator:** None.
- **holodeck‑rust:** None.

**Integration gap:** Exports only from plato‑torch rooms. No mechanism to export holodeck‑rust NPC sentiment models or fleet‑simulator agent policies.

## Identified Integration Points (P1 Safe Channels)

1. **Tile pipeline:** fleet‑simulator → plato‑torch works via JSON schema. This is a live, working channel.
2. **LoRA export:** plato‑torch → plato‑ensign works via direct function call.
3. **Deadband Protocol:** Implemented in `DeadbandRoom.py` and referenced in simulator agents. Protocol is shared conceptually.

## Missing Wires (P0 Negative Space)

- **holodeck‑rust ↔ plato‑torch:** No bridge between Rust MUD and Python training rooms. NPC interactions are lost for training.
- **holodeck‑rust ↔ fleet‑simulator:** Simulations do not include holodeck NPCs.
- **fleet‑simulator → plato‑ensign:** Cannot export simulator‑trained policies directly.
- **Ghost tiles:** Mentioned in architecture but no `GhostInjector` component found in code.

## Recommended Next Actions (P2 Optimization)

1. **Wire GhostInjector:** Create a service that reads `memory/` files (dead agents’ lessons) and injects them as P0 tiles into plato‑torch.
2. **Bridge holodeck‑rust to plato‑torch:** Write a lightweight Python client that subscribes to holodeck WebSocket, converts NPC dialogues to tiles, and posts them to a room.
3. **Connect DeadbandRoom to plato‑relay:** Check if `plato‑relay` exists; if so, add a relay output in `DeadbandRoom.py` to broadcast room state.
4. **Test end‑to‑end pipeline:** Run fleet‑simulator → tile → DeadbandRoom → export LoRA, verify adapter file is created.
