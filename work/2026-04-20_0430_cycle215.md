## 1. Core Technology Stack Dependencies

Based on reading `fleet-knowledge/` and repository structures:

### **plato-torch** (Training Rooms)
- **Primary Purpose**: 25 preset training rooms, self-training via interaction tiles.
- **Key Files**: `DeadbandRoom.py`, `GhostInjector.py`, `plato_relay.py`
- **Imports From**:
  - `fleet-simulator` (via `from fleet_simulator import Agent` in `DeadbandRoom.py`)
  - `holodeck-rust` (not directly imported — uses socket connection via `HolodeckBridge`)
  - `plato-ensign` (not imported — separate export pipeline)
- **Exports To**: Tile data → PLATO room server (HTTP POST)

### **fleet-simulator** (Agent Simulation)
- **Primary Purpose**: Generate training data via multi-agent simulation.
- **Key Files**: `agent.py`, `environment.py`, `simulation.py`
- **Imports From**: None (standalone simulator)
- **Exports To**: `plato-torch` imports its `Agent` class for training rooms.

### **holodeck-rust** (MUD Server)
- **Primary Purpose**: Multi-user dungeon with sentiment-aware NPCs.
- **Key Files**: `src/server.rs`, `src/npc/sentiment.rs`
- **Imports From**: None (Rust native)
- **Exports To**: Socket API — `plato-torch` connects via `HolodeckBridge` in `DeadbandRoom.py`.

### **plato-ensign** (LoRA Export)
- **Primary Purpose**: Export room experience to LoRA adapters.
- **Key Files**: `export_lora.py`, `train_ensign.py`
- **Imports From**: `plato-torch` (reads tile archives from room training)
- **Exports To**: `.safetensors` files (LoRA weights)
