## 1. plato-torch → fleet-simulator
**Connection:** **Strong**  
- `plato-torch/rooms/deadband_room.py` imports `from fleet_simulator.simulator import FleetSimulator` (line 7).  
- `DeadbandRoom` instantiates `FleetSimulator` as `self.simulator` (line 37).  
- Simulator provides `step()` method called in room's `_process_tile()` (line 89).  
- Simulator state influences room's reward/penalty logic.  
**Data Flow:** Tiles from room → simulator step → updated simulator state → reward/penalty calculation.

## 2. plato-torch → plato-ensign  
**Connection:** **Direct (one-way export)**  
- `plato-torch/export/ensign_exporter.py` exists.  
- Exporter reads room experience (tiles) and creates LoRA adapters via `plato-ensign` library.  
- `plato-torch/rooms/__init__.py` includes `EnsginExporter` in `__all__`.  
- No reverse import: `plato-ensign` does not import `plato-torch`.  
**Data Flow:** Room accumulates tiles → exporter packages → LoRA adapter file.
