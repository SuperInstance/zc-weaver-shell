## 1. plato-torch → fleet-simulator
**Connection:** ✅ **Connected via `fleet-simulator` as a data source for training rooms.**  
- **plato-torch/rooms/deadband_room.py** imports `from fleet_simulator.simulator import FleetSimulator`.  
- **plato-torch/configs/room_presets/deadband.yaml** specifies `data_source: fleet_simulator`.  
- **fleet-simulator/output/** contains generated `.jsonl` logs used by `DeadbandRoom` to create training tiles.  
- **Integration point:** `DeadbandRoom.load_tiles()` calls `FleetSimulator.generate_scenario()`.

**Gap:** No direct feedback loop from plato-torch back to fleet-simulator. Tiles generated in the room are not fed back to adjust simulation parameters.

---

## 2. plato-torch → plato-ensign
**Connection:** ✅ **Connected via export pipeline.**  
- **plato-ensign/src/export.py** imports `from plato_torch.rooms.base import BaseRoom`.  
- **plato-torch/rooms/deadband_room.py** has method `export_to_lora()` that calls `plato_ensign.export.room_to_lora()`.  
- **plato-ensign/configs/export_config.yaml** references `room_class: DeadbandRoom`.  
- **Integration point:** Room training loops can trigger `room.export()` to generate a LoRA adapter.

**Gap:** Export is manual/optional. No automatic triggering based on training convergence metrics.

---

## 3. holodeck-rust → plato-torch
**Connection:** ❌ **No direct connection found.**  
- **holodeck-rust/src/** has no Python imports; it's a Rust MUD server.  
- **plato-torch/** has no Rust bindings or references to holodeck.  
- **Potential indirect link:** Both may consume similar `.jsonl` log formats, but no shared code or configuration.

**Gap:** Holodeck NPC sentiment data is not feeding into plato-torch rooms. No integration between Rust MUD and Python training pipeline.

---

## 4. fleet-simulator → holodeck-rust
**Connection:** ⚠️ **Partial via shared log format.**  
- **fleet-simulator/output/** generates `npc_interactions.jsonl`.  
- **holodeck-rust/assets/npc_data/** contains similarly structured `.json` files.  
- No import or runtime connection. Data appears manually copied/adapted.

**Gap:** No automated pipeline from simulator to holodeck. Changes in fleet-simulator NPC behavior are not reflected in holodeck without manual update.

---

## 5. plato-ensign → Others
**Connection:** 🔄 **One-way export from plato-torch only.**  
- **plato-ensign** reads room state and tiles, outputs LoRA adapters.  
- No connections to fleet-simulator or holodeck-rust.  
- Output adapters are not automatically consumed by other systems.

**Gap:** Ensigns are not automatically deployed back into fleet-simulator or holodeck to close the loop.
