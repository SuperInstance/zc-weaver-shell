# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 225

**Status:** Based on direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch
**Role:** Training room server with 25 preset rooms.  
**Connections:**
- **plato-ensign:** Direct integration via `plato_torch/ensign_exporter.py`. Exports room experience to LoRA adapters. Uses `EnsembleExporter` class.
- **fleet-simulator:** No direct import found. Potential indirect link through shared training data format (tiles).
- **holodeck-rust:** No direct import. Both consume/produce training tiles, but no code-level coupling.
- **Deadband Protocol:** Implemented in `plato_torch/rooms/deadband_room.py` as `DeadbandRoom` preset.
- **plato-relay:** Referenced in `plato_torch/config/relay_config.yaml` for tile forwarding.

## 2. fleet-simulator
**Role:** Generates synthetic training data for fleet agents.  
**Connections:**
- **plato-torch:** No direct import. Outputs training tiles that could be consumed by plato-torch rooms (implied by shared tile schema).
- **holodeck-rust:** No direct import. Simulator may generate NPC sentiment data usable by holodeck.
- **plato-ensign:** No direct import. Simulated data could train ensigns indirectly via plato-torch.
- **Constraint Theory:** Referenced in `fleet_simulator/scenarios/constraint_scenarios.py` for geometric snapping simulations.

## 3. holodeck-rust
**Role:** MUD server with sentiment-aware NPCs.  
**Connections:**
- **plato-torch:** No direct import. Could send interaction tiles to plato-torch via API (not yet implemented).
- **fleet-simulator:** No direct import. May consume simulated NPC behavior data.
- **plato-ensign:** No direct import. NPC sentiment data could be exported to ensigns.
- **GhostInjector:** Not yet integrated. `holodeck_rust/src/ghost_injector.rs` exists but is stub implementation.

## 4. plato-ensign
**Role:** Exports room experience from plato-torch into LoRA adapters.  
**Connections:**
- **plato-torch:** Direct dependency via `plato_torch` Python package. Uses `RoomExperience` class.
- **fleet-simulator:** No direct import.
- **holodeck-rust:** No direct import.
- **cudaclaw:** Referenced in `plato_ensign/exporters/cuda_exporter.py` for GPU-accelerated LoRA training.

## Integration Gaps (P1 Channels)
1. **holodeck-rust → plato-torch:** No live tile streaming. NPC interactions not feeding training rooms.
2. **fleet-simulator → plato-torch:** No direct pipeline for simulated tiles to enter room training.
3. **GhostInjector → holodeck-rust:** Stub implementation. Dead agents' lessons (ghost tiles) not injected.
4. **DeadbandRoom → plato-relay:** Config exists but untested end-to-end.
5. **Constraint Theory → fleet-simulator:** Only referenced in scenarios, not integrated as library.
