## 1. plato-torch
**Repository:** `plato-torch`  
**Purpose:** Training room presets (25 rooms), self-training rooms, tile ingestion and processing.

### Key Files Inspected:
- `plato-torch/rooms/deadband_room.py` — Implements `DeadbandRoom` (training room preset).
- `plato-torch/core/tile_processor.py` — Processes incoming training tiles.
- `plato-torch/config/room_presets.yaml` — Lists 25 room presets including `deadband`, `mirror_play`, `ghost_injector`.

### Verified Connections:
- **To plato-ensign:**  
  `plato-torch/rooms/deadband_room.py` contains:  
  ```python
  from plato_ensign.export import export_room_lora
  ```
  Used in `DeadbandRoom.finalize()` to export room experience as a LoRA adapter.

- **To fleet-simulator:**  
  `plato-torch/config/room_presets.yaml` references `fleet_simulator` as a data source for the `simulation_room` preset.

- **To holodeck-rust:**  
  No direct import found in code. Configuration reference in `room_presets.yaml` under `holodeck_room` preset, but no Python import.

### Integration Gaps:
- No live data pipe from `holodeck-rust` to `plato-torch`. Preset exists but appears to be a placeholder.
- `DeadbandRoom` does not call `plato-relay` (no import found).

## 2. fleet-simulator
**Repository:** `fleet-simulator`  
**Purpose:** Generate synthetic training data simulating fleet agent interactions.

### Key Files Inspected:
- `fleet-simulator/src/simulator.rs` — Core simulation engine.
- `fleet-simulator/scripts/generate_tiles.py` — Outputs training tiles in Plato tile format.
- `fleet-simulator/config/scenarios/` — Contains scenario definitions.

### Verified Connections:
- **To plato-torch:**  
  `generate_tiles.py` writes `.tile` files to `plato-torch/data/raw/` directory (configured in `config.yaml`).  
  `plato-torch`’s `tile_processor.py` reads from that directory.

- **To holodeck-rust:**  
  No import or data exchange found. Scenarios do not reference holodeck NPCs.

### Integration Gaps:
- Simulator runs independently; no trigger from `plato-torch` to generate scenario-specific data.
- No connection to `plato-ensign` — simulated tiles are not automatically exported as LoRAs.

- `holodeck-rust/src/npc/sentiment.rs` — NPC sentiment engine.
- `holodeck-rust/config/world.yaml` — World layout and NPC placements.

### Verified Connections:
- **To plato-torch:**  
  Configuration reference only (`holodeck_room` preset). No Rust‑to‑Python IPC or tile streaming found.

- **To fleet-simulator:**  
  None.

- **To plato-ensign:**  
  None.

### Integration Gaps:
- NPC interactions are not captured as training tiles.
- No `GhostInjector` integration (mentioned in task backlog) — ghost tiles cannot be injected into live holodeck sessions.
- Standalone MUD; not wired into the Plato training pipeline.

## 4. plato-ensign
**Repository:** `plato-ensign`  
**Purpose:** Export room experience (accumulated tiles) into LoRA adapters for specialist ensign models.

### Key Files Inspected:
- `plato-ensign/src/export.rs` — `export_room_lora()` function.
- `plato-ensign/config/export_config.yaml` — LoRA hyperparameters.
- `plato-ensign/scripts/train_lora.py` — Training script.

### Verified Connections:
- **To plato-torch:**  
  Imported by `DeadbandRoom.finalize()` as noted above.  
  `plato-torch` passes a room’s tile history to `export_room_lora()`.

- **To fleet-simulator:**  
  None.

- **To holodeck-rust:**  
  None.

### Integration Gaps:
- Only triggered manually via `DeadbandRoom.finalize()`. No automatic export scheduler.
- Does not ingest data directly from `fleet-simulator` or `holodeck-rust`.
