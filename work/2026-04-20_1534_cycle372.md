**Cycle:** 372

## 1. plato-torch → fleet-simulator
**Connection:** Strong, direct integration via `fleet_simulator` module import.  
**Evidence:**  
- `plato-torch/src/rooms/deadband_room.py` imports `from fleet_simulator import FleetSimulator`.  
- `plato-torch/src/training/trainer.py` uses `FleetSimulator` to generate synthetic training data.  
**Data Flow:** Fleet simulator generates agent interaction logs → formatted as training tiles → ingested by plato-torch rooms.  
**Gap:** No observed reverse dependency (fleet-simulator does not import plato-torch). This is likely intentional — simulator is a standalone data source.

## 2. plato-torch → plato-ensign
**Connection:** Direct export pipeline.  
**Evidence:**  
- `plato-torch/src/export/ensign_exporter.py` contains `export_to_lora()` function.  
- `plato-torch/configs/ensign_config.yaml` defines LoRA adapter parameters (rank, alpha).  
**Data Flow:** Trained room checkpoints → LoRA adapters via plato-ensign’s adapter merging utilities.  
**Gap:** plato-ensign is a separate repo; integration is file-based (checkpoint → export), not runtime-linked.

## 3. holodeck-rust → (plato-torch / fleet-simulator)
**Connection:** Weak, indirect.  
**Evidence:**  
- `holodeck-rust/src/protocol/mod.rs` defines a TCP/WebSocket MUD server.  
- No imports of plato-torch or fleet-simulator in Rust source.  
- `holodeck-rust/README.md` mentions “sentiment-aware NPCs” but no explicit link to fleet training pipelines.  
**Potential Integration Point:** holodeck could be a tile *source* (player logs) or a tile *consumer* (ensign-powered NPCs). Currently neither connection is implemented.  
**Gap:** No code-level integration observed. This is a prime candidate for wiring.

## 4. fleet-simulator → holodeck-rust
**Connection:** None observed.  
**Evidence:**  
- `fleet-simulator/src/simulator.rs` focuses on multi-agent debate and task simulation.  
- No references to “holodeck” or MUD protocols in code or configs.  
**Gap:** Simulator does not drive holodeck environments. Could be a future enhancement — use simulator to generate NPC behavior trees.

## 5. plato-ensign → Others
**Connection:** Output-only.  
**Evidence:**  
- `plato-ensign/src/merge.py` imports `peft` (LoRA library) and `torch`.  
- No imports of other fleet repos.  
**Role:** Plato-ensign is a downstream tool — it takes trained room checkpoints and produces LoRA adapters for deployment elsewhere.

## 6. Ghost Tiles & Mirror Play

**Evidence:**
