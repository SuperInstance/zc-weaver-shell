## 1. plato-torch (Training Rooms)
**Location:** `plato-torch/`  
**Purpose:** Provides 25 training room presets and self-training rooms.  
**Key Files:**  
- `plato-torch/rooms/deadband_room.py` — Implements Deadband Protocol (P0/P1/P2).  
- `plato-torch/rooms/__init__.py` — Exports room classes.  
- `plato-torch/tile_server.py` — Receives and validates training tiles.  
- `plato-torch/config/room_presets.yaml` — Defines 25 preset configurations.

**Connections:**  
- **To plato-ensign:** `plato-torch/rooms/base_room.py` has method `export_to_ensign()` (line 47) that calls `plato_ensign.export_lora_adapter()`.  
- **To fleet-simulator:** No direct import found. However, `plato-torch/rooms/simulator_room.py` references `fleet_simulator` in docstring (potential planned integration).  
- **To holodeck-rust:** No direct import found.  
- **Shared Data:** Training tiles (JSON) follow schema in `plato-torch/schemas/tile_schema.json`.

## 2. plato-ensign (LoRA Adapter Export)
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**  
- `plato-ensign/export.py` — Contains `export_lora_adapter()` function.  
- `plato-ensign/config/ensign_config.yaml` — Configures adapter parameters.

**Connections:**  
- **From plato-torch:** Called by `base_room.export_to_ensign()`.  
- **To fleet-simulator:** No import found.  
- **To holodeck-rust:** No import found.  
- **Shared Data:** Uses `room.state` dictionary from plato-torch to generate adapter.

## 3. fleet-simulator (Fleet Simulation)
**Location:** `fleet-simulator/`  
**Purpose:** Generates training data via fleet simulation.  
**Key Files:**  
- `fleet-simulator/src/simulator.rs` — Core simulation engine.  
- `fleet-simulator/src/agents.rs` — Defines agent behaviors.  
- `fleet-simulator/config/scenarios.yaml` — Simulation scenarios.

**Connections:**  
- **To plato-torch:** No direct import. However, `fleet-simulator/README.md` mentions "training data for PLATO rooms".  
- **To plato-ensign:** No import found.  
- **To holodeck-rust:** No import found.  
- **Shared Data:** Outputs JSON logs in `fleet-simulator/outputs/`. Schema not yet aligned with plato-torch tile schema.

## 4. holodeck-rust (MUD Server)

**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**

**Connections:**  
- **To plato-torch:** No direct import.  
- **To plato-ensign:** No import found.  
- **To fleet-simulator:** No import found.  
- **Shared Data:** Uses custom event format (`HolodeckEvent`). No shared schema with other repos.
