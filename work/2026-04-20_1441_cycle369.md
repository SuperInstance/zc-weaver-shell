## 6. Integration Gaps & Recommendations
**P0 (Negative Space):**  
- Do not assume shared data formats without validation.  
- Do not modify core room logic without testing tile compatibility.

**P1 (Safe Channels):**  
- Create a `TileBridge` service that converts simulation logs and NPC interactions into tiles.  
- Extend `EnsignExporter` to accept generic experience data (not just room state).  
- Define a common tile schema (`tile_schema.json`) for all systems.

**P2 (Optimization):**  
- Implement `TileBridge` in Rust for performance.  
- Add HTTP endpoints to `holodeck-rust` for tile emission.  
- Benchmark LoRA adapter quality from different data sources.
