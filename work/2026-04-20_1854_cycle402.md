## 1. Plato-Torch (Training Rooms)

**Key Files Examined:**
- `plato-torch/rooms/deadband_room.py`
- `plato-torch/rooms/__init__.py`
- `plato-torch/config/room_presets.yaml`
- `plato-torch/core/tile_processor.py`

**Connections Found:**

1. **To Plato-Ensign (Export)**
   - `deadband_room.py` imports `from plato_ensign.export import export_lora_adapter`
   - Contains method `export_ensign()` that calls `export_lora_adapter(room_state, config)`
   - **Status:** Direct integration via Python import. Functional.

2. **To Fleet-Simulator (Data Source)**
   - `room_presets.yaml` references `fleet_simulator` as a data source for "navy-seal" preset
   - `tile_processor.py` has placeholder `# TODO: integrate fleet-simulator batch loader`
   - **Status:** Configuration reference only. No active import or data flow.

3. **To Holodeck-Rust**
   - No imports found in Python files
   - No configuration references to holodeck
   - **Status:** No direct connection.

## 2. Plato-Ensign (LoRA Export)

**Key Files Examined:**
- `plato-ensign/export/__init__.py`
- `plato-ensign/config/export_config.yaml`

**Connections Found:**

1. **From Plato-Torch (Input)**
   - Receives `room_state` dict and `config` dict from plato-torch rooms
   - Processes training tiles into LoRA adapter files
   - **Status:** Passive receiver. No back-connection to plato-torch.

2. **To External Storage**
   - Exports `.safetensors` files to `output/ensigns/`
   - Configurable output path in `export_config.yaml`
   - **Status:** One-way export.

3. **To Other Systems**
   - No imports of fleet-simulator or holodeck-rust
   - **Status:** Isolated export module.

## Integration Gaps (P0 Negative Space)

1. **Plato-Torch → Fleet-Simulator**
   - Configuration reference exists but no active data loading
   - Missing: `from fleet_simulator import load_batch` or equivalent
   - **Risk:** "navy-seal" preset may fail if called.

2. **Holodeck-Rust → Plato-Torch**
   - No connection in either direction
   - Missing: Socket client in plato-torch or HTTP API in holodeck
   - **Risk:** Cannot use holodeck NPC interactions as training tiles.

3. **Plato-Ensign → Fleet Systems**
   - One-way export only
   - Missing: Feedback loop where ensigns improve fleet-simulator or holodeck
   - **Risk:** Static export without iterative improvement.

4. **Data Format Consistency**
   - Plato-torch expects tiles as Python dicts
   - Fleet-simulator outputs JSON
   - Holodeck outputs TCP stream
   - **Risk:** Manual conversion needed between systems.
