## 1. plato-torch

**Location:** `plato-torch/`  
**Purpose:** Training room presets and self-training rooms (25 presets).  
**Key Files:**
- `presets/DeadbandRoom.py` — Implements Deadband Protocol training.
- `presets/__init__.py` — Exports presets.
- `plato_torch/room.py` — Base room class.
- `plato_torch/tile.py` — Tile ingestion and training loop.

**Connections Found:**
- **To plato-ensign:**  
  `presets/DeadbandRoom.py` imports `from plato_ensign import export_lora`.  
  Room can export its trained adapter via `export_lora()` call (line 47).
- **To fleet-simulator:**  
  No direct imports found. However, `presets/DeadbandRoom.py` uses `fleet_simulator` as a string identifier in its `metadata` (line 12). No module import.
- **To holodeck-rust:**  
  No imports or references found.

**Integration Notes:**  
plato-torch is a training front-end. It ingests tiles, runs training loops, and can export to plato-ensign. It does not directly call fleet-simulator or holodeck-rust.

## Summary: What’s Connected vs. What’s Not

### Connected:
1. **plato-torch → plato-ensign**  
   - Direct Python import.  
   - Rooms can export LoRA adapters.  
   - **Integration point:** `presets/DeadbandRoom.py` calls `export_lora()`.

2. **fleet-simulator → (implied) plato-torch**  
   - No code imports, but shared protocol (Deadband).  
   - Simulator outputs could be formatted as tiles for plato-torch ingestion.  
   - **Integration gap:** No automated tile pipeline observed.

### Not Connected:
1. **holodeck-rust** is isolated.  
2. **plato-ensign** only receives calls from plato-torch.  
3. **fleet-simulator** does not call plato-torch or plato-ensign.  
4. **constraint-theory-core**, **cudaclaw**, **flux-runtime** are standalone.
