# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 245  
**Phase:** 4 — Build & Test  
**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.  

## 1. plato-torch → fleet-simulator  
**Connection:** **Strong** (direct import)  
- `plato-torch/src/training_rooms/deadband_room.py` imports `fleet_simulator` as a module.  
- Uses `fleet_simulator.simulate_fleet()` to generate training data.  
- Simulated fleet states (JSON) are passed directly into room training loops.  
- **Data flow:** fleet-simulator → DeadbandRoom → tile generation.  

**Integration gap:** None observed. This is a working pipeline.  

## 2. plato-torch → plato-ensign  
**Connection:** **Strong** (direct export)  
- `plato-torch/src/exports/ensign_exporter.py` imports `plato_ensign`.  
- Room training logs are converted to LoRA adapters via `plato_ensign.export_to_lora()`.  
- Configurable export triggers (epoch count, loss threshold).  
- **Data flow:** room training logs → ensign_exporter → LoRA file.  

**Integration gap:** Export is optional (config-driven). Not all rooms automatically produce ensigns.  

## 3. holodeck-rust → plato-torch  
**Connection:** **Weak** (indirect, via shared data format)  
- `holodeck-rust/src/npc/sentiment.rs` defines `NpcState` structs.  
- `plato-torch/src/data_ingestion/holodeck_loader.py` reads JSON dumps from holodeck, but no live API.  
- No Rust→Python direct calls found. Integration is file-based (JSON logs).  
- **Data flow:** holodeck logs → filesystem → plato-torch loader → training tiles.  

**Integration gap:** No real-time connection. Batch-only ingestion.  

## 4. fleet-simulator → holodeck-rust  
**Connection:** **None** (no imports or shared configs found)  
- `fleet-simulator/src/lib.rs` has no mention of holodeck.  
- `holodeck-rust/Cargo.toml` does not include fleet-simulator as dependency.  
- Simulator does not generate NPC sentiment data; holodeck does not simulate fleet dynamics.  

**Integration gap:** Complete separation. These components model different domains (fleet ops vs. NPC interaction).  

## 5. plato-ensign → holodeck-rust  
**Connection:** **None** (no cross-references)  
- Ensign exports are PyTorch-based (LoRA). Holodeck is Rust/MUD.  
- No evidence of LoRA adapters being loaded into holodeck NPCs.  

**Integration gap:** Ensigns are not currently consumed by holodeck.  

## 6. fleet-simulator → plato-ensign  
**Connection:** **Indirect** (via plato-torch)  
- Simulator data trains rooms; rooms may export ensigns.  
- No direct simulator→ensign pipeline found.  

**Integration gap:** Ensigns do not capture fleet simulation patterns directly.  

## 7. GhostInjector (plato-torch) → holodeck-rust  
**Connection:** **Planned but not implemented**  
- `plato-torch/src/tools/ghost_injector.py` exists (reads ghost tiles).  
- No references to holodeck in its code.  
- Task backlog mentions “Wire GhostInjector into holodeck.”  

**Integration gap:** Ghost tiles (P0 knowledge) are not yet injected into NPC sentiment system.  

## 8. DeadbandRoom → plato-relay  
**Connection:** **Unknown** (plato-relay not found in inspected repos)  
- `DeadbandRoom` has `broadcast()` method but no visible plato-relay import.  
- May be configured via environment variables (not visible in code).  

**Integration gap:** Cannot verify without plato-relay source.
