**Cycle:** 208

## 1. Verified Connections (What's Connected)

### **plato-torch ↔ fleet-simulator**
- **Connection:** Strong, direct integration via `fleet-simulator` as a data source.
- **Evidence:** `plato-torch/rooms/deadband_room.py` imports `fleet_simulator` and uses `FleetSimulator` class to generate training episodes.
- **Data Flow:** `fleet-simulator` simulates fleet interactions → produces episode logs → `DeadbandRoom` ingests logs as training tiles.
- **Interface:** `DeadbandRoom` calls `simulator.generate_episode()` and processes `episode_log`.

### **plato-torch ↔ plato-ensign**
- **Connection:** Direct export pipeline.
- **Evidence:** `plato-torch/export/ensign_exporter.py` exists, with `export_to_ensign()` function.
- **Data Flow:** Trained room state (weights, logs) → exported as LoRA adapter via `plato-ensign`'s adapter builder.
- **Interface:** Exporter reads room checkpoints, calls `plato_ensign.build_lora_adapter()`.

### **holodeck-rust ↔ (Indirect)**
- **Connection:** No direct code imports found between `holodeck-rust` and the other three.
- **Evidence:** `holodeck-rust/Cargo.toml` shows no dependencies on `plato-torch`, `fleet-simulator`, or `plato-ensign`. No Rust FFI bindings present in Python code.
- **Note:** May be connected at runtime via APIs or data files (e.g., shared tile format), but no compile-time linkage.

## 2. Integration Gaps (What's Not Connected)

### **Gap 1: holodeck-rust ↔ plato-torch**
- **Issue:** No direct integration. `holodeck-rust`'s sentiment-aware NPC interactions are not feeding into `plato-torch` training rooms.
- **Potential:** NPC dialogue logs could become training tiles for social reasoning rooms.
- **Missing:** Rust → Python bridge, tile ingestion pipeline.

### **Gap 2: holodeck-rust ↔ fleet-simulator**
- **Issue:** Separate simulation domains. `fleet-simulator` models fleet agents; `holodeck-rust` models NPCs in a MUD.
- **Potential:** Combined simulation where fleet agents interact with holodeck NPCs.
- **Missing:** Shared simulation state, event passing.

### **Gap 3: plato-ensign ↔ fleet-simulator**
- **Issue:** `plato-ensign` exports room experience, but no direct hook to export `fleet-simulator` episodes as adapters.
- **Potential:** Simulator episodes could train specialized "fleet tactics" ensigns.
- **Missing:** Episode → adapter conversion in `plato-ensign`.

### **Gap 4: DeadbandRoom ↔ plato-relay**
- **Issue:** `DeadbandRoom` is isolated; no real-time tile submission to `plato-relay` server.
- **Evidence:** `DeadbandRoom` saves tiles locally; no HTTP POST to relay endpoint.
- **Missing:** `submit_tile()` method calling relay API.
