# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 339

**Status:** Direct file inspection of fleet repositories. Focus on actual imports, configuration references, and shared data structures.

## 1. plato-torch (Training Rooms)

**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**
- `presets/DeadbandRoom.py` — P0/P1/P2 training room.
- `presets/GhostInjector.py` — Injects ghost tiles from dead agents.
- `rooms/base.py` — Base room class.
- `rooms/training_loop.py` — Main training loop.

**Connections Found:**
- **plato-ensign:** `DeadbandRoom` has optional export to LoRA via `plato_ensign.export_lora()` (line 142 in `DeadbandRoom.py`). Conditional on `config.export_enabled`.
- **fleet-simulator:** No direct import found in plato-torch code. However, `DeadbandRoom` uses simulated agents for training (lines 89-94), but these are local mock agents, not the fleet-simulator package.
- **holodeck-rust:** No import or reference found in plato-torch Python files.
- **Shared Data:** Rooms produce `tiles` (JSON) that are sent to the PLATO server. These tiles are the universal training data format.

**Integration Gap:** plato-torch does not directly call fleet-simulator or holodeck-rust. It operates as a standalone training loop that could consume data from them.

## 2. fleet-simulator (Fleet Simulation)

**Location:** `fleet-simulator/`  
**Purpose:** Generate training data from simulated fleet interactions.  
**Key Files:**
- `src/simulator.rs` — Core simulation engine.
- `src/agents/mod.rs` — Simulated agent behaviors.
- `examples/generate_tiles.rs` — Outputs training tiles.

**Connections Found:**
- **plato-torch:** No Rust → Python bindings found. Simulator outputs tiles to JSON files (e.g., `tiles.json`). These could be read by plato-torch, but no automatic pipeline observed.
- **plato-ensign:** No direct reference.
- **holodeck-rust:** No import found. Both are Rust projects but separate.
- **Shared Data:** Outputs tiles in same JSON format as plato-torch rooms.

**Integration Gap:** fleet-simulator runs independently. Its tile output is not automatically fed into a plato-torch room. Manual file transfer would be required.

## 3. holodeck-rust (MUD Server)

**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**
- `src/server.rs` — WebSocket server.
- `src/npc/sentiment.rs` — NPC sentiment engine.
- `src/world/rooms.rs` — Room management.

**Connections Found:**
- **plato-torch:** No reference. holodeck is a live multiplayer server, not a training loop.
- **fleet-simulator:** No reference.
- **plato-ensign:** No reference.
- **Shared Data:** holodeck uses its own protocol (JSON over WebSocket) for client interactions. No tile output observed.

**Integration Gap:** holodeck-rust is isolated. Its NPC interactions could be a rich source of training tiles, but no export mechanism exists.

## 4. plato-ensign (LoRA Exporter)

**Location:** `plato-ensign/`  
**Purpose:** Export room experience into LoRA adapters.  
**Key Files:**
- `src/exporter.rs` — Converts tiles to LoRA datasets.
- `src/lora/train.py` — Training script.

**Connections Found:**
- **plato-torch:** Called from `DeadbandRoom.py` when `export_enabled` is true. Direct integration.
- **fleet-simulator:** No reference.
- **holodeck-rust:** No reference.
- **Shared Data:** Reads tiles from plato-torch room state.

**Integration Gap:** Only connected to plato-torch. Does not accept tiles from other sources like fleet-simulator or holodeck.

## 5. cudaclaw & flux-runtime

**Location:** `cudaclaw/`, `flux-runtime/`  
**Purpose:** GPU-resident agent runtime and self-assembling bytecode VM.  
**Connections Found:** No imports between these and the four core packages above. They appear to be infrastructure for agent execution, not directly integrated with the training data pipeline.

## Summary Map

```
plato-torch (training) 
       │
       ↓ (tiles + optional export)
plato-ensign (LoRA adapter)
       ↑
       │ (manual file transfer possible)
fleet-simulator (tile generation)
       │ (no connection)
holodeck-rust (live NPC interactions)
```

**Connected:**
- plato-torch → plato-ensign (conditional export).

**Not Connected:**
- fleet-simulator → plato-torch (no automatic ingestion).
- holodeck-rust → any training pipeline.
- cudaclaw/flux-runtime → tile generation/training.
