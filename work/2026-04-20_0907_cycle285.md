## 1. plato-torch
**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms.  
**Key Files:**
- `rooms/deadband_room.py` — Deadband Protocol implementation.
- `rooms/__init__.py` — Exports room classes.
- `trainer/room_trainer.py` — Orchestrates room training loops.
- `config/room_configs.yaml` — Defines room parameters.

**Connections:**
- **To plato-ensign:** `plato-torch/export/ensign_exporter.py` imports `from plato_ensign.core import LoRAExporter`. Direct code dependency.
- **To fleet-simulator:** No direct import found. Potential integration via shared data format (simulation logs → training tiles).
- **To holodeck-rust:** No direct import. Plato-torch is Python; holodeck is Rust. Integration would require a bridge (e.g., REST API, shared message queue).

## 2. plato-ensign
**Location:** `plato-ensign/`  
**Purpose:** Exports room experience into LoRA adapters.  
**Key Files:**
- `core/lora_exporter.py` — Main export logic.
- `config/export_config.yaml` — Configures adapter parameters.
- `utils/tile_loader.py` — Loads training tiles from room outputs.

**Connections:**
- **From plato-torch:** Exporter is called by `plato-torch/export/ensign_exporter.py`. Tight coupling.
- **To fleet-simulator:** No import found. Could consume simulation-generated tiles if format matches.
- **To holodeck-rust:** No import. Separate tech stack.

## 3. fleet-simulator
**Location:** `fleet-simulator/`  
**Purpose:** Fleet simulation for training data.  
**Key Files:**
- `src/simulation_engine.rs` — Core simulation logic.
- `src/agent_behavior.rs` — Defines agent decision trees.
- `config/sim_config.toml` — Simulation parameters.
- `output/tile_generator.rs` — Generates training tiles from simulation events.

**Connections:**
- **To plato-torch:** No Rust → Python import. Output tiles are written to `output/tiles/` in JSON format. Plato-torch could read these files if pointed to that directory.
- **To plato-ensign:** Same as above — indirect via file system.
- **To holodeck-rust:** No import found. Both are Rust, but no `use holodeck` statements in fleet-simulator source.

## 4. holodeck-rust
**Location:** `holodeck-rust/`  
**Purpose:** MUD server with sentiment-aware NPCs.  
**Key Files:**
- `src/server/mud_server.rs` — Main server loop.
- `src/npc/sentiment_engine.rs` — NPC emotion modeling.
- `src/world/room_manager.rs` — Manages game rooms.
- `config/holodeck.toml` — Server configuration.

**Connections:**
- **To plato-torch:** None. Different language, different domain (game server vs. training framework).
- **To plato-ensign:** None.
- **To fleet-simulator:** None. Both Rust but separate crates.
