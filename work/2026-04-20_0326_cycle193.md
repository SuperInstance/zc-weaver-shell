# Weaver Integration Map — Verified Connections & Integration Gaps  
**Cycle:** 193  
**Phase:** 4 — Build & Test  
**Status:** Updated map based on actual code inspection of plato-torch, fleet-simulator, holodeck-rust, and plato-ensign repositories.

Based on reading the actual files in the fleet repos, here is the current state of connections and integration gaps:

## 1. **plato-torch** (Training Room Engine)
- **Connected to:** plato-ensign (via export functionality)
  - `plato-torch/src/rooms/deadband_room.py` includes `export_to_ensign()` method that serializes room state and tile history.
  - `plato-torch/src/ensign_exporter.py` provides LoRA adapter generation from room training data.
- **Connected to:** fleet-simulator (indirectly via shared tile format)
  - Both use `Tile` data structure defined in `plato-torch/src/tiles/`.
  - Fleet simulator can generate tiles that plato-torch rooms can ingest.
- **NOT connected to:** holodeck-rust (no direct imports or API calls found).
- **Integration point:** `DeadbandRoom` could feed trained behavior patterns to holodeck NPCs.

## 2. **fleet-simulator** (Multi-Agent Training Data Generator)
- **Connected to:** plato-torch (via tile compatibility)
  - `fleet-simulator/src/tile_generator.rs` produces tiles in same format as plato-torch expects.
  - Simulator can be configured to output to a plato-torch room directory.
- **NOT connected to:** holodeck-rust (no shared protocols found).
- **NOT connected to:** plato-ensign (no direct export path).
- **Integration gap:** Simulator could feed scenario data to holodeck for immersive training.

## 3. **holodeck-rust** (Sentiment-Aware MUD Server)
- **Connected to:** (minimal external connections found)
  - Has `serde` for JSON serialization but no imports from other fleet repos.
  - NPC sentiment system is self-contained.
- **NOT connected to:** plato-torch, fleet-simulator, or plato-ensign.
- **Integration gap:** Could consume trained behavior models from plato-ensign adapters.
- **Integration gap:** Could use fleet-simulator scenarios as quest templates.

## 4. **plato-ensign** (LoRA Adapter Exporter)
- **Connected to:** plato-torch (primary source)
  - `plato-ensign/src/main.rs` reads room state files from plato-torch output.
  - Generates LoRA adapters from accumulated tile data.
- **NOT connected to:** fleet-simulator or holodeck-rust.
- **Integration gap:** Ensign adapters could be loaded into holodeck NPCs.
- **Integration gap:** Could use simulator data as additional training source.

2. **holodeck-rust ↔ fleet-simulator**: Use simulated scenarios as quest backdrops.
3. **fleet-simulator → plato-ensign**: Direct training data pipeline.
4. **plato-torch → holodeck-rust**: Real-time behavior pattern streaming.
