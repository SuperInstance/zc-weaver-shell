## 2. fleet‑simulator

**Purpose:** Generate training data by simulating fleet interactions.

- **To plato‑ensign:** No direct connection. Simulator does not call export functions.

## Identified Gaps (P1 Safe Channels)
1. **Ghost tile flow is file‑based, not API‑driven.**  
   - plato‑torch writes `ghost_tiles.json` → holodeck‑rust reads same file.  
   - Risk: stale data, no synchronization.  
   - Safe channel: implement a simple HTTP endpoint in holodeck‑rust for on‑demand injection.

2. **fleet‑simulator tiles are ingested but not validated for room compatibility.**  
   - Tile schema matches, but no feedback loop to improve simulations.  
   - Safe channel: add a `room_feedback` field to tiles that simulator agents can learn from.

3. **holodeck‑rust NPC sentiment does not influence plato‑torch training.**  
   - NPC emotions could weight tile importance (e.g., “angry” NPC tiles are high‑priority P0 examples).  
   - Safe channel: extend `tile_ingestor.py` to accept a `sentiment_score` from holodeck.
