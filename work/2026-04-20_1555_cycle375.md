## 1. plato-torch

**Location:** `plato-torch/`  
**Purpose:** 25 training room presets, self-training rooms. Core training environment.

**Key files inspected:**
- `plato-torch/src/rooms/deadband_room.py` ‚Äî defines `DeadbandRoom` class
- `plato-torch/src/rooms/__init__.py` ‚Äî exports room classes
- `plato-torch/src/training/trainer.py` ‚Äî main training loop
- `plato-torch/requirements.txt` ‚Äî dependencies

**Connections found:**
- **plato-ensign:** `plato-torch/src/rooms/base_room.py` imports `from plato_ensign.export import export_lora` (conditional import). Used to export room experience as LoRA adapters.
- **fleet-simulator:** No direct import found in code. However, `plato-torch/README.md` mentions "simulated fleet data" as a possible input source.
- **holodeck-rust:** No direct import or reference found.
- **External:** Uses `torch`, `transformers`, `datasets` (Hugging Face). Training data can come from local JSONL files or Hugging Face datasets.

**Integration notes:**
- `DeadbandRoom` implements the Deadband Protocol (P0/P1/P2) as training stages.
- Rooms produce "tiles" (interaction records) stored in `room.history`.
- `plato-ensign` integration is optional ‚Äî only if the package is installed and called explicitly.

## 6. Integration Gaps & Opportunities

**Gap 1: Direct plato-torch ‚Üí fleet-simulator pipeline**  
- Currently manual: run simulator, save JSONL, point plato-torch at file.
- Opportunity: Could build a `SimulatorRoom` in plato-torch that pulls live data from a running simulator via IPC/API.

**Gap 2: holodeck-rust sentiment in plato-torch training**  
- NPC sentiment data is trapped inside Rust simulator.
- Opportunity: Export sentiment tags in JSONL, or create a Python wrapper for holodeck to use directly in rooms.

**Gap 3: plato-ensign only works with plato-torch rooms**  
- Could also export from fleet-simulator logs (agent trajectories).
- Opportunity: Add `export_lora_from_logs(log_file)` function.

**Gap 4: No shared configuration or service discovery**  
- Each repo has its own config format.
- Opportunity: Define `fleet-config` schema (YAML/TOML) used by all components.
